History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Canvas was initially introduced by <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a> for use in their own <a href="/wiki/Mac_OS_X" class="mw-redirect" title="Mac OS X">Mac OS X</a> <a href="/wiki/WebKit" title="WebKit">WebKit</a> component in 2004,<sup id="cite_ref-1" class="reference"><a Later, in 2005 it was adopted in version 1.8 of <a href="/wiki/Gecko_(layout_engine)" class="mw-redirect" title="Gecko (layout engine)">Gecko</a> browsers,<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> and <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a> in 2006,<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> and standardized by the <a href="/wiki/Web_Hypertext_Application_Technology_Working_Group" class="mw-redirect" title="Web Hypertext Application Technology Working Group">Web Hypertext Application Technology Working Group</a> (WHATWG) on new proposed specifications for next generation web technologies.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (February 2011)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Canvas_element&amp;action=edit&amp;section=2" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>A <code>canvas</code> consists of a drawable region defined in <a href="/wiki/HTML" title="HTML">HTML</a> code with <i>height</i> and <i>width</i> attributes. <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> code may access the area through a full set of drawing functions similar to those of other common 2D APIs, thus allowing for dynamically generated graphics. Setting the element\'s width and height attributes sets both of these sizes; CSS attributes affect only the element\'s size and not the drawing surface.\n</p><p>By default, both the canvas element\'s size and the size of its drawing surface is 300 screen pixels wide and 150 screen pixels high. In the listing shown in the example, which uses CSS to set the canvas element\'s size, the size of the element is 600 pixels wide and 300 pixels high, but the size of the drawing surface remains unchanged at the default value of 300 pixels \xc3\x97 150 pixels. id="Canvas_versus_Scalable_Vector_Graphics_(SVG)">Canvas versus Scalable Vector Graphics (SVG)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Canvas_element&amp;action=edit&amp;section=5" title="Edit section: Canvas versus Scalable Vector Graphics (SVG)">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<table class="box-Unreferenced_section plainlinks metadata ambox ambox-content ambox-Unreferenced" role="presentation"><tbody><tr><td class="mbox-image"><div style="width:52px"><a href="/wiki/File:Question_book-new.svg" class="image"><img alt="" class="mbox-text-span">This section <b>does not <a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">cite</a> any <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">sources</a></b>.<span class="hide-when-compact"> Please help <a class="external text" Unsourced material may be challenged and <a href="/wiki/Wikipedia:Verifiability#Burden_of_evidence" title="Wikipedia:Verifiability">removed</a>.</span>  <small class="date-container"><i>(<span class="date">October 2017</span>)</i></small><small class="hide-when-compact"><i> (<a href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this template message</a>)</i></small></div></td></tr></tbody></table>\n<p><a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">SVG</a> is an earlier standard for drawing shapes in browsers. However, unlike canvas, which is <a href="/wiki/Raster_graphics" title="Raster graphics">raster</a>-based, SVG is <a href="/wiki/Vector_graphics" title="Vector graphics">vector</a>-based, so that each drawn shape is remembered as an object in a <a href="/wiki/Scene_graph" title="Scene graph">scene graph</a> or <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a>, which is subsequently rendered to a bitmap.  This means that if attributes of an SVG object are changed, the browser can automatically re-render the scene.\n</p><p>Canvas objects, on the other hand, are drawn in <a href="/wiki/Immediate_mode_(computer_graphics)" title="Immediate mode (computer graphics)">immediate mode</a>. In the canvas example above, once the rectangle is drawn the model it was drawn from is forgotten by the system. If its position were to be changed, the entire scene would need to be redrawn, including any objects that might have been covered by the rectangle. But in the equivalent SVG case, one could simply change the position attributes of the rectangle and the browser would determine how to repaint it. There are additional JavaScript libraries that add scene-graph capabilities to the canvas element. It is also possible to paint a canvas in layers and then recreate specific layers.\n</p><p>SVG images are represented in <a href="/wiki/XML" title="XML">XML</a>, and complex scenes can be created and maintained with XML editing tools.\n</p><p>The SVG scene graph enables <a href="/wiki/Event_(computing)" title="Event (computing)">event handlers</a> to be associated with objects, so a rectangle may respond to an <code>onClick</code> event. To get the same functionality with canvas, one must manually match the coordinates of the mouse click with the coordinates of the drawn rectangle to determine whether it was clicked.\n</p><p>Conceptually, canvas is a lower-level API upon which an engine, supporting for example SVG, might be built. There are JavaScript libraries that provide partial SVG implementations using canvas for browsers that do not provide SVG but support canvas, such as the browsers in Android 2.x. However, this is not normally the case\xe2\x80\x94they are independent standards. The situation is complicated because there are scene graph libraries for canvas, and SVG has some bitmap manipulation functionality.\n</p>\n<h2><span class="mw-headline" id="Reactions">Reactions</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Canvas_element&amp;action=edit&amp;section=6" title="Edit section: Reactions">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>At the time of its introduction the canvas element was met with mixed reactions from the web standards community. There have been arguments against Apple\'s decision to create a new proprietary element instead of supporting the <a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">SVG</a> standard. There are other concerns about syntax, such as the absence of a <a href="/wiki/Namespace" title="Namespace">namespace</a>.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Intellectual_property_over_canvas">Intellectual property over canvas</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Canvas_element&amp;action=edit&amp;section=7" title="Edit section: Intellectual property over canvas">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>On March 14, 2007, WebKit developer <a href="/wiki/Dave_Hyatt" title="Dave Hyatt">Dave Hyatt</a> forwarded an email from Apple\'s Senior Patent Counsel, Helene Plotka Workman,<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup> which stated that Apple reserved all <a href="/wiki/Intellectual_property" title="Intellectual property">intellectual property</a> rights relative to WHATWG\'s Web Applications 1.0 Working Draft, dated March 24, 2005, Section 10.1, entitled \xe2\x80\x9cGraphics:  The bitmap canvas\xe2\x80\x9d,<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> but left the door open to licensing the patents should the specification be transferred to a standards body with a <a href="/wiki/Patent" title="Patent">formal patent policy</a>. This caused considerable discussion among web developers, and raised questions concerning the WHATWG\'s lack of a policy on patents in comparison to the <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a> (W3C)\'s explicit favoring of royalty-free licenses. Apple later disclosed the patents under the W3C\'s royalty-free patent licensing terms.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> The disclosure means that Apple is required to provide royalty-free licensing for the patent whenever the Canvas element becomes part of a future W3C recommendation created by the HTML working group.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Privacy_concerns">Privacy concerns</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Canvas_element&amp;action=edit&amp;section=8" title="Edit section: Privacy concerns">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p><a href="/wiki/Canvas_fingerprinting" title="Canvas fingerprinting">Canvas fingerprinting</a> is one of a number of <a href="/wiki/Browser_fingerprinting" class="mw-redirect" title="Browser fingerprinting">browser fingerprinting</a> techniques of tracking online users that allow websites to identify and track visitors using <a href="/wiki/HTML5" title="HTML5">HTML5</a> canvas element. The technique received wide media coverage in 2014<sup id="cite_ref-Knibbs_9-0" class="reference"><a href="#cite_note-Knibbs-9">&#91;9&#93;</a></sup><sup id="cite_ref-Steinberg_10-0" class="reference"><a href="#cite_note-Steinberg-10">&#91;10&#93;</a></sup><sup id="cite_ref-Angwin_11-0" class="reference"><a href="#cite_note-Angwin-11">&#91;11&#93;</a></sup><sup id="cite_ref-Kirk_12-0" class="reference"><a href="#cite_note-Kirk-12">&#91;12&#93;</a></sup> after researchers from <a href="/wiki/Princeton_University" title="Princeton University">Princeton University</a> and <a href="/wiki/Katholieke_Universiteit_Leuven" class="mw-redirect" title="Katholieke Universiteit Leuven">KU Leuven University</a> described it in their paper <i>The Web never forgets</i>.<sup id="cite_ref-WebNeverForgets_13-0" class="reference"><a href="#cite_note-WebNeverForgets-13">&#91;13&#93;</a></sup> The privacy concerns regarding canvas fingerprinting centre around the fact that even deleting cookies and clearing the cache will not be sufficient for users to avoid online tracking.\n</p>\n<h2><span class="mw-headline" id="Browser_support">Browser support</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Canvas_element&amp;action=edit&amp;section=9" title="Edit section: Browser
