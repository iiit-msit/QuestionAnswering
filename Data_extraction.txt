The <a href="/wiki/Data_import" class="mw-redirect" title="Data import">import</a> into the intermediate extracting system is thus usually followed by <a href="/wiki/Data_transformation" title="Data transformation">data transformation</a> and possibly the addition of <a href="/wiki/Metadata" title="Metadata">metadata</a> prior to <a href="/wiki/Data_export" class="mw-redirect" title="Data export">export</a> to another stage in the data <a href="/wiki/Workflow" title="Workflow">workflow</a>.\n</p><p>Usually, the term data extraction is applied when (<a href="/wiki/Experiment" title="Experiment">experimental</a>) data is first imported into a computer from primary sources, like <a href="/wiki/Measuring_device" class="mw-redirect" title="Measuring device">measuring</a> or <a href="/wiki/Data_logger" title="Data logger">recording devices</a>. Today\'s <a href="/wiki/Electronic_device" class="mw-redirect" title="Electronic device">electronic devices</a> will usually present an <a href="/wiki/Electrical_connector" title="Electrical connector">electrical connector</a> (e.g. <a href="/wiki/USB" title="USB">USB</a>) through which \'<a href="/wiki/Raw_data" title="Raw data">raw data</a>\' can be <a href="/wiki/Data_stream" title="Data stream">streamed</a> into a <a href="/wiki/Personal_computer" title="Personal computer">personal computer</a>.\n</p>\n<div id="toc" class="toc" role="navigation" class="mw-headline" id="Data_sources">Data sources</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Data_extraction&amp;action=edit&amp;section=1" title="Edit section: Data sources">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Typical unstructured data sources include <a href="/wiki/Web_page" title="Web page">web pages</a>, <a href="/wiki/Email" title="Email">emails</a>, documents, <a href="/wiki/PDF" title="PDF">PDFs</a>, scanned text, mainframe reports, spool files, classifieds, etc. which is further used for sales or marketing leads. Extracting data from these unstructured sources has grown into a considerable technical challenge where as historically data extraction has had to deal with changes in physical hardware formats, the majority of current data extraction deals with extracting data from these unstructured data sources, and from different software formats. This growing process of data extraction<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> from the web is referred to as "Web data extraction" or "<a href="/wiki/Web_scraping" title="Web scraping">Web scraping</a>".\n</p>\n<h2><span class="mw-headline" id="Imposing_structure">Imposing structure</span><span class="mw-editsection"><span class="mw-editsection-bracket">]</span></span></h2>\n<p>The act of adding structure to unstructured data takes a number of forms\n</p>\n<ul><li>Using text <a href="/wiki/Pattern_matching" title="Pattern matching">pattern matching</a> such as <a href="/wiki/Regular_expression" title="Regular expression">regular expressions</a> to identify small or large-scale structure e.g. records in a report and their associated data from headers and footers;</li>\n<li>Using a table-based approach to identify common sections within a limited domain e.g. in emailed resumes, identifying skills, previous work experience, qualifications etc. using a standard set of commonly used headings (these would differ from language to language), e.g. Education might be found under Education/Qualification/Courses;</li>\n<li>Using <a href="/wiki/Text_analytics" class="mw-redirect" title="Text analytics">text analytics</a> to attempt to understand the text and link it to other
