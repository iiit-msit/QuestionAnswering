Screen readers are essential to people who are <a href="/wiki/Blindness" class="mw-redirect" title="Blindness">blind</a>,<sup id="cite_ref-Braille_display_2-0" class="reference"><a href="#cite_note-Braille_display-2">&#91;2&#93;</a></sup> and are useful to people who are <a href="/wiki/Visual_impairment" title="Visual impairment">visually impaired</a>,<sup id="cite_ref-Braille_display_2-1" class="reference"><a href="#cite_note-Braille_display-2">&#91;2&#93;</a></sup> <a href="/wiki/Illiteracy" class="mw-redirect" title="Illiteracy">illiterate</a>, or have a <a href="/wiki/Learning_disability" title="Learning disability">learning disability</a>.<sup id="cite_ref-Screen1_3-0" class="reference"><a href="#cite_note-Screen1-3">&#91;3&#93;</a></sup> Screen readers are <a href="/wiki/Application_software" title="Application software">software applications</a> that attempt to convey what people with normal eyesight see on a <a href="/wiki/Display_device" title="Display device">display</a> to their users via non-visual means, like <a href="/wiki/Text-to-speech" class="mw-redirect" title="Text-to-speech">text-to-speech</a>,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> sound icons,<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup> or a <a href="/wiki/Refreshable_Braille_display" class="mw-redirect" title="Refreshable Braille display">Braille device</a>.<sup id="cite_ref-Braille_display_2-2" class="reference"><a href="#cite_note-Braille_display-2">&#91;2&#93;</a></sup> They do this by applying a wide variety of techniques that include, for example, interacting with dedicated <a href="#Accessibility_APIs">accessibility APIs</a>, using various <a href="/wiki/Operating_system" title="Operating system">operating system</a> features (like <a href="/wiki/Inter-process_communication" title="Inter-process communication">inter-process communication</a> and querying <a href="/wiki/User_interface" title="User interface">user interface</a> properties), and employing <a href="/wiki/Hooking" title="Hooking">hooking</a> techniques.<sup id="cite_ref-SR_Overview_6-0" class="reference"><a href="#cite_note-SR_Overview-6">&#91;6&#93;</a></sup>\n</p><p><a href="/wiki/Microsoft_Windows" title="Microsoft Windows">Microsoft Windows</a> <a href="/wiki/Operating_systems" class="mw-redirect" title="Operating systems">operating systems</a> have included the <a href="/wiki/Microsoft_Narrator" title="Microsoft Narrator">Microsoft Narrator</a> screen reader since <a href="/wiki/Windows_2000" title="Windows 2000">Windows 2000</a>, though separate products such as the <a href="/wiki/Free_and_open_source" class="mw-redirect" title="Free and open source">free and open source</a> screen reader <a href="/wiki/NonVisual_Desktop_Access" title="NonVisual Desktop Access">NVDA</a> by NV Access and <a href="/wiki/Freedom_Scientific" title="Freedom Scientific">Freedom Scientific</a>\'s commercially available <a href="/wiki/JAWS_(screen_reader)" title="JAWS (screen reader)">JAWS</a> screen reader and <a href="/wiki/ZoomText" title="ZoomText">ZoomText</a> screen magnifier are more popular for that operating system.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> <a href="/wiki/Apple_Inc." title="Apple Inc.">Apple Inc.</a>\'s <a href="/wiki/MacOS" title="MacOS">macOS</a>, <a href="/wiki/IOS" title="IOS">iOS</a>, and <a href="/wiki/TvOS" title="TvOS">tvOS</a> include <a href="/wiki/VoiceOver" title="VoiceOver">VoiceOver</a> as a built-in screen reader, while <a href="/wiki/Google" title="Google">Google</a>\'s <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a> provides the <a href="/wiki/Google_TalkBack" title="Google TalkBack">Talkback screen reader</a> and its <a href="/wiki/Chrome_OS" title="Chrome OS">Chrome OS</a> can use ChromeVox.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> Similarly, Android-based devices from Amazon provide the VoiceView screen reader.  There are also free and open source screen readers for <a href="/wiki/Linux" title="Linux">Linux</a> and <a href="/wiki/Unix-like" title="Unix-like">Unix-like</a> systems, such as Speakup and <a href="/wiki/Orca_(assistive_technology)" title="Orca (assistive technology)">Orca</a>.\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=1" title="Edit section: Types">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<h3><span id="Command-line_.28text.29"></span><span class="mw-headline" id="Command-line_(text)">Command-line (text)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=2" title="Edit section: Command-line (text)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>In early <a href="/wiki/Operating_system" title="Operating system">operating systems</a>, such as <a href="/wiki/MS-DOS" title="MS-DOS">MS-DOS</a>, which employed <a href="/wiki/Command-line_interface" title="Command-line interface">command-line interfaces</a> (<abbr>CLI</abbr>s), the screen display consisted of <a href="/wiki/Character_(computing)" title="Character (computing)">characters</a> <a href="/wiki/Memory-mapped_I/O" title="Memory-mapped I/O">mapping</a> directly to a <a href="/wiki/Screen_buffer" class="mw-redirect" title="Screen buffer">screen buffer</a> in <a href="/wiki/Computer_memory" title="Computer memory">memory</a> and a <a href="/wiki/Cursor_(computers)" class="mw-redirect" title="Cursor (computers)">cursor</a> position. Input was by keyboard. All this information could therefore be obtained from the system either by <a href="/wiki/Hooking" title="Hooking">hooking</a> the flow of information around the system and reading the screen buffer or by using a standard hardware output socket<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup> and communicating the results to the user.\n</p><p>In the 1980s, the Research Centre for the Education of the Visually Handicapped (<abbr>RCEVH</abbr>) at the <a href="/wiki/University_of_Birmingham" title="University of Birmingham">University of Birmingham</a> developed Screen Reader for the <a href="/wiki/BBC_Micro" title="BBC Micro">BBC Micro</a> and NEC Portable.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Graphical">Graphical</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=3" title="Edit section: Graphical">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<h4><span class="mw-headline" id="Off-screen_models">Off-screen models</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=4" title="Edit section: Off-screen models">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>With the arrival of <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interfaces</a> (<abbr>GUI</abbr>s), the situation became more complicated. A <abbr>GUI</abbr> has characters and graphics drawn on the screen at particular positions, and therefore there is no purely textual representation of the graphical contents of the display. Screen readers were therefore forced to employ new low-level techniques, gathering messages from the <a href="/wiki/Operating_system" title="Operating system">operating system</a> and using these to build up an "off-screen model", a representation of the display in which the required text content is stored.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">&#91;12&#93;</a></sup>\n</p><p>For example, the operating system might send messages to draw a command button and its caption. These messages are intercepted and used to construct the off-screen model. The user can switch between controls (such as buttons) available on the screen and the captions and control contents will be read aloud and/or shown on refreshable <a href="/wiki/Braille" title="Braille">Braille</a> display.\n</p><p>Screen readers can also communicate information on menus, controls, and other visual constructs to permit blind users to interact with these constructs. However, maintaining an off-screen model is a significant technical challenge; hooking the low-level messages and maintaining an accurate model are both difficult tasks.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h4><span class="mw-headline" id="Accessibility_APIs">Accessibility APIs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=5" title="Edit section: Accessibility APIs">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>Operating system and application designers have attempted to address these problems by providing ways for screen readers to access the display contents without having to maintain an off-screen model. These involve the provision of alternative and accessible representations of what is being displayed on the screen accessed through an <abbr><a href="/wiki/Application_programming_interface" class="mw-redirect" title="Application programming interface">API</a></abbr>. Existing <abbr>API</abbr>s include:\n</p>\n<ul><li>Android Accessibility Framework<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup></li>\n<li>Apple Accessibility <abbr>API</abbr><sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup></li>\n<li><a href="/wiki/AT-SPI" class="mw-redirect" title="AT-SPI">AT-SPI</a></li>\n<li><a href="/wiki/IAccessible2" title="IAccessible2">IAccessible2</a></li>\n<li><a href="/wiki/Microsoft_Active_Accessibility" title="Microsoft Active Accessibility">Microsoft Active Accessibility</a> (<abbr>MSAA</abbr>)</li>\n<li><a href="/wiki/Microsoft_UI_Automation" title="Microsoft UI Automation">Microsoft UI Automation</a></li>\n<li><a href="/wiki/Java_Access_Bridge" title="Java Access Bridge">Java Access Bridge</a><sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup></li></ul>\n<p>Screen readers can query the operating system or application for what is currently being displayed and receive updates when the display changes. For example, a screen reader can be told that the current focus is on a button and the button caption to be communicated to the user. This approach is considerably easier for the developers of screen readers, but fails when applications do not comply with the accessibility <abbr>API</abbr>: for example, <a href="/wiki/Microsoft_Word" title="Microsoft Word">Microsoft Word</a> does not comply with the <abbr>MSAA</abbr> <abbr>API</abbr>, so screen readers must still maintain an off-screen model for Word or find another way to access its contents.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup> One approach is to use available operating system messages and application object models to supplement accessibility <abbr>API</abbr>s.\n</p><p>Screen readers can be assumed to be able to access all display content that is not intrinsically inaccessible. Web browsers, word processors, icons and windows and email programs are just some of the applications used successfully by screen reader users. However, according to some users,<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Words_to_watch#Unsupported_attributions" title="Wikipedia:Manual of Style/Words to watch"><span title="The material near this tag possibly uses too-vague attribution or weasel words. (January 2015)">who?</span></a></i>&#93;</sup> using a screen reader is considerably more difficult than using a GUI, and many applications have specific problems resulting from the nature of the application (e.g. animations) or failure to comply with accessibility standards for the platform (e.g. Microsoft Word and Active Accessibility).<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h3><span class="mw-headline" id="Self-voicing_programs_and_applications">Self-voicing programs and applications</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=6" title="Edit section: Self-voicing programs and applications">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Some programs and applications have voicing technology built in alongside their primary functionality. These programs are termed <a href="/wiki/Self-voicing" title="Self-voicing">self-voicing</a> and can be a form of <a href="/wiki/Assistive_technology" title="Assistive technology">assistive technology</a> if they are designed to remove the need to use a screen reader.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h3><span class="mw-headline" id="Cloud-based">Cloud-based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=7" title="Edit section: Cloud-based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Some telephone services allow users to interact with the internet remotely. For example, TeleTender can read web pages over the phone and does not require special programs or devices on the user side.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h3><span class="mw-headline" id="Web-based">Web-based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=8" title="Edit section: Web-based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>A relatively new development in the field is web-based applications like Spoken-Web that act as web portals, managing content like news updates, weather, science and business articles for visually-impaired or blind computer users.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup> Other examples are ReadSpeaker or <a href="/wiki/BrowseAloud" title="BrowseAloud">BrowseAloud</a> that add <a href="/wiki/Text-to-speech" class="mw-redirect" title="Text-to-speech">text-to-speech</a> functionality to web content.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup> The primary audience for such applications is those who have difficulty reading because of learning disabilities or language barriers.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup> Although functionality remains limited compared to equivalent desktop applications, the major benefit is to increase the accessibility of said websites when viewed on public machines where users do not have permission to install custom software, giving people greater "freedom to roam".<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p><p>With the development of <a href="/wiki/Smartphone" title="Smartphone">smartphones</a>, the ability to listen to written documents (textual web content, <abbr><a href="/wiki/Portable_Document_Format" class="mw-redirect" title="Portable Document Format">PDF</a></abbr> documents, e-mails etc.) while driving or during a similar activity in the same way that listening to music, will benefit a much broader audience than visually-impaired people.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup> The best-known examples are <a href="/wiki/Siri_(software)" class="mw-redirect" title="Siri (software)">Siri</a> for <a href="/wiki/IOS" title="IOS">iOS</a>, and <a href="/wiki/Google_Now" title="Google Now">Google Now</a> and <a href="/w/index.php?title=Iris_(software)&amp;action=edit&amp;redlink=1" class="new" title="Iris (software) (page does not exist)">Iris</a> for <a href="/wiki/Android_(operating_system)" title="Android (operating system)">Android</a>. With the release of the <a href="/wiki/Samsung_Galaxy_S_III" title="Samsung Galaxy S III">Galaxy S III</a>, <a href="/wiki/Samsung" title="Samsung">Samsung</a> also introduced a similar <a href="/wiki/Intelligent_personal_assistant" class="mw-redirect" title="Intelligent personal assistant">intelligent personal assistant</a> called <a href="/wiki/S_Voice" title="S Voice">S Voice</a>. On the <a href="/wiki/BlackBerry_10_operating_system_voice_control" class="mw-redirect" title="BlackBerry 10 operating system voice control">BlackBerry 10 operating system</a>, their <a href="/wiki/BlackBerry_Z30" title="BlackBerry Z30">Z30 smartphone</a> also features spoken interaction features, which are similar to the other <a href="/wiki/Mobile_operating_system" title="Mobile operating system">mobile operating systems</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="Similarity unverified. (October 2013)">citation needed</span></a></i>&#93;</sup>\n</p><p>This functionality depends on the quality of the software but also on a logical structure of the text. Use of headings, punctuation, presence of alternate attributes for images, etc. is crucial for a good vocalization. Also a web site may have a nice look because of the use of appropriate two dimensional positioning with CSS but its standard linearization, for example, by suppressing any CSS and Javascript in the browser may not be comprehensible.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h2><span class="mw-headline" id="Customization">Customization</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=9" title="Edit section: Customization">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Most screen readers allow the user to select whether most <a href="/wiki/Punctuation" title="Punctuation">punctuation</a> is announced or silently ignored. Some screen readers can be tailored to a particular application through <a href="/wiki/Script_(computing)" class="mw-redirect" title="Script (computing)"><b>scripting</b></a>. One advantage of scripting is that it allows customizations to be shared among users, increasing accessibility for all. <abbr><a href="/wiki/Job_Access_With_Speech" class="mw-redirect" title="Job Access With Speech">JAWS</a></abbr> enjoys an active script-sharing community, for example.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h3><span class="mw-headline" id="Verbosity">Verbosity</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=10" title="Edit section: Verbosity">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Verbosity is a feature of screen reading software that supports vision-impaired computer users. Speech verbosity controls enable users to choose how much speech feedback they wish to hear. Specifically, verbosity settings allow users to construct a mental model of web pages displayed on their computer screen. Based on verbosity settings, a screen-reading program informs users of certain formatting changes, such as when a frame or table begins and ends, where graphics have been inserted into the text, or when a list appears in the document.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h3><span class="mw-headline" id="Language">Language</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Screen_reader&amp;action=edit&amp;section=11" title="Edit section: Language">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Some screen readers can read text in more than one <a href="/wiki/Language" title="Language">language</a>, provided that the language of the material is encoded in its <a href="/wiki/Metadata_Encoding_and_Transmission_Standard" title="Metadata Encoding and Transmission Standard">metadata</a>.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup>\n</p><p>Some screen reading programs<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Avoid_weasel_words" class="mw-redirect" title="Wikipedia:Avoid weasel words"><span title="The material near this tag possibly uses too vague attribution or weasel words. (January 2015)">which?</span></a></i>&#93;</sup> also include language verbosity, which automatically detects verbosity settings related to speech output language. For example, if a user navigated to a website based in the United Kingdom, the text would be read with <a href="/wiki/British_English" title="British English">an English accent</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (January 2015)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
