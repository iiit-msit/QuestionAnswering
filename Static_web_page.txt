message</a>)</i></small></div></td></tr></tbody></table>\n<p>A <b>static web page</b> (sometimes called a <b>flat page</b> or a <b>stationary page</b>) is a <a href="/wiki/Web_page" title="Web page">web page</a> that is delivered to the user\'s web <a href="/wiki/Web_browser" title="Web browser">browser</a> exactly as stored,<sup id="cite_ref-Melendez,_SmallBussiness,_2018_1-0" class="reference"><a href="#cite_note-Melendez,_SmallBussiness,_2018-1">&#91;1&#93;</a></sup> in contrast to <a href="/wiki/Dynamic_web_page" title="Dynamic web page">dynamic web pages</a> which are generated by a web application.<sup id="cite_ref-Encyclopedia,_PCMag,_2017_2-0" class="reference"><a href="#cite_note-Encyclopedia,_PCMag,_2017-2">&#91;2&#93;</a></sup>\n</p><p>Consequently, a static web page displays the same information for all users, from all contexts, subject to modern capabilities of a <a href="/wiki/Web_server" title="Web server">web server</a> to <a href="/wiki/Content_negotiation" title="Content negotiation">negotiate</a> <a href="/wiki/MIME_type" class="mw-redirect" title="MIME type">content-type</a> or language of the document where such versions are available and the <a href="/wiki/Server_(computing)" title="Server (computing)">server</a> is configured to do so.<sup id="cite_ref-morphenZero,_2016_3-0" class="reference"><a href="#cite_note-morphenZero,_2016-3">&#91;3&#93;</a></sup>\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" role="button" id="Overview">Overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Static_web_page&amp;action=edit&amp;section=1" title="Edit section: Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Static web pages are often <a href="/wiki/HTML" title="HTML">HTML</a> documents<sup id="cite_ref-Techopedia,_2018_4-0" class="reference"><a href="#cite_note-Techopedia,_2018-4">&#91;4&#93;</a></sup> stored as files in the <a href="/wiki/File_system" title="File system">file system</a> and made available by the web server over <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> (nevertheless <a href="/wiki/URL" title="URL">URLs</a> ending with ".html" are not always static). However, loose interpretations of the term could include web pages stored in a <a href="/wiki/Database" title="Database">database</a>, and could even include pages formatted using a template and served through an application server, as long as the page served is unchanging and presented essentially as stored.\n</p><p>Static web pages are suitable for content that never or rarely needs to be updated, though modern web template systems are changing this. Maintaining large numbers of static pages as files can be impractical without automated tools, such as <a href="/wiki/Web_template_system#Static_site_generators" title="Web template system">static site generators</a>. Another way to manage static pages is <a href="/wiki/Comparison_of_online_source_code_playgrounds#Online_compiled_source_code_playgrounds" title="Comparison of online source code playgrounds">online compiled source code playgrounds</a>, e.g. <a href="/w/index.php?title=GatsbyJS&amp;action=edit&amp;redlink=1" class="new" title="GatsbyJS (page does not exist)">GatsbyJS</a> and <a href="/wiki/GitHub" title="GitHub">GitHub</a> may be utilized for migrating a <a href="/wiki/WordPress" title="WordPress">WordPress</a> site into static web pages.<sup id="cite_ref-Rascia,_2019_5-0" class="reference"><a href="#cite_note-Rascia,_2019-5">&#91;5&#93;</a></sup> Any personalization or interactivity has to run client-side, which is restricting.<sup id="cite_ref-javascript_6-0" class="reference"><a href="#cite_note-javascript-6">&#91;6&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Advantages_of_a_static_website">Advantages of a static website</span><span title="Web shell">web shell</a> attacks if a <a href="/wiki/Vulnerability_(computing)" title="Vulnerability (computing)">vulnerability</a> is present)<sup id="cite_ref-CloudCannon,_2017_7-0" class="reference"><a href="#cite_note-CloudCannon,_2017-7">&#91;7&#93;</a></sup></li>\n<li>Improved performance for end users compared to dynamic websites<sup id="cite_ref-:0_8-0" class="reference"><a href="#cite_note-:0-8">&#91;8&#93;</a></sup></li>\n<li>Fewer or no dependencies on systems such as databases or other application servers <sup id="cite_ref-Bou\xc3\xa7as,_2019_9-0" class="reference"><a href="#cite_note-Bou\xc3\xa7as,_2019-9">&#91;9&#93;</a></sup></li>\n<li>Cost savings from utilizing cloud storage, as opposed to a hosted environment<sup id="cite_ref-Vince,_2018_10-0" class="reference"><a href="#cite_note-Vince,_2018-10">&#91;10&#93;</a></sup></li>\n<li>Security configurations are easy to setup, which makes it more secure<sup
