HTML attributes are a modifier of an <i><a href="/wiki/HTML_element" title="HTML element">HTML element type</a></i>. An attribute either modifies the default functionality of an element type or provides functionality to certain element types unable to function correctly without them. In HTML syntax, an attribute is added to an <i><a href="/wiki/HTML_tags" class="mw-redirect" title="HTML tags">HTML start tag</a></i>.\n</p><p>Several basic attributes types have been recognized, including: (1) <i>required attributes</i>, needed by a particular element type for that element type to function correctly; (2) <i>optional attributes</i>, used to modify the default functionality of an element type; (3) <i>standard attributes</i>, supported by many element types; and (4) <i>event attributes</i>, used to cause element types to specify scripts to be run under specific circumstances.\n</p><p>Some attribute types function differently when used to modify different element types. For example, the attribute <i>name</i> is used by several element types, but has slightly different functions in each.<sup id="cite_ref-html4attributes_1-0" class="reference"><a href="#cite_note-html4attributes-1">&#91;1&#93;</a></sup>\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" role="button" provided <code>value</code>. The value may be enclosed in single or double quotes, although values consisting of certain characters can be left unquoted in HTML (but not XHTML).<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> Leaving attribute values unquoted is considered unsafe.<sup id="cite_ref-4" class="mw-editsection-bracket">]</span></span></h3>\n<p>Usually, <a href="/wiki/HTML_element" title="HTML element">HTML elements</a> can take any of several most common standard attributes (<a href="#Standard_attributes">See the complete list</a>):\n</p>\n<ul><li>The <b><code>id</code></b> attribute provides a document-wide unique identifier for an element.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup><sup id="cite_ref-9" class="reference"><a to focus attention on the specific element, or by scripts to alter the contents or presentation of an element. Appended to the URL of the page, the URL directly targets the specific element within the document, typically a sub-section of the page. For example, the ID "Attributes" in <code>http://en.wikipedia.org/wiki/HTML#Attributes</code> (to refer to the section "Attributes" in the "HTML" page).</li>\n<li>The <b><code>class</code></b> attribute provides a way of classifying similar elements.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup> This can be used for <a href="/wiki/Semantics" title="Semantics">semantic</a> purposes, or for presentation purposes. Semantically, for example, classes are used in <a href="/wiki/Microformat" title="Microformat">microformats</a>. Presentationally, for example, an HTML document might use the designation <code>class="notation"</code> to indicate that all elements with this class value are subordinate to the main text of the document. Such elements might be gathered together and presented as footnotes on a page instead of appearing in the place where they occur in the HTML source. Another presentation use would be as a <a href="/wiki/CSS_selector" class="mw-redirect" title="CSS selector">CSS selector</a>.</li>\n<li>An author may use the <b><code>style</code></b> non-attributal codes presentational properties to a particular element.The <b><code>style</code></b> attribute can be used on any HTML element (it will validate on any HTML element  element; however, it is not necessarily useful). It is considered better practice to use an element\'s <code>id</code> or <code>class</code> attributes to select the element with a <a href="/wiki/Style_sheet_(web_development)" title="Style sheet (web development)">stylesheet</a>, though sometimes this can be too cumbersome for a simple and specific or ad hoc application of styled properties.</li>\n<li>The <b><code>title</code></b> attribute is used to attach subtextual explanation to an element. In most browsers this attribute is displayed as what is often referred to as a <a href="/wiki/Tooltip" title="Tooltip">tooltip</a>.</li></ul>\n<h2><span class="mw-headline"
