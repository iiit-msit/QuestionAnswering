search</a>\n\t\t<div id="mw-content-text" lang="en" dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><p>Change detection and notification refers to automatic detection of changes made to <a href="/wiki/World_Wide_Web" title="World Wide Web">World Wide Web</a> pages and notification to interested users by email or other means.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> Whereas <a href="/wiki/Search_engine" title="Search engine">search engines</a> are designed to find web pages, CDN systems are designed to monitor changes to web pages. Before change detection and notification, it was necessary for users to manually check for web page changes, either by revisiting web sites or periodically searching again. Efficient and effective change detection and notification is hampered by the fact that most servers do not accurately track content changes through Last-Modified or <a href="/wiki/HTTP_ETag" title="HTTP ETag">ETag</a> headers. A comprehensive analysis regarding CDN systems can be found\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" role="button" id="toctogglecheckbox" class="toctogglecheckbox" style="display:none" /><div class="toctitle" lang="en" dir="ltr"><h2 id="mw-toc-heading">Contents</h2><span class="toctogglespan"><label class="toctogglelabel" for="toctogglecheckbox"></label></span></div>\n<ul>\n<li class="toclevel-1 tocsection-1"><a href="#History"><span class="tocnumber">1</span> <span class="toctext">History</span></a></li>\n<li class="toclevel-1 tocsection-2"><a href="#Architectural_approaches"><span class="tocnumber">2</span> <span class="toctext">Architectural approaches</span></a>\n<ul>\n<li class="toclevel-2 tocsection-3"><a href="#Server_based"><span class="tocnumber">2.1</span> <span class="toctext">Server based</span></a></li>\n<li class="toclevel-2 tocsection-4"><a href="#Client_based"><span class="tocnumber">2.2</span> <span class="toctext">Client based</span></a></li>\n</ul>\n</li>\n<li class="toclevel-1 tocsection-5"><a href="#Considerations"><span class="tocnumber">3</span> <span class="toctext">Considerations</span></a></li>\n<li class="toclevel-1 tocsection-6"><a href="#References"><span class="tocnumber">4</span> <span class="toctext">References</span></a></li>\n</ul>\n</div>\n\n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Change_detection_and_notification&amp;action=edit&amp;section=1" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>In 1996, <a href="/wiki/NetMind" title="NetMind">NetMind</a> developed the first change detection and notification tool, known as Mind-it, which ran for six years. This spawned new services such as ChangeDetection (1999), ChangeDetect (2002), <a href="/wiki/Google_Alerts" title="Google Alerts">Google Alerts</a> (2003), and Versionista (2007) which was used by the <a href="/wiki/John_McCain_2008_presidential_campaign" title="John McCain 2008 presidential campaign">John McCain 2008 presidential campaign</a> in the race for the <a href="/wiki/2008_United_States_presidential_election" title="2008 United States presidential election">2008 United States presidential election</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> Historically, change polling has been done either by a server which sent email notifications or a desktop program which audibly alerted the user to a change. Change alerting is also possible directly to mobile devices and through <a href="/wiki/Push_notification" class="mw-redirect" title="Push notification">push notifications</a>, <a href="/wiki/Webhooks" class="mw-redirect" title="Webhooks">webhooks</a> and HTTP callbacks for application integration.\n</p><p>Monitoring options vary by service or product and range from monitoring a single web page at a time to entire web sites.  What is actually monitored also varies by service or product with the possibilities of monitoring text, links, documents, scripts, images or screen shots.\n</p><p>With the notable exception of Google\'s patent filings related to <a href="/wiki/Google_Alerts" title="Google Alerts">Google Alerts</a>, <a href="/wiki/Intellectual_property" title="Intellectual property">intellectual property</a> activity by change detection and notification vendors is minimal.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup>  No one vendor has successfully leveraged exclusive rights to change detection and notification technology through patents or other legal means.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="Just as one can not prove a negative, one can not cite sources that do not exist (September 2016)">citation needed</span></a></i>&#93;</sup>  This has resulted in significant functional overlap between products and services.\n</p>\n<h2><span class="mw-headline" id="Architectural_approaches">Architectural approaches</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Change_detection_and_notification&amp;action=edit&amp;section=2" title="Edit section: Architectural approaches">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Change detection and notification services can be categorized by the <a href="/wiki/Software_architecture" title="Software architecture">software architecture</a> they use. Two principal approaches can be distinguished:\n</p>\n<h3><span class="mw-headline" id="Server_based">Server based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Change_detection_and_notification&amp;action=edit&amp;section=3" title="Edit section: Server based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>A server polls content, tracks changes and logs data, sending alerts in the form of email notifications, <a href="/wiki/Webhook" title="Webhook">webhooks</a>, <a href="/wiki/RSS" title="RSS">RSS</a>. Typically, an associated website with a configuration is managed by the user.  Some services also have a mobile device application which connects to a <a href="/wiki/Cloud_server" class="mw-redirect" title="Cloud server">cloud server</a> and provides alerts to the mobile device.\n</p>\n<h3><span class="mw-headline" id="Client_based">Client based</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Change_detection_and_notification&amp;action=edit&amp;section=4" title="Edit section: Client based">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>A local client application with a <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> polls content, tracks changes and logs data.\n</p>\n<h2><span class="mw-headline" id="Considerations">Considerations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Change_detection_and_notification&amp;action=edit&amp;section=5" title="Edit section: Considerations">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Some web pages change regularly, due to the inclusion of adverts or feeds in the presented page. This can trigger false-positives in the change-detection, since users are often only interested in changes to the main content. Some approaches to mitigate this issue exist.\n</p>\n<ul><li>Create a metric of difference between two versions of a page (calculated for example from change in total size, changes in HTML file, or changes in the <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> <a href="/wiki/Tree_structure" title="Tree structure">tree</a>) and ignore changes below some threshold. The threshold may be set by the user, or estimated automatically by comparing some early versions of the page.</li>\n<li>Content extraction. For popular sites, or sites running popular software, content may be actively separated from chaff by selecting a sub-tree of the DOM, for example using <a href="/wiki/XPath" title="XPath">XPath</a>. Another typical method is the use of <a href="/wiki/Regular_expressions" class="mw-redirect" title="Regular expressions">regular expressions</a> to extract only the text the user is interested in.</li></ul>\n<h2><span class="mw-headline" id="References">References</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
