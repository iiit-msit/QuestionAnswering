class="infobox-data"><span class="url"><a rel="nofollow" class="external text" href="http://microformats.org/">microformats<wbr />.org</a></span></td></tr></tbody></table>\n<p><b>Microformats</b> (sometimes abbreviated <b>\xce\xbcF</b> although this conflicts with the <a href="/wiki/International_System_of_Units" title="International System of Units">SI</a> designation for <a href="/wiki/Farad" title="Farad">micro Farad</a>) are a set of defined <a href="/wiki/HTML_classes" class="mw-redirect" title="HTML classes">HTML classes</a> created to serve as consistent and descriptive <a href="/wiki/Metadata" title="Metadata">metadata</a> about an <a href="/wiki/HTML_element" title="HTML element">element</a>, designating it as representing a certain type of data (such as <a href="/wiki/Address_book" title="Address book">contact information</a>, <a href="/wiki/Geographic_coordinate_system" title="Geographic coordinate system">geographic coordinates</a>, events, blog posts, products, recipes, etc.).<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> They allow <a href="/wiki/Software_agent" title="Software agent">software</a> to process the information reliably by having set classes refer to a specific type of data rather than being arbitrary. Microformats emerged around 2005 and were predominantly designed for use by search engines and <a href="/wiki/News_aggregator" title="News aggregator">aggregators</a> such as <a href="/wiki/RSS" title="RSS">RSS</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>\n</p><p>Although the content of web pages has been capable of some "automated processing" since the inception of the web, such processing is difficult because the <a href="/wiki/Markup_language" title="Markup language">markup elements</a> used to display information on the web do not describe what the information means.<sup id="cite_ref-Wharton000_3-0" class="reference"><a href="#cite_note-Wharton000-3">&#91;3&#93;</a></sup> Microformats can bridge this gap by attaching <a href="/wiki/Semantics" title="Semantics">semantics</a>, and thereby obviating other, more complicated, methods of automated processing, such as <a href="/wiki/Natural_language_processing" title="Natural language processing">natural language processing</a> or <a href="/wiki/Screen_scraping" class="mw-redirect" title="Screen scraping">screen scraping</a>. The use, adoption and processing of microformats enables data items to be indexed, searched for, saved or cross-referenced, so that information can be reused or combined.<sup id="cite_ref-Wharton000_3-1" class="reference"><a href="#cite_note-Wharton000-3">&#91;3&#93;</a></sup>\n</p><p>As of 2013<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Microformat&amp;action=edit">&#91;update&#93;</a></sup>, microformats allow the encoding and extraction of event details, contact information, social relationships and similar information.\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" links</span></a></li>\n</ul>\n</div>\n\n<h2><span class="mw-headline" id="Background">Background</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=1" title="Edit section: Background">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Microformats emerged around 2005<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;note 1&#93;</a></sup> as part of a grassroots movement to make recognizable data items (such as events, contact details or geographical locations) capable of automated processing by software, as well as directly readable by end-users.<sup id="cite_ref-Wharton000_3-2" class="reference"><a href="#cite_note-Wharton000-3">&#91;3&#93;</a></sup><sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;note 2&#93;</a></sup> Link-based microformats emerged first. These include vote links that express opinions of the linked page, which search engines can tally into instant polls.<sup id="cite_ref-Khare000_6-0" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup>\n</p><p><a href="/wiki/CommerceNet" title="CommerceNet">CommerceNet</a>, a nonprofit organization that promotes <a href="/wiki/E-commerce" title="E-commerce">e-commerce</a> on the Internet, has helped sponsor and promote the technology and support the microformats community in various ways.<sup id="cite_ref-Khare000_6-1" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup> CommerceNet also helped co-found the Microformats.org community site.<sup id="cite_ref-Khare000_6-2" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup>\n</p><p>Neither CommerceNet nor Microformats.org operates as a <a href="/wiki/Standards_body" class="mw-redirect" title="Standards body">standards body</a>. The microformats community functions through an open <a href="/wiki/Wiki" title="Wiki">wiki</a>, a mailing list, and an Internet relay chat (<a href="/wiki/Internet_Relay_Chat" title="Internet Relay Chat">IRC</a>) channel.<sup id="cite_ref-Khare000_6-3" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup> Most of the existing microformats originated at the Microformats.org wiki and the associated mailing list<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (October 2012)">citation needed</span></a></i>&#93;</sup> by a process of gathering examples of web-publishing behaviour, then codifying it. Some other microformats (such as <a href="/wiki/Nofollow" title="Nofollow">rel=nofollow</a> and <a href="/wiki/UnAPI" title="UnAPI">unAPI</a>) have been proposed, or developed, elsewhere.\n</p>\n<h2><span class="mw-headline" id="Technical_overview">Technical overview</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=2" title="Edit section: Technical overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p><a href="/wiki/XHTML" title="XHTML">XHTML</a> and HTML standards allow for the embedding and encoding of semantics within the <a href="/wiki/HTML_element" title="HTML element">attributes of markup elements</a>. Microformats take advantage of these standards by indicating the presence of metadata using the following attributes:\n</p>\n<dl><dt><code>class</code></dt>\n<dd><a href="/wiki/Class_(computer_programming)" title="Class (computer programming)">Classname</a></dd></dl>\n<dl><dt><code>rel</code></dt>\n<dd>relationship, description of the target address in an anchor-element (<code>&lt;a href=... rel=...&gt;...&lt;/a&gt;</code>)</dd></dl>\n<dl><dt><code>rev</code></dt>\n<dd>reverse relationship, description of the referenced document (in one case, otherwise deprecated in microformats<sup id="cite_ref-uF-rel-faq_7-0" class="reference"><a href="#cite_note-uF-rel-faq-7">&#91;5&#93;</a></sup>)</dd></dl>\n<p>For example, in the text "The birds roosted at <span class="geo"><span class="latitude">52.48</span>, <span class="longitude">-1.89</span></span>" is a pair of numbers which may be understood, from their context, to be a set of <a href="/wiki/Geographic_coordinate_system" title="Geographic coordinate system">geographic coordinates</a>. With wrapping in <a href="/wiki/Span_and_div" title="Span and div">spans</a> (or other HTML elements) with specific class names (in this case <code>geo</code>, <code>latitude</code> and <code>longitude</code>, all part of the <a href="/wiki/Geo_(microformat)" title="Geo (microformat)">geo microformat</a> specification):\n</p>\n <div class="mw-highlight mw-highlight-lang-xml mw-content-ltr" dir="ltr"><pre><span></span>The birds roosted at\n   <span class="nt">&lt;span</span> <span class="na">class=</span><span class="s">&quot;geo&quot;</span><span class="nt">&gt;</span>\n     <span class="nt">&lt;span</span> <span class="na">class=</span><span class="s">&quot;latitude&quot;</span><span class="nt">&gt;</span>52.48<span class="nt">&lt;/span&gt;</span>,\n     <span class="nt">&lt;span</span> <span class="na">class=</span><span class="s">&quot;longitude&quot;</span><span class="nt">&gt;</span>-1.89<span class="nt">&lt;/span&gt;</span>\n   <span class="nt">&lt;/span&gt;</span>\n</pre></div>\n<p>software agents can recognize exactly what each value represents and can then perform a variety of tasks such as indexing, locating it on a map and exporting it to a <a href="/wiki/GPS" class="mw-redirect" title="GPS">GPS</a> device.\n</p>\n<h3><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=3" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>In this example, the contact information is presented as follows:\n</p>\n <div class="mw-highlight mw-highlight-lang-xml mw-content-ltr" dir="ltr"><pre><span></span> <span class="nt">&lt;ul&gt;</span>\n   <span class="nt">&lt;li&gt;</span>Joe Doe<span class="nt">&lt;/li&gt;</span>\n   <span class="nt">&lt;li&gt;</span>The Example Company<span class="nt">&lt;/li&gt;</span>\n   <span class="nt">&lt;li&gt;</span>604-555-1234<span class="nt">&lt;/li&gt;</span>\n   <span class="nt">&lt;li&gt;&lt;a</span> <span class="na">href=</span><span class="s">&quot;http://example.com/&quot;</span><span class="nt">&gt;</span>http://example.com/<span class="nt">&lt;/a&gt;&lt;/li&gt;</span>\n <span class="nt">&lt;/ul&gt;</span>\n</pre></div>\n<p>With hCard microformat markup, that becomes:\n</p>\n <div class="mw-highlight mw-highlight-lang-xml mw-content-ltr" dir="ltr"><pre><span></span> <span class="nt">&lt;ul</span> <span class="na">class=</span><span class="s">&quot;vcard&quot;</span><span class="nt">&gt;</span>\n   <span class="nt">&lt;li</span> <span class="na">class=</span><span class="s">&quot;fn&quot;</span><span class="nt">&gt;</span>Joe Doe<span class="nt">&lt;/li&gt;</span>\n   <span class="nt">&lt;li</span> <span class="na">class=</span><span class="s">&quot;org&quot;</span><span class="nt">&gt;</span>The Example Company<span class="nt">&lt;/li&gt;</span>\n   <span class="nt">&lt;li</span> <span class="na">class=</span><span class="s">&quot;tel&quot;</span><span class="nt">&gt;</span>604-555-1234<span class="nt">&lt;/li&gt;</span>\n   <span class="nt">&lt;li&gt;&lt;a</span> <span class="na">class=</span><span class="s">&quot;url&quot;</span> <span class="na">href=</span><span class="s">&quot;http://example.com/&quot;</span><span class="nt">&gt;</span>http://example.com/<span class="nt">&lt;/a&gt;&lt;/li&gt;</span>\n <span class="nt">&lt;/ul&gt;</span>\n</pre></div>\n<p>Here, the formatted name (<code>fn</code>), organisation (<code>org</code>), telephone number (<code>tel</code>) and <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">web address</a> (<code>url</code>) have been identified using specific class names and the whole thing is wrapped in <code>class="vcard"</code>, which indicates that the other classes form an hCard (short for "HTML <a href="/wiki/VCard" title="VCard">vCard</a>") and are not merely coincidentally named. Other, optional, hCard classes also exist. Software, such as browser plug-ins, can now extract the information, and transfer it to other applications, such as an address book.\n</p>\n<div class="noprint"> \n<h3><span class="mw-headline" id="In-context_examples">In-context examples</span></h3>\n<p>For annotated examples of microformats on live pages, see <a href="/wiki/HCard#Live_example" title="HCard">HCard#Live example</a> and <a href="/wiki/Geo_(microformat)#Usage" title="Geo (microformat)">Geo (microformat)#Usage</a>.\n</p>\n</div>\n<h2><span class="mw-headline" id="Specific_microformats">Specific microformats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=4" title="Edit section: Specific microformats">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Several microformats have been developed to enable semantic markup of particular types of information. href="/w/index.php?title=Microformat&amp;action=edit&amp;section=5" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Using microformats within HTML code provides additional formatting and semantic data that applications can use. For example, applications such as <a href="/wiki/Web_crawler" title="Web crawler">web crawlers</a> can collect data about on-line resources, or desktop applications such as e-mail clients or scheduling software can compile details. The use of microformats can also facilitate "mash ups" such as exporting all of the geographical locations on a web page into (for example) <a href="/wiki/Google_Maps" title="Google Maps">Google Maps</a> to visualize them spatially.\n</p><p>Several browser extensions, such as <a href="/wiki/Operator_(extension)" title="Operator (extension)">Operator</a> for <a href="/wiki/Firefox" title="Firefox">Firefox</a> and Oomph for <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>, provide the ability to detect microformats within an HTML document. When hCard or hCalendar are involved, such browser extensions allow microformats to be exported into formats compatible with contact management and calendar utilities, such as <a href="/wiki/Microsoft_Outlook" title="Microsoft Outlook">Microsoft Outlook</a>. When dealing with geographical coordinates, they allow the location to be sent to applications such as <a href="/wiki/Google_Maps" title="Google Maps">Google Maps</a>. <a href="/wiki/Yahoo!_query_language" class="mw-redirect" title="Yahoo! query language">Yahoo! Query Language</a> can be used to extract microformats from web pages.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;12&#93;</a></sup> On 12 May 2009 <a href="/wiki/Google_Search" title="Google Search">Google</a> announced that they would be parsing the hCard, hReview and hProduct microformats, and using them to populate search result pages.<sup id="cite_ref-Rich-Snippets_15-0" class="reference"><a href="#cite_note-Rich-Snippets-15">&#91;13&#93;</a></sup> They subsequently extended this in 2010 to use hCalendar for events and hRecipe for cookery recipes.<sup id="cite_ref-Google-recipes_16-0" class="reference"><a href="#cite_note-Google-recipes-16">&#91;14&#93;</a></sup> Similarly, microformats are also processed by <a href="/wiki/Bing_(search_engine)" class="mw-redirect" title="Bing (search engine)">Bing</a><sup id="cite_ref-Bing_17-0" class="reference"><a href="#cite_note-Bing-17">&#91;15&#93;</a></sup> and <a href="/wiki/Yahoo!" title="Yahoo!">Yahoo!</a>.<sup id="cite_ref-YSearch_18-0" class="reference"><a href="#cite_note-YSearch-18">&#91;16&#93;</a></sup> As of late 2010, these are the world\'s top three search engines.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;17&#93;</a></sup>\n</p><p><a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> said in 2006 that they needed to incorporate Microformats into upcoming projects,<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">&#91;18&#93;</a></sup> as did other software companies.\n</p><p>Alex Faaborg summarizes the arguments for putting the responsibility for microformat user interfaces in the web browser rather than making more complicated HTML:<sup id="cite_ref-21" class="reference"><a href="#cite_note-21">&#91;19&#93;</a></sup>\n</p>\n<ul><li>Only the web browser knows what applications are accessible to the user and what the user\'s preferences are</li>\n<li>It lowers the barrier to entry for web site developers if they only need to do the markup and not handle "appearance" or "action" issues</li>\n<li>Retains backwards compatibility with web browsers that don\'t support microformats</li>\n<li>The web browser presents a single point of entry from the web to the user\'s computer, which simplifies security issues</li></ul>\n<h2><span class="mw-headline" id="Evaluation">Evaluation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=6" title="Edit section: Evaluation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Various commentators have offered review and discussion on the design principles and practical aspects of microformats. Microformats have been compared to other approaches that seek to serve the same or similar purpose.<sup id="cite_ref-criticism000_22-0" class="reference"><a href="#cite_note-criticism000-22">&#91;20&#93;</a></sup> As of 2007<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Microformat&amp;action=edit">&#91;update&#93;</a></sup>, there had been some criticism of one, or all, microformats.<sup id="cite_ref-criticism000_22-1" class="reference"><a href="#cite_note-criticism000-22">&#91;20&#93;</a></sup> The spread and use of microformats was being advocated as of 2007<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=Microformat&amp;action=edit">&#91;update&#93;</a></sup>.<sup id="cite_ref-advocacy000_23-0" class="reference"><a href="#cite_note-advocacy000-23">&#91;21&#93;</a></sup><sup id="cite_ref-spread000_24-0" class="reference"><a href="#cite_note-spread000-24">&#91;22&#93;</a></sup> <a href="/wiki/Opera_Software" class="mw-redirect" title="Opera Software">Opera Software</a> CTO and <a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">CSS</a> creator <a href="/wiki/H%C3%A5kon_Wium_Lie" title="H\xc3\xa5kon Wium Lie">H\xc3\xa5kon Wium Lie</a> said in 2005 "We will also see a bunch of microformats being developed, and that\xe2\x80\x99s how the <a href="/wiki/Semantic_Web" title="Semantic Web">semantic web</a> will be built, I believe."<sup id="cite_ref-advocacy001_25-0" class="reference"><a href="#cite_note-advocacy001-25">&#91;23&#93;</a></sup> However, in August 2008 Toby Inkster, author of the "Swignition" (formerly "Cognition") microformat parsing service, pointed out that no new microformat specifications had been published since 2005.<sup id="cite_ref-threeyears_26-0" class="reference"><a href="#cite_note-threeyears-26">&#91;24&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Design_principles">Design principles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=7" title="Edit section: Design principles">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Computer scientist and entrepreneur, <a href="/wiki/Rohit_Khare" title="Rohit Khare">Rohit Khare</a> stated that <i>reduce, reuse, and recycle</i> is "shorthand for several design principles" that motivated the development and practices behind microformats.<sup id="cite_ref-Khare000_6-4" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup><sup class="reference" style="white-space:nowrap;">:<span>71\xe2\x80\x9372</span></sup> These aspects can be summarized as follows:\n</p>\n<ul><li>Reduce: favor the simplest solutions and focus attention on specific problems;</li>\n<li>Reuse: work from experience and favor examples of current practice;</li>\n<li>Recycle: encourage modularity and the ability to embed, valid XHTML can be reused in blog posts, <a href="/wiki/RSS" title="RSS">RSS</a> feeds, and anywhere else you can access the web.<sup id="cite_ref-Khare000_6-5" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup></li></ul>\n<h3><span class="mw-headline" id="Accessibility">Accessibility</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=8" title="Edit section: Accessibility">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Because some microformats make use of title attribute of HTML\'s <code class="nowrap" style="">&lt;abbr&gt;</code> element to conceal <a href="/wiki/Machine-readable_data" title="Machine-readable data">machine-readable data</a> (particularly date-times and geographical coordinates) in the "<a rel="nofollow" class="external text" href="http://microformats.org/wiki/abbr-design-pattern">abbr design pattern</a>", the plain text content of the element is inaccessible to <a href="/wiki/Screen_reader" title="Screen reader">screen readers</a> that expand abbreviations.<sup id="cite_ref-ATF_27-0" class="reference"><a href="#cite_note-ATF-27">&#91;25&#93;</a></sup> In June 2008 the <a href="/wiki/BBC" title="BBC">BBC</a> announced that it would be dropping use of microformats using the <code>abbr</code> design pattern because of accessibility concerns.<sup id="cite_ref-BBCabbr_28-0" class="reference"><a href="#cite_note-BBCabbr-28">&#91;26&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Comparison_with_alternative_approaches">Comparison with alternative approaches</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Microformat&amp;action=edit&amp;section=9" title="Edit section: Comparison with alternative approaches">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Microformats are not the only solution for providing "more intelligent data" on the web; alternative approaches are used and are under development. For example, the use of <a href="/wiki/XML" title="XML">XML</a> markup and standards of the Semantic Web are cited as alternative approaches.<sup id="cite_ref-Khare000_6-6" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup> Some contrast these with microformats in that they do not necessarily coincide with the design principles of "reduce, reuse, and recycle", at least not to the same extent.<sup id="cite_ref-Khare000_6-7" class="reference"><a href="#cite_note-Khare000-6">&#91;4&#93;</a></sup>\n</p><p>\nOne advocate of microformats, <a href="/wiki/Tantek_%C3%87elik" title="Tantek \xc3\x87elik">Tantek \xc3\x87elik</a>, characterized a problem with alternative approaches: <style data-mw-deduplicate="TemplateStyles:r996844942">.mw-parser-output .templatequote{overflow:hidden;margin:1em 0;padding:0 40px}.mw-parser-output .templatequote .templatequotecite{line-height:1.5em;text-align:left;padding-left:1.6em;margin-top:0}</style></p><blockquote class="templatequote"><p>Here\'s a new language we want you to learn, and now you need to output these additional files on your server. It\'s a hassle. (Microformats) lower the barrier to entry.<sup id="cite_ref-Wharton000_3-3" class="reference"><a href="#cite_note-Wharton000-3">&#91;3&#93;</a></sup></p></blockquote>\n<p>For some applications the use of other approaches may be valid. If the type of data to be described does not map to an existing microformat, <a href="/wiki/RDFa" title="RDFa">RDFa</a> can embed arbitrary vocabularies into HTML, such as for example domain-specific scientific data such as zoological or chemical data for which there is no microformat. Standards such as W3C\'s <a href="/wiki/GRDDL" title="GRDDL">GRDDL</a> allow microformats to be converted into data compatible with the Semantic Web.<sup id="cite_ref-King007_29-0" class="reference"><a href="#cite_note-King007-29">&#91;27&#93;</a></sup>\n</p><p>\nAnother advocate of microformats, Ryan King, put the compatibility of microformats with other approaches this way: <link rel="mw-deduplicated-inline-style" href="mw-data:TemplateStyles:r996844942"/></p><blockquote class="templatequote"><p>Microformats provide an easy way for many people to contribute semantic data to the web. With GRDDL all of that data is made available for RDF Semantic Web tools. Microformats and GRDDL can work together to build a better web.<sup id="cite_ref-King007_29-1" class="reference"><a href="#cite_note-King007-29">&#91;27&#93;</a></sup></p></blockquote>\n<h2><span
