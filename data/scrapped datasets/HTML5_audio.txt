id=".3Caudio.3E_element"></span><span class="mw-headline" id="&lt;audio&gt;_element">&lt;audio&gt; element</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5_audio&amp;action=edit&amp;section=1" title="Edit section: &lt;audio&gt; element">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>The &lt;audio&gt; element represents a sound, or an audio stream.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> It is commonly used to play back a single audio file within a web page, showing a GUI widget with play/pause/volume controls.\n</p><p>The &lt;audio&gt; element has these attributes:\n</p>\n<ul><li>global attributes (accesskey; class; contenteditable; contextmenu; dir; draggable; dropzone; hidden; id; lang; spellcheck; style; tabindex; title; translate)</li>\n<li>autoplay = "autoplay" or "" (empty string) or empty<br />Instructs the User-Agent to automatically begin playback of the audio stream as soon as it can do so without stopping.</li>\n<li>preload = "none" or "metadata" or "auto" or "" (empty string) or empty<br />Represents a hint to the User-Agent about whether optimistic downloading of the audio stream itself or its metadata is considered worthwhile.\n<ul><li>"none": Hints to the User-Agent that the user is not expected to need the audio stream, or that minimizing unnecessary traffic is desirable.</li>\n<li>"metadata": Hints to the User-Agent that the user is not expected to need the audio stream, but that fetching its metadata (duration and so on) is desirable.</li>\n<li>"auto": Hints to the User-Agent that optimistically downloading the entire audio stream is considered desirable.</li></ul></li>\n<li>controls = "controls" or "" (empty string) or empty<br />Instructs the User-Agent to expose a user interface for controlling playback of the audio stream.</li>\n<li>loop = "loop" or "" (empty string) or empty<br />Instructs the User-Agent to seek back to the start of the audio stream upon reaching the end.</li>\n<li>mediagroup = string<br />Instructs the User-Agent to link multiple videos and/or audio streams together.</li>\n<li>muted = "muted" or "" (empty string) or empty<br />Represents the default state of the audio stream, potentially overriding user preferences.</li>\n<li>src = non-empty [URL] potentially surrounded by spaces<br />The URL for the audio stream.</li></ul>\n<p>Example:<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>\n</p>\n<div id="Supported_audio_coding_formats">Supported audio coding formats</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5_audio&amp;action=edit&amp;section=3" title="Edit section: Supported audio coding formats">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>The adoption of HTML5 audio, as with <a href="/wiki/HTML5_video" title="HTML5 video">HTML5 video</a>, has become polarized between proponents of <a href="/wiki/Free_software" title="Free software">free</a> and <a href="/wiki/Software_patent" title="Software patent">patent-encumbered</a> formats. In 2007, the recommendation to use <a href="/wiki/Vorbis" title="Vorbis">Vorbis</a> was <a href="/wiki/Use_of_Ogg_formats_in_HTML5" title="Use of Ogg formats in HTML5">retracted</a> from the specification by the <a href="/wiki/W3C" class="mw-redirect" title="W3C">W3C</a> together with that to use <a href="/wiki/Ogg_Theora" class="mw-redirect" title="Ogg Theora">Ogg Theora</a>, citing the lack of a format accepted by all the major browser vendors.\n</p><p><a href="/wiki/Apple_Inc." title="Apple Inc.">Apple</a> and <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> support the <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">ISO</a>/<a href="/wiki/International_Electrotechnical_Commission" title="International Electrotechnical Commission">IEC</a>-<a href="/wiki/Technical_standard" title="Technical standard">defined</a> formats <a href="/wiki/Advanced_Audio_Coding" title="Advanced Audio Coding">AAC</a> and the older <a href="/wiki/MP3" title="MP3">MP3</a>. <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a> and <a href="/wiki/Opera_Software" class="mw-redirect" title="Opera Software">Opera</a> support the free and <a href="/wiki/Open-source_software" title="Open-source software">open</a>, <a href="/wiki/Royalty-free" title="Royalty-free">royalty-free</a> <a href="/wiki/Vorbis" title="Vorbis">Vorbis</a> format in <a href="/wiki/Ogg" title="Ogg">Ogg</a> and <a href="/wiki/WebM" title="WebM">WebM</a> containers, and criticize the patent-encumbered nature of MP3 and AAC, which are guaranteed to be \xe2\x80\x9cnon-free\xe2\x80\x9d. <a href="/wiki/Google" title="Google">Google</a> has so far provided support for all common formats.\n</p><p>Most AAC files with finite length are wrapped in an MPEG-4 container (.mp4, .m4a), which is supported natively in Internet Explorer, Safari, and Chrome, and supported by the OS in Firefox and Opera.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> Most AAC live streams with infinite length are wrapped in an Audio Data Transport Stream container (.aac, .adts), which is supported by Chrome, Safari, Firefox and Edge.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup><sup id="cite_ref-developer.apple.com_6-0" class="reference"><a href="#cite_note-developer.apple.com-6">&#91;6&#93;</a></sup><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup>\n</p><p>Many browsers also support uncompressed <a Audio API and MediaStream Processing API</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=HTML5_audio&amp;action=edit&amp;section=4" title="Edit section: Web Audio API and MediaStream Processing API">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>The Web Audio API specification developed by <a href="/wiki/W3C" class="mw-redirect" title="W3C">W3C</a> describes a high-level JavaScript API for processing and synthesizing audio in web applications. The primary paradigm is of an audio routing graph, where a number of AudioNode objects are connected together to define the overall audio rendering. The actual processing will primarily take place in the underlying implementation (typically optimized Assembly / C / C++ code), but direct JavaScript processing and synthesis is also supported.<sup id="cite_ref-27" class="reference"><a href="#cite_note-27">&#91;25&#93;</a></sup>\n</p><p><a href="/wiki/Firefox_4" title="Firefox 4">Mozilla\'s Firefox browser</a> implements a similar Audio Data API extension since version 4, implemented in 2010 <sup id="cite_ref-28" class="reference"><a href="#cite_note-28">&#91;26&#93;</a></sup> and released in 2011, but Mozilla warns it is non-standard and deprecated, and recommends the Web Audio API instead.<sup id="cite_ref-29" class="reference"><a href="#cite_note-29">&#91;27&#93;</a></sup>\nSome JavaScript audio processing and synthesis libraries such as <a rel="nofollow" class="external text" href="https://oampo.github.com/Audiolet/">Audiolet</a> support both APIs.\n</p><p>The <a rel="nofollow" class="external text" href="http://www.w3.org/2011/audio/">W3C Audio Working Group</a> is also considering the <a href="/wiki/MediaStream_Processing_API" class="mw-redirect" title="MediaStream Processing API">MediaStream Processing API</a> specification developed by <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a>.<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">&#91;28&#93;</a></sup>\nIn addition to audio mixing and processing, it covers more general media streaming, including synchronization with HTML elements, capture of audio and video streams, and <a href="/wiki/WebRTC"
