title="Template:Sockpuppet">Template:Sockpuppet</a>.<br />\nApparently, this template seems to be a copy of <code>{{sockpuppet|\'\'Example\'\'|proven}}</code>, which provides the same sockpuppet template type. Also, this template can be ambiguous with <code>{{sockpuppet|\'\'Example\'\'|blocked}}</code>, which provides as "suspected sockpuppet" and <code>{{sockpuppet|\'\'Example\'\'|confirmed}}</code>, which provides as "confirmed by CheckUser as a sockpuppet". I propose to merge this template unless justification can be provided. <b><a href="/wiki/User:Seventyfiveyears" title="User:Seventyfiveyears"><span style="color:blue;">Seventyfiveyears</span></a> (<a href="/wiki/User_talk:Seventyfiveyears" title="User talk:Seventyfiveyears">talk</a>)</b> 21:04, 30 April 2021 (UTC)\n</p>\n<ul><li>Although I don\'t necessarily see much issue with merging <a href="/wiki/Template:Blocked_sockpuppet" title="Template:Blocked sockpuppet">Template:blocked sockpuppet</a> with <a href="/wiki/Template:Sockpuppet" title="Template:Sockpuppet">Template:sockpuppet</a>, blocked sockpuppet does by default mark a sockpuppet as "proven". The default behaviour of sockpuppet is to instead mark as suspected. If Blocked sockpuppet is to be merged, uses should be substed and then redirected as Sockpuppet already handles the behaviour if you provide two extra parameters to sockpuppet template. <a href="/wiki/User:Dreamy_Jazz" title="User:Dreamy Jazz">Dreamy <i style="color:#d00"><b>Jazz</b></i></a> <sup><i><a href="/wiki/User_talk:Dreamy_Jazz" title="User talk:Dreamy Jazz">talk to me</a></i> &#124; <i><a href="/wiki/Special:Contributions/Dreamy_Jazz" title="Special:Contributions/Dreamy Jazz">my contributions</a></i></sup> 22:56, 30 April 2021 (UTC)</li>\n<li><b>Comment</b>: Current default behaviour of &#123;&#123;<a href="/wiki/Template:Sockpuppet" title="Template:Sockpuppet">sockpuppet</a>&#125;&#125; or &#123;&#123;<a href="/wiki/Template:Sock" class="mw-redirect" title="Template:Sock">sock</a>&#125;&#125; is to mark as suspected but not blocked which has created a somewhat convoluted tagging issue&#160;&#8211;&#32;resolving it would likely some technical effort; the discussion <a href="/wiki/Template_talk:Sockpuppet#Deprecation_of_&quot;an_editor_has_expressed_a_concern&quot;_version" title="Template talk:Sockpuppet">here</a> is probably relevant to this conversation.
