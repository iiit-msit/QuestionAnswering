title="Comparison of layout engines (XHTML 1.1)">1.1</a></li></ul></li></ul></td>\n</tr></tbody></table>\n<p><b>Dynamic HTML</b>, or <b>DHTML</b>, is a collection of technologies used together to create interactive and animated <a href="/wiki/Website" title="Website">websites</a> by using a combination of a static <a href="/wiki/Markup_language" title="Markup language">markup language</a> (such as <a href="/wiki/HTML" title="HTML">HTML</a>), a <a href="/wiki/Client-side_scripting" class="mw-redirect" title="Client-side scripting">client-side scripting</a> language (such as <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>), a presentation definition language (such as <a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">CSS</a>), and the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM).<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> The application of DHTML was introduced by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> with the release of <a href="/wiki/Internet_Explorer_4" title="Internet Explorer 4">Internet Explorer 4</a> in 1997. Today, references to <a href="/wiki/Unobtrusive_JavaScript" title="Unobtrusive JavaScript">unobtrusive JavaScript</a> coding (DOM Scripting) have replaced the usage of the term DHTML.\n</p><p>DHTML allows scripting languages to change <a href="/wiki/Variable_(programming)" class="mw-redirect" title="Variable (programming)">variables</a> in a web page\'s definition language, which in turn affects the look and function of otherwise "static" HTML page content, after the page has been fully loaded and during the viewing process. Thus the dynamic characteristic of DHTML is the way it functions while a page is viewed, not in its ability to generate a unique page with each page load.\n</p><p>By contrast, a <a href="/wiki/Dynamic_web_page" title="Dynamic web page">dynamic web page</a> is a broader concept, covering any web page generated differently for each user, load occurrence, or specific variable values. This includes pages created by client-side scripting, and ones created by <a href="/wiki/Server-side_scripting" title="Server-side scripting">server-side scripting</a> (such as <a href="/wiki/PHP" title="PHP">PHP</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/JavaServer_Pages" class="mw-redirect" title="JavaServer Pages">JSP</a> or <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>) where the web server generates content before sending it to the client.\n</p><p>DHTML is differentiated from <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> by the fact that a DHTML page is still request/reload-based. With DHTML, there may not be any interaction between the client and server after the page is loaded; all processing happens in JavaScript on the client side. By contrast, an Ajax page uses features of DHTML to initiate a request (or \'subrequest\') to the server to perform additional actions. For example, if there are multiple tabs on a page, pure DHTML approach would load the contents of all tabs and then dynamically display only the one that is active, while AJAX could load each tab only when it is really needed.\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" role="button" id="toctogglecheckbox" class="toctogglecheckbox" id="Uses">Uses</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_HTML&amp;action=edit&amp;section=1" title="Edit section: Uses">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>DHTML allows authors to add effects to their pages that are otherwise difficult to achieve, by changing the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) and page style. The combination of HTML, CSS and JavaScript offers ways to:\n</p>\n<ul><li>Animate text and images in their document.</li>\n<li>Embed a ticker or other dynamic display that automatically refreshes its content with the latest news, stock quotes, or other data.</li>\n<li>Use a form to capture user input, and then process, verify and respond to that data without having to send data back to the server.</li>\n<li>Include rollover buttons or drop-down menus.</li></ul>\n<p>A less common use is to create browser-based action games. Although a number of games were created using DHTML during the late 1990s and early 2000s,<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (October 2008)">citation needed</span></a></i>&#93;</sup> differences between browsers made this difficult: many techniques had to be implemented in code to enable the games to work on multiple platforms. Recently browsers have been converging towards <a href="/wiki/Web_standards" title="Web standards">web standards</a>, which has made the design of DHTML games more viable. Those games can be played on all major browsers and they can also be ported to <a href="/wiki/KDE_Plasma_Workspaces" class="mw-redirect" title="KDE Plasma Workspaces">Plasma</a> for <a href="/wiki/KDE" title="KDE">KDE</a>, Widgets for <a href="/wiki/MacOS" title="MacOS">macOS</a> and Gadgets for <a href="/wiki/Windows_Vista" title="Windows Vista">Windows Vista</a>, which are based on DHTML code.\n</p><p>The term "DHTML" has fallen out of use in recent years as it was associated with practices and conventions that tended to not work well between various web browsers. DHTML may now be referred to as <a href="/wiki/Unobtrusive_JavaScript" title="Unobtrusive JavaScript">unobtrusive JavaScript</a> coding (DOM Scripting), in an effort to place an emphasis on agreed-upon best practices while <a href="/wiki/Progressive_Enhancement" class="mw-redirect" title="Progressive Enhancement">allowing similar effects in an accessible, standards-compliant way</a>.\n</p><p>DHTML support with extensive DOM access was introduced with <a href="/wiki/Internet_Explorer_4.0" class="mw-redirect" title="Internet Explorer 4.0">Internet Explorer 4.0</a>. Although there was a basic dynamic system with <a href="/wiki/Netscape_Navigator" title="Netscape Navigator">Netscape Navigator 4.0</a>, not all HTML elements were represented in the DOM. When DHTML-style techniques became widespread, varying degrees of support among web browsers for the technologies involved made them difficult to develop and <a href="/wiki/Debug" class="mw-redirect" title="Debug">debug</a>. Development became easier when <a href="/wiki/Internet_Explorer_5" title="Internet Explorer 5">Internet Explorer 5.0+</a>, <a href="/wiki/Firefox" title="Firefox">Mozilla Firefox</a> 2.0+, and <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a> 7.0+ adopted a shared <a href="/wiki/Document_Object_Model" title="Document Object Model">DOM</a> inherited from <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a>.\n</p><p>More recently, <a href="/wiki/JavaScript_library" title="JavaScript library">JavaScript libraries</a> such as <a href="/wiki/JQuery" title="JQuery">jQuery</a> have abstracted away many of the day-to-day difficulties in cross-browser DOM manipulation.\n</p>\n<h2><span class="mw-headline" id="Structure_of_a_web_page">Structure of a id="Document_Object_Model">Document Object Model</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_HTML&amp;action=edit&amp;section=4" title="Edit section: Document Object Model">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a></div>\n<p>DHTML is not a technology in and of itself; rather, it  is the product of three related and complementary technologies: HTML, Cascading Style Sheets (CSS), and <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>. To allow scripts and components to access features of HTML and CSS, the contents of the document are represented as objects in a programming model known as the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM).\n</p><p>The DOM API is the foundation of DHTML, providing a structured interface that allows access and manipulation of virtually anything in the document. The HTML elements in the document are available as a hierarchical <a href="/wiki/Tree_(data_structure)" title="Tree (data structure)">tree</a> of individual objects, making it possible to examine and modify an element and its attributes by reading and setting properties and by calling methods. The text between elements is also available through DOM properties and methods.\n</p><p>The DOM also provides access to user actions such as pressing a key and clicking the mouse. It is possible to intercept and process these and other events by creating event handler functions and routines. The event handler receives control each time a given event occurs and can carry out any appropriate action, including using the DOM to change the document.\n</p>\n<h2><span class="mw-headline" id="Dynamic_styles">Dynamic styles</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_HTML&amp;action=edit&amp;section=5" title="Edit section: Dynamic styles">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Dynamic styles are a key feature of DHTML. By using CSS, one can quickly change the appearance and formatting of elements in a document without adding or removing elements. This helps keep documents small and the scripts that manipulate the document fast.\n</p><p>The object model provides programmatic access to styles. This means you can change inline styles on individual elements and change style rules using simple JavaScript programming.\n</p><p>Inline styles are CSS style assignments that have been applied to an element using the style attribute. You can examine and set these styles by retrieving the style object for an individual element.
