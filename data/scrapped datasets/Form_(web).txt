dir="ltr" class="mw-content-ltr"><div class="mw-parser-output"><div class="shortdescription nomobile noexcerpt noprint searchaux" style="display:none">Input fields on a Web page</div>\n<p>A <b>webform</b>, <b>web form</b> or <b>HTML form</b> on a <a href="/wiki/Web_page" title="Web page">web page</a> allows a user to enter data that is sent to a <a href="/wiki/Server_(computing)" title="Server (computing)">server</a> for processing. Forms can resemble <a href="/wiki/Form_(document)" title="Form (document)">paper</a> or <a href="/wiki/Database" title="Database">database</a> forms because web users fill out the forms using checkboxes, <a href="/wiki/Radio_buttons" class="mw-redirect" title="Radio buttons">radio buttons</a>, or <a href="/wiki/Text_fields" class="mw-redirect" title="Text fields">text fields</a>.  For example, forms can be used to enter <a href="/wiki/Shipping" class="mw-redirect" title="Shipping">shipping</a> or <a href="/wiki/Credit_card" title="Credit card">credit card</a> data to order a product, or can be used to retrieve search results from a <a href="/wiki/Search_engine" title="Search engine">search engine</a>.\n</p>\n<div id="toc" class="toc" role="navigation" class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=1" title="Edit section: Description">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<div class="thumb tright"><div class="thumbinner" style="width:395px;"><a href="/wiki/File:Sample_web_form.png" class="image"><img alt="test" src="//upload.wikimedia.org/wikipedia/commons/3/34/Sample_web_form.png" decoding="async" width="393" height="320" class="thumbimage" data-file-width="393" data-file-height="320" /></a>  <div class="thumbcaption"><div class="magnify"><a href="/wiki/File:Sample_web_form.png" class="internal" title="Enlarge"></a></div>Sample form. The form is enclosed in an <a href="/wiki/HTML_table" class="mw-redirect" title="HTML table">HTML table</a> for visual layout.</div></div></div>\n<p>Forms are enclosed in the <a href="/wiki/HTML" title="HTML">HTML</a> <code>&lt;form&gt;</code> element. This element specifies the <a href="/wiki/Communication_endpoint" title="Communication endpoint">communication endpoint</a> the data entered into the form should be submitted to, and the <a href="/wiki/HTTP_request" class="mw-redirect" title="HTTP request">method</a> of submitting the data, <code>GET</code> or <code>POST</code>.\n</p>\n<h3><span class="mw-headline" id="Elements">Elements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=2" title="Edit section: Elements">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Forms can be made up of standard <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> elements:\n</p>\n<ul><li><code>&lt;text&gt;</code> \xe2\x80\x94 a simple <a href="/wiki/Text_box" title="Text box">text box</a> that allows input of a single line of text.</li>\n<li><code>&lt;email&gt;</code> - a type of  <code>&lt;text&gt;</code> that requires a partially validated email address</li>\n<li><code>&lt;number&gt;</code> - a type of  <code>&lt;text&gt;</code> that requires a number</li>\n<li><code>&lt;password&gt;</code> \xe2\x80\x94 similar to  <code>&lt;text&gt;</code>, it is used for security purposes, in which the characters typed in are invisible or replaced by symbols such as *)</li>\n<li><code>&lt;radio&gt;</code> \xe2\x80\x94 a <a href="/wiki/Radio_button" title="Radio button">radio button</a></li>\n<li><code>&lt;file&gt;</code> \xe2\x80\x94 a <a href="/wiki/File_select" title="File select">file select</a> control for uploading a file</li>\n<li><code>&lt;reset&gt;</code> \xe2\x80\x94 a <a href="/wiki/Reset_button" title="Reset button">reset button</a> that, when activated, tells the browser to restore the values to their initial values.</li>\n<li><code>&lt;submit&gt;</code> \xe2\x80\x94 a <a href="/wiki/Button_(computing)" title="Button (computing)">button</a> that tells the browser to take action on the form (typically to send it to a server)</li>\n<li><code>&lt;textarea&gt;</code> \xe2\x80\x94 much like the  <code>&lt;text&gt;</code> input field except a  <code>&lt;textarea&gt;</code> allows for multiple rows of data to be shown and entered</li>\n<li><code>&lt;select&gt;</code> \xe2\x80\x94 a <a href="/wiki/Drop-down_list" title="Drop-down list">drop-down list</a> that displays a list of items a user can select from</li></ul>\n<p>The sample image on the right shows most of these elements:\n</p>\n<ul><li>a text box asking for your name</li>\n<li>a pair of radio buttons asking you to pick your sex</li>\n<li>a <a href="/wiki/Drop-down_list" title="Drop-down list">select box</a> giving you a list of eye colors to choose from</li>\n<li>a pair of check boxes to click on if they apply to you</li>\n<li>a text area to describe your athletic ability</li>\n<li>a submit button to send it to the server</li></ul>\n<p>These basic elements provide most common <a href="/wiki/Graphical_user_interface" title="Graphical user interface">graphical user interface</a> (GUI) elements, but not all. For example, there are no equivalents to a <a href="/wiki/Tree_view" title="Tree view">tree view</a> or <a href="/wiki/Grid_view" title="Grid view">grid view</a>.\n</p><p>A grid view, however, can be mimicked by using a standard HTML <a href="/wiki/Table_(HTML)" class="mw-redirect" title="Table (HTML)">table</a> with each cell containing a text input element. A tree view could also be mimicked through nested tables or, more <a href="/wiki/HTML#Semantic_HTML" title="HTML">semantically</a> appropriately, nested <a href="/wiki/HTML_element#Lists" title="HTML element">lists</a>. In both cases, a server-side process is responsible for processing the information, while JavaScript handles the user-interaction. Implementations of these interface elements are available through <a href="/wiki/JavaScript_library" title="JavaScript library">JavaScript libraries</a> such as <a href="/wiki/JQuery" title="JQuery">jQuery</a>.\n</p><p>HTML 4 introduced the <code>&lt;label&gt;</code> tag, which is intended to represent a caption in a user interface, and can be associated with a specific form control by specifying the <code>id</code> attribute of the control in the label tag\'s <code>for</code> attribute.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> This allows labels to stay with their elements when a window is resized and to allow more desktop-like functionality (e.g. clicking a radio button or checkbox\'s label will activate the associated input element).\n</p><p>HTML 5 introduces a number of input tags that can be represented by other interface elements. Some are based upon text input fields and are intended to input and validate specific common data. These include <code>&lt;email&gt;</code> to enter email addresses, <code>&lt;tel&gt;</code> for telephone numbers, <code>&lt;number&gt;</code> for numeric values. There are additional attributes to specify required fields, fields that should have keyboard <a href="/wiki/Focus_(computing)" title="Focus (computing)">focus</a> when the web page containing the form is loaded, and placeholder text that is displayed within the field but is not user input (such as the \'Search\' text displayed in many search input fields before a search term is entered). These tasks used to be handled with <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, but had become so common that support for them was added to the standard. The <code>&lt;date&gt;</code> input type displays a calendar from which the user can select a date or date range.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup><sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> And the <code>color</code> input type can be represented as an input text simply checking the value entered is a correct hexadecimal representation of a color, according to the specification,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> or a color picker widget (the latter being the solution used in most browsers which support this attribute).\n</p>\n<h2><span class="mw-headline" id="Submission">Submission</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=3" title="Edit section: Submission">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>When data that has been entered into HTML forms is submitted, the names and values in the form elements are encoded and sent to the server in an <a href="/wiki/HTTP" class="mw-redirect" title="HTTP">HTTP</a> request message using <a href="/wiki/Hypertext_Transfer_Protocol#Request_methods" title="Hypertext Transfer Protocol">GET</a> or <a href="/wiki/POST_(HTTP)" title="POST (HTTP)">POST</a>. Historically, an <a href="/wiki/Email" title="Email">email</a> transport was also used.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup> The default <a href="/wiki/Mime_type" class="mw-redirect" title="Mime type">mime type</a>, <a href="/wiki/Internet_media_type" class="mw-redirect" title="Internet media type">Internet media type</a> <a href="/wiki/Application/x-www-form-urlencoded" class="mw-redirect" title="Application/x-www-form-urlencoded">application/x-www-form-urlencoded</a>, is based on a very early version of the general URI <a href="/wiki/Percent-encoding" title="Percent-encoding">percent-encoding</a> rules, with a number of modifications such as <a href="/wiki/Newline" title="Newline">newline</a> normalization and replacing spaces with "<code>+</code>" instead of "<code>%20</code>". Another possible encoding, Internet media type <a href="/wiki/Multipart/form-data" class="mw-redirect" title="Multipart/form-data">multipart/form-data</a>, is also available and is common for POST-based file submissions.\n</p>\n<h2><span class="mw-headline" id="Use_with_programming_languages">Use with programming languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=4" title="Edit section: Use with programming languages">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Forms are usually combined with programs written in various <a href="/wiki/Programming_language" title="Programming language">programming language</a> to allow <a href="/wiki/Software_developer" class="mw-redirect" title="Software developer">developers</a> to create dynamic <a href="/wiki/Web_site" class="mw-redirect" title="Web site">web sites</a>. The most popular languages include both client-side and/or server-side languages.\n</p><p>Although any programming language can be used on the server to process a form\'s data, the most commonly used languages are <a href="/wiki/Scripting_languages" class="mw-redirect" title="Scripting languages">scripting languages</a>, which tend to have stronger <a href="/wiki/String_(programming)" class="mw-redirect" title="String (programming)">string</a> handling functionality than programming languages such as C, and also have automatic memory management which helps to prevent <a href="/wiki/Buffer_overrun" class="mw-redirect" title="Buffer overrun">buffer overrun</a> attacks.\n</p>\n<h3><span class="mw-headline" id="Client-side">Client-side</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=5" title="Edit section: Client-side">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The <a href="/wiki/De_facto_standard" title="De facto standard"><i>de facto</i></a> <a href="/wiki/Client-side_scripting" class="mw-redirect" title="Client-side scripting">client-side scripting</a> language for web sites is <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>.\nUsing JavaScript on the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) leads to the method of <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a> that allows dynamic creation and modification of a web page within the browser.\n</p><p>While client-side languages used in conjunction with forms are limited, they often can serve to do pre-<a href="/wiki/Data_validation" title="Data validation">validation</a> of the form data and/or to prepare the form data to send to a server-side program. This usage is being replaced, however, by <a href="/wiki/HTML5" title="HTML5">HTML5</a>\'s new <tt>input</tt> field types and <tt>required</tt> attribute.\n</p>\n<h3><span class="mw-headline" id="Server-side_execution">Server-side execution</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=6" title="Edit section: Server-side execution">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Server-side code can do a vast assortment of tasks to create dynamic web sites that, for technical or security reasons, client-side code cannot &#8212; from <a href="/wiki/Authentication" title="Authentication">authenticating</a> a <a href="/wiki/Login" title="Login">login</a>, to retrieving and storing data in a <a href="/wiki/Database" title="Database">database</a>, to <a href="/wiki/Spell_checker" title="Spell checker">spell checking</a>, to sending <a href="/wiki/E-mail" class="mw-redirect" title="E-mail">e-mail</a>. A significant advantage to server-side over client-side execution is the concentration of functionality onto the server rather than relying on different <a href="/wiki/Web_browser" title="Web browser">web browsers</a> to implement various functions in consistent, <a href="/wiki/Web_standards" title="Web standards">standardized</a> ways. In addition, processing forms on a server often results in increased security if server-side execution is designed not to trust the data supplied by the client and includes such techniques as <a href="/wiki/HTML_sanitization" title="HTML sanitization">HTML sanitization</a>. One disadvantage to server side code is <a href="/wiki/Scalability" title="Scalability">scalability</a>&#8212;server side processing for all users occurs on the server, while client side processing occurs on individual client computers.\n</p>\n<div class="thumb tright"><div class="thumbinner" style="width:222px;"><a href="/wiki/File:Zen-cart_Web-Shop_Frontend_Registration_Form.png" languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=7" title="Edit section: Interpreted Perl scripts are traditionally used as <a href="/wiki/Common_Gateway_Interface" title="Common Gateway Interface">Common Gateway Interface</a> applications (CGIs). In fact, Perl is such a common way to write CGIs that the two are often confused. This kind of script is frequently exploited by <a href="/wiki/Spammer" class="mw-redirect" title="Spammer">spammers</a>, however, and many of the most popular form-to-email scripts in use are vulnerable to hijacking for the purpose of sending spam emails. One of the most popular scripts of this type was <a rel="nofollow" class="external text" href="http://www.scriptarchive.com/formmail.html">"FormMail.pl"</a> made by <a href="/wiki/Matt%27s_Script_Archive" title="Matt&#39;s Script Archive">Matt\'s Script Archive</a>. Today, this script is no longer widely used in new development due to lack of updates, security concerns, and difficulty of configuration.\n</p>\n<h3><span class="mw-headline" id="Form_builders">Form builders</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Form_(HTML)&amp;action=edit&amp;section=11" title="Edit section: Form builders">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Some companies offer forms as a <a href="/wiki/Software_as_a_service" title="Software as a service">hosted service</a>.
