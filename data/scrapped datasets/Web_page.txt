browser</div></div></div>\n<p>A <b>web page</b> (or <b>webpage</b>) is a <a href="/wiki/Hypertext" title="Hypertext">hypertext</a> document provided by a <a href="/wiki/Website" title="Website">website</a> and displayed to a <a href="/wiki/User_(computing)" title="User (computing)">user</a> in a <a href="/wiki/Web_browser" title="Web browser">web browser</a>.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> A website typically consists of many web pages <a href="/wiki/Hyperlink" title="Hyperlink">linked</a> together in a coherent fashion. The name "web page" is a metaphor of <a href="/wiki/Page_(paper)" title="Page (paper)">paper pages</a> bound together into a <a href="/wiki/Book" title="Book">book</a>.\n</p>\n<div id="toc" class="toctext">References</span></a></li>\n</ul>\n</div>\n\n<h2><span class="mw-headline" id="Elements">Elements</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_page&amp;action=edit&amp;section=1" title="Edit section: Elements">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>The core element of a web page is one or more <a href="/wiki/Text_file" title="Text file">text files</a> written in the <a href="/wiki/HTML" title="HTML">Hypertext Markup Language</a> (HTML).<sup id="cite_ref-elems_2-0" class="reference"><a href="#cite_note-elems-2">&#91;2&#93;</a></sup> Many web pages also make use of <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> <a href="/wiki/Source_code" title="Source code">code</a> for <a href="/wiki/JavaScript#Website_client-side_usage" title="JavaScript">dynamic behavior</a> and <a href="/wiki/CSS" title="CSS">Cascading Style Sheets</a> (CSS) code for <a href="/wiki/Presentation_semantics" title="Presentation semantics">presentation semantics</a>.<sup id="cite_ref-elems_2-1" class="reference"><a href="#cite_note-elems-2">&#91;2&#93;</a></sup> <a href="/wiki/WebAssembly" title="WebAssembly">WebAssembly</a> <a href="/wiki/Executable" title="Executable">executables</a> can also be used for portions of page behavior. <a href="/wiki/Image" title="Image">Images</a>, <a href="/wiki/Video" title="Video">videos</a>, and other <a href="/wiki/Multimedia" title="Multimedia">multimedia</a> files are often embedded in web pages.\n</p>\n<h2><span class="mw-headline" id="Navigation">Navigation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_page&amp;action=edit&amp;section=2" title="Edit section: Navigation">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Each web page is identified by a distinct <a href="/wiki/URL" title="URL">Uniform Resource Locator</a> (URL). When the user inputs a URL into their browser, that page\'s elements are downloaded from <a href="/wiki/Web_server" title="Web server">web servers</a>. The browser then <a href="/wiki/Browser_engine" title="Browser engine">transforms</a> all of the elements into an interactive visual representation on the user\'s device.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup>\n</p><p>If the user <a href="/wiki/Point_and_click" title="Point and click">clicks</a> or <a href="/wiki/Touchscreen" title="Touchscreen">taps</a> a <a href="/wiki/Hyperlink" title="Hyperlink">link</a> to another page, the browser repeats this process to display the new page, which could be part of the current website or a different one.\n</p>\n<h2><span class="mw-headline" id="Deployment">Deployment</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_page&amp;action=edit&amp;section=3" title="Edit section: Deployment">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>From the perspective of <a href="/wiki/Server-side" title="Server-side">server-side</a> website deployment, there are two types of web pages: <a href="/wiki/Static_web_page" title="Static web page">static</a> and <a href="/wiki/Dynamic_web_page" title="Dynamic web page">dynamic</a>. Static pages are retrieved from the web server\'s <a href="/wiki/File_system" title="File system">file system</a> without any modification,<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> while dynamic pages must be created by the server <a href="https://en.wiktionary.org/wiki/on_the_fly" class="extiw" title="wikt:on the fly">on the fly</a>, typically reading from a <a href="/wiki/Database" title="Database">database</a> to fill out a <a href="/wiki/Web_template_system" title="Web template system">template</a>, before being sent to the user\'s browser.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>\n</p>\n<h2><span class="mw-headline" id="References">References</span><span
