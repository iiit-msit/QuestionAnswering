In <a href="/wiki/Server-side_scripting" title="Server-side scripting">server-side scripting</a>, <a href="/wiki/Parameter_(computer_programming)" title="Parameter (computer programming)">parameters</a> determine how the assembly of every new <a href="/wiki/Web_page" title="Web page">web page</a> proceeds, including the setting up of more client-side processing.\n</p><p>A <b>client-side dynamic web page</b> processes the web page using <a href="/w/index.php?title=HTML_scripting&amp;action=edit&amp;redlink=1" class="new" title="HTML scripting (page does not exist)">HTML scripting</a> running in the browser as it loads.  <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> and other <a href="/wiki/Scripting_language" title="Scripting language">scripting languages</a> determine the way the HTML in the received page is <a href="/wiki/Parsing" title="Parsing">parsed</a> into the <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a>, or DOM, that represents the loaded web page. The same client-side techniques can then dynamically update or change the DOM in the same way. Even though a web page can be dynamic on the client-side, it can still be hosted on a static <a href="/wiki/Web_hosting_service" title="Web hosting service">hosting service</a> such as <a href="/wiki/GitHub_Pages" class="mw-redirect" title="GitHub Pages">GitHub Pages</a> or <a href="/wiki/Amazon_S3" title="Amazon S3">Amazon S3</a> as long as there isn\'t any server-side code included.\n</p><p>A dynamic web page is then reloaded by the user or by a <a href="/wiki/Computer_program" title="Computer program">computer program</a> to change some variable content. The updating information could come from the server, or from changes made to that page\'s DOM. This may or may not truncate the <a href="/wiki/Web_browsing_history" title="Web browsing history">browsing history</a> or create a saved version to go back to, but a <i>dynamic web page update</i> using <a href="/wiki/Ajax_(programming)" title="Ajax (programming)">AJAX</a> technologies will neither create a page to go back to, nor truncate the web browsing history forward of the displayed page. Using AJAX, the end <a href="/wiki/User_(computing)" title="User (computing)">user</a> gets one dynamic page managed as a single page in the <a href="/wiki/Web_browser" title="Web browser">web browser</a> while the actual <a href="/wiki/Web_content" title="Web content">web content</a> rendered on that page can vary. The AJAX engine sits only on the browser requesting parts of its DOM, <i>the</i> DOM, for its client, from an application server.\n</p><p><a href="/wiki/DHTML" class="mw-redirect" title="DHTML">DHTML</a> is the umbrella term for technologies and methods used to create web pages that are not <a href="/wiki/Static_web_page" title="Static web page">static web pages</a>, though it has fallen out of common use since the popularization of AJAX, a term which is now itself rarely used. Client-side-scripting, server-side scripting, or a combination of these make for the dynamic web experience in a browser.\n</p>\n<div id="toc" class="toc" role="navigation" class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_web_page&amp;action=edit&amp;section=1" title="Edit section: Basic concepts">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Classical <a href="/wiki/Hypertext" title="Hypertext">hypertext</a> navigation, with <a href="/wiki/HTML" title="HTML">HTML</a> or <a href="/wiki/XHTML" title="XHTML">XHTML</a> alone, provides "static" content, meaning that the user requests a web page and simply views the page and the information on that page.\n</p><p>However, a web page can also provide a "live", "dynamic", or "interactive" user experience. Content (text, images, form fields, etc.) on a <a href="/wiki/Web_page" title="Web page">web page</a> can change, in response to different contexts or conditions.\n</p><p>There are two ways to create this kind of effect:\n</p>\n<ul><li>Using client-side scripting to change interface behaviors <i>within</i> a specific <a href="/wiki/Web_page" title="Web page">web page</a>, in response to mouse or keyboard actions or at specified timing events. In this case the dynamic behavior occurs within the <a href="/wiki/Look_and_feel" title="Look and feel">presentation</a>.</li>\n<li>Using <a href="/wiki/Server-side_scripting" title="Server-side scripting">server-side scripting</a> to change the supplied page source <i>between</i> pages, adjusting the sequence or reload of the <a href="/wiki/Web_page" title="Web page">web pages</a> or <a href="/wiki/Web_content" title="Web content">web content</a> supplied to the browser. Server responses may be determined by such conditions as data in a posted <a href="/wiki/HTML_form" class="mw-redirect" title="HTML form">HTML form</a>, parameters in the <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a>, the type of browser being used, the passage of time, or a database or server <a href="/wiki/State_(computer_science)" title="State (computer science)">state</a>.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup></li></ul>\n<p>Web pages that use client-side scripting must use presentation technology broadly called <a href="/wiki/Rich_Internet_application#Methods_and_techniques" class="mw-redirect" title="Rich Internet application">rich interfaced pages</a>. <a href="/wiki/Client-side" title="Client-side">Client-side</a> <a href="/wiki/Scripting_language" title="Scripting language">scripting languages</a> like <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> or <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a>, used for <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a> (DHTML) and <a href="/wiki/Adobe_Flash" title="Adobe Flash">Flash</a> technologies respectively, are frequently used to orchestrate media types (sound, animations, changing text, etc.) of the presentation. The scripting also allows use of <a href="/wiki/Remote_scripting" title="Remote scripting">remote scripting</a>, a technique by which the DHTML page requests additional information from a server, using a <a href="/wiki/HTML_element#Frames" title="HTML element">hidden Frame</a>, <a href="/wiki/XMLHttpRequest" title="XMLHttpRequest">XMLHttpRequests</a>, or a <a href="/wiki/Web_service" title="Web service">web service</a>.\n</p><p>Web pages that use server-side scripting are often created with the help of <a href="/wiki/Server-side" title="Server-side">server-side</a> languages such as <a href="/wiki/PHP" title="PHP">PHP</a>, <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/Active_Server_Pages" title="Active Server Pages">ASP</a>, <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>, <a href="/wiki/JavaServer_Pages" class="mw-redirect" title="JavaServer Pages">JSP</a>, <a href="/wiki/ColdFusion" class="mw-redirect" title="ColdFusion">ColdFusion</a> and other languages. These server-side languages typically use the <a href="/wiki/Common_Gateway_Interface" title="Common Gateway Interface">Common Gateway Interface</a> (CGI) to produce <i>dynamic web pages</i>. These kinds of pages can also use, on the client-side, the first kind (DHTML, etc.).\n</p>\n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_web_page&amp;action=edit&amp;section=2" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>It is difficult to be precise about "dynamic web page beginnings" or chronology because the precise concept makes sense only after the "widespread development of web pages". <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">HTTP</a> has been in use since 1990, <a href="/wiki/Hypertext_Markup_Language" class="mw-redirect" title="Hypertext Markup Language">HTML</a>, as standard, since 1996. The web browser\'s rise in popularity started with <a href="/wiki/Mosaic_(web_browser)" title="Mosaic (web browser)">Mosaic</a> in 1993. It is obvious, however, that the concept of dynamically driven websites predates the Internet, and in fact HTML. For example, in 1990, before general public use of the Internet, a dynamically driven remotely accessed menu system was implemented by Susan Biddlecomb, who was Director of Computer Support of the USC Health Care system at the <a href="/wiki/University_of_Southern_California" title="University of Southern California">University of Southern California</a> <a href="/wiki/Bulletin_board_system" title="Bulletin board system">BBS</a> on a 16 line <a href="/wiki/The_Bread_Board_System" title="The Bread Board System">TBBS</a> system with TDBS add-on.database.\n</p><p>The introduction of JavaScript (then known as LiveScript) enabled the production of client-side dynamic web pages, with JavaScript code executed in the client\'s browser.<sup id="cite_ref-JavaScript_2-0" class="reference"><a href="#cite_note-JavaScript-2">&#91;2&#93;</a></sup> The letter "J" in the term <a href="/wiki/AJAX" class="mw-redirect" title="AJAX">AJAX</a> originally indicated the use of JavaScript, as well as <a href="/wiki/XML" title="XML">XML</a>. With the rise of server side JavaScript processing, for example, <a href="/wiki/Node.js" title="Node.js">Node.js</a>, originally developed in 2009, JavaScript is also used to dynamically create pages on the server that are sent fully formed to clients.\n</p><p>Execusite introduced the first dynamic website solution for the professional marketplace in June 1997. Execusite was acquired by Website Pros (now <a href="/wiki/Web.com" title="Web.com">Web.com</a>) in January 2000. During the bust cycle of the <a href="/wiki/Dot-com_bubble" title="Dot-com bubble">Dot-com bubble</a>, the original Execusite founders bought back the company from Website Pros (December 2000). Execusite was later acquired by <a href="/wiki/Wolters-Kluwer" class="mw-redirect" title="Wolters-Kluwer">Wolters-Kluwer</a> in December 2001 and was re-branded as CCH Site Builder.\n</p>\n<h2><span class="mw-headline" id="Server-side_scripting">Server-side scripting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_web_page&amp;action=edit&amp;section=3" title="Edit section: Server-side scripting">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<div class="thumb tright"><div class="thumbinner" style="width:222px;"><a href="/wiki/File:2013vmvClientServer.png" class="image"><img alt="There are four steps in the client-server ecosystem: the browser, the HTTP request, the web server, and the response. Server-side scripting occurs when a dynamic web page or resource is processed and generated by the web server." src="//upload.wikimedia.org/wikipedia/commons/thumb/c/ce/2013vmvClientServer.png/220px-2013vmvClientServer.png" decoding="async" width="220" height="132" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/c/ce/2013vmvClientServer.png/330px-2013vmvClientServer.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/c/ce/2013vmvClientServer.png/440px-2013vmvClientServer.png 2x" data-file-width="800" data-file-height="481" /></a>  <div class="thumbcaption"><div class="magnify"><a href="/wiki/File:2013vmvClientServer.png" class="internal" title="Enlarge"></a></div>When a request to a dynamic web page or resource is made, the <a href="/wiki/Application_server" title="Application server">application server</a> processes the request using its server-side language.</div></div></div>\n<p>A program running on a <a href="/wiki/Web_server" title="Web server">web server</a> (<a href="/wiki/Server-side_scripting" title="Server-side scripting">server-side scripting</a>) is used to generate the web content on various web pages, manage user sessions, and control workflow. Server responses may be determined by such conditions as data in a posted <a href="/wiki/HTML_form" class="mw-redirect" title="HTML form">HTML form</a>, parameters in the <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a>, the type of browser being used, the passage of time, or a database or server <a href="/wiki/State_(computer_science)" title="State (computer science)">state</a>.\n</p><p>Such web pages are often created with the help of <a href="/wiki/Server-side" title="Server-side">server-side</a> languages such as <a href="/wiki/Active_Server_Pages" title="Active Server Pages">ASP</a>, <a href="/wiki/ColdFusion" class="mw-redirect" title="ColdFusion">ColdFusion</a>, <a href="/wiki/Go_(programming_language)" title="Go (programming language)">Go</a>, <a href="/wiki/JavaScript#Server-side_JavaScript" title="JavaScript">JavaScript</a>, <a href="/wiki/Perl" title="Perl">Perl</a>, <a href="/wiki/PHP" title="PHP">PHP</a>, <a href="/wiki/Ruby_(programming_language)" title="Ruby (programming language)">Ruby</a>, <a href="/wiki/Python_(programming_language)" title="Python (programming language)">Python</a>, <a href="/wiki/WebDNA" title="WebDNA">WebDNA</a> and other languages, by a <b>support server</b> that can run on the same hardware as the web server. These server-side languages often use the <a href="/wiki/Common_Gateway_Interface" title="Common Gateway Interface">Common Gateway Interface</a> (CGI) to produce <i>dynamic web pages</i>. Two notable exceptions are <a href="/wiki/ASP.NET" title="ASP.NET">ASP.NET</a>, and <a href="/wiki/JavaServer_Pages" class="mw-redirect" title="JavaServer Pages">JSP</a>, which reuse CGI concepts in their APIs but actually dispatch all web requests into a shared virtual machine.\n</p><p>The server-side languages are used to embed tags or markers within the source file of the web page on the web server.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> When a user on a client computer requests that web page, the web server interprets these tags or markers to perform actions on the server. For example, the server may be instructed to insert information from a database or information such as the current date.\n</p><p>Dynamic web pages are often <a href="/wiki/Cache_(computing)" title="Cache (computing)">cached</a> when there are few or no changes expected and the page is anticipated to receive considerable amount of web traffic that would create slow load times for the server if it had to generate the pages on the fly for each request.\n</p>\n<h2><span class="mw-headline" id="Client-side_scripting">Client-side scripting</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_web_page&amp;action=edit&amp;section=4" title="Edit section: Client-side scripting">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Client-side scripting is changing interface behaviors within a specific web page in response to mouse or keyboard actions, or at specified timing events. In this case, the dynamic behavior occurs within the <a href="/wiki/Look_and_feel" title="Look and feel">presentation</a>.  The client-side content is generated on the user\'s local computer system.<sup id="cite_ref-Doyle2008_4-0" class="reference"><a href="#cite_note-Doyle2008-4">&#91;4&#93;</a></sup>\n</p><p>Such web pages use presentation technology called <a href="/wiki/Rich_Internet_application#Methods_and_techniques" class="mw-redirect" title="Rich Internet application">rich interfaced pages</a>. Client-side scripting languages like <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> or <a href="/wiki/ActionScript" title="ActionScript">ActionScript</a>, used for <a href="/wiki/Dynamic_HTML" title="Dynamic HTML">Dynamic HTML</a> (DHTML) and <a href="/wiki/Adobe_Flash" title="Adobe Flash">Flash</a> technologies respectively, are frequently used to orchestrate media types (sound, animations, changing text, etc.) of the presentation. Client-side scripting also allows the use of <a href="/wiki/Remote_scripting" title="Remote scripting">remote scripting</a>, a technique by which the DHTML page requests additional information from a server, using a <a href="/wiki/HTML_element#Frames" title="HTML element">hidden frame</a>, <a href="/wiki/XMLHttpRequest" title="XMLHttpRequest">XMLHttpRequests</a>, or a <a href="/wiki/Web_service" title="Web service">Web service</a>.\n</p><p>The first widespread use of JavaScript was in 1997, when the language was standardized as <a href="/wiki/ECMAScript" title="ECMAScript">ECMAScript</a> and implemented in <a href="/wiki/Netscape" title="Netscape">Netscape 3</a>.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>\n</p>\n<dl><dt>Example</dt></dl>\n<p>The client-side content is generated on the client\'s computer. The web browser retrieves a page from the server, then processes the code embedded in the page (typically written in <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>) and displays the retrieved page\'s content to the user.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup>\n</p><p>The <code>innerHTML</code> property (or write command) can illustrate the client-side dynamic page generation: two distinct pages, A and B, can be regenerated (by an "event response dynamic") as <code>document.innerHTML = A</code> and <code>document.innerHTML = B</code>; or "on load dynamic" by <code>document.write(A)</code> and <code>document.write(B)</code>.\n</p>\n<h2><span class="mw-headline" id="Combination_technologies">Combination technologies</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Dynamic_web_page&amp;action=edit&amp;section=5" title="Edit section: Combination technologies">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>All of the client and server components that collectively build a dynamic web page are called a <a href="/wiki/Web_application" title="Web application">web application</a>.  Web applications manage user interactions, state, security, and performance.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup>\n</p><p><a href="/wiki/Ajax_(programming)" title="Ajax (programming)">Ajax</a> uses a combination of both client-side scripting and server-side requests. It is a web application development technique for dynamically interchanging content, and it sends requests to the server for data in order to do so.  The server returns the requested data which is then processed by a client-side script.  This technique can reduce server load time because the client does not request the entire webpage to be regenerated by the server\'s language parser; only the content that will change is transmitted.  <a href="/wiki/Google_Maps" title="Google Maps">Google Maps</a> is an example of a web application that uses Ajax techniques.\n</p><p>A <a href="/wiki/Web_client" class="mw-redirect" title="Web client">web client</a>, such as a web browser, can act as its own server, accessing data from many different servers, such as Gopher, FTP, NNTP (Usenet) and HTTP, to build a page. <a href="/wiki/Hypertext_Transport_Protocol" class="mw-redirect" title="Hypertext Transport Protocol">HTTP</a> supports uploading documents from the client back to the server. There are several <a href="/wiki/HTTP_method" class="mw-redirect" title="HTTP method">HTTP methods</a> for doing this.\n</p>\n<h2><span class="mw-headline" id="See_also">See also</span><span
