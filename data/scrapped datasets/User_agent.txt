nomobile noexcerpt noprint searchaux" style="display:none">Software acting on behalf of a user</div>\n<p>\nIn <a href="/wiki/Computing" title="Computing">computing</a>, a <b>user agent</b> is any software, acting on behalf of a <a href="/wiki/User_(computing)" title="User (computing)">user</a>, which "retrieves, renders and facilitates end-user interaction with Web content."<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> A user agent is therefore a special kind of <a href="/wiki/Software_agent" title="Software agent">software agent</a>. \n</p><p>Some prominent examples of user agents are <a href="/wiki/Web_browsers" class="mw-redirect" title="Web browsers">web browsers</a> and <a href="/wiki/Email_client" title="Email client">email readers</a>. Often, a user agent acts as the <a href="/wiki/Client_(computing)" title="Client (computing)">client</a> in a <a href="/wiki/Client%E2%80%93server_model" title="Client\xe2\x80\x93server model">client\xe2\x80\x93server system</a>. In some contexts, such as within the <a href="/wiki/Session_Initiation_Protocol" title="Session Initiation Protocol">Session Initiation Protocol</a> (SIP), the term <i>user agent</i> refers to both end points of a communications session.<sup id="cite_ref-rfc3261_2-0" class="reference"><a href="#cite_note-rfc3261-2">&#91;2&#93;</a></sup>\n</p>\n<div id="toc" class="toc" role="navigation" class="tocnumber">4</span> <span class="toctext">References</span></a></li>\n</ul>\n</div>\n\n<h2><span class="mw-headline" id="User_agent_identification">User agent identification</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=1" title="Edit section: User agent identification">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>When a software agent operates in a network protocol, it often identifies itself, its application type, <a href="/wiki/Operating_system" title="Operating system">operating system</a>, software vendor, or software revision, by submitting a characteristic identification <a href="/wiki/String_(computer_science)" title="String (computer science)">string</a> to its operating peer. In <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">HTTP</a>,<sup id="cite_ref-rfc7231_3-0" class="reference"><a href="#cite_note-rfc7231-3">&#91;3&#93;</a></sup> SIP,<sup id="cite_ref-rfc3261_2-1" class="reference"><a href="#cite_note-rfc3261-2">&#91;2&#93;</a></sup> and NNTP<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup> protocols, this identification is transmitted in a header field <i>User-Agent</i>. <a href="/wiki/Internet_bot" title="Internet bot">Bots</a>, such as Web crawlers, often also include a <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a> and/or <a href="/wiki/E-mail_address" class="mw-redirect" title="E-mail address">e-mail address</a> so that the <a href="/wiki/Webmaster" title="Webmaster">Webmaster</a> can contact the operator of the bot.\n</p>\n<h2><span class="mw-headline" id="Use_in_HTTP">Use in HTTP</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=2" title="Edit section: Use in HTTP">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>In HTTP, the <b>User-Agent string</b> is often used for <a href="/wiki/Content_negotiation" title="Content negotiation">content negotiation</a>, where the origin server selects suitable content or operating parameters for the response. For example, the User-Agent string might be used by a web server to choose variants based on the known capabilities of a particular version of client software. The concept of content tailoring is built into the HTTP standard in <a rel="nofollow" class="external text" href="https://tools.ietf.org/html/rfc1945#page-46">RFC 1945</a> "for the sake of tailoring responses to avoid particular user agent limitations.\xe2\x80\x9d\n</p><p>The User-Agent string is one of the criteria by which Web crawlers may be excluded from accessing certain parts of a website using the <a href="/wiki/Robots_Exclusion_Standard" class="mw-redirect" title="Robots Exclusion Standard">Robots Exclusion Standard</a> (<i>robots.txt</i> file).\n</p><p>As with many other HTTP request headers, the information in the "User-Agent" string contributes to the information that the client sends to the server, since the string can vary considerably from user to user.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Format_for_human-operated_web_browsers">Format for human-operated web browsers</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=3" title="Edit section: Format for human-operated web browsers">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The User-Agent string format is currently specified by section 5.5.3 of  <a rel="nofollow" class="external text" href="http://tools.ietf.org/html/rfc7231#section-5.5.3">HTTP/1.1 Semantics and Content</a>. The format of the User-Agent string in HTTP is a list of product tokens (keywords) with optional comments. For example, if a user\'s product were called WikiBrowser, their user agent string might be <i>WikiBrowser/1.0 Gecko/1.0</i>. The "most important" product component is listed first.\n</p><p>The parts of this string are as follows:\n</p>\n<ul><li>product name and version (<i>WikiBrowser/1.0</i>)</li>\n<li>layout engine and version (<i>Gecko/1.0</i>)</li></ul>\n<p>During the first <a href="/wiki/Browser_wars" title="Browser wars">browser war</a>, many web servers were configured to send web pages that required advanced features, including <a href="/wiki/Framing_(World_Wide_Web)" class="mw-redirect" title="Framing (World Wide Web)">frames</a>, to clients that were identified as some version of <a href="/wiki/Mozilla" title="Mozilla">Mozilla</a> only.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> Other browsers were considered to be older products such as <a href="/wiki/Mosaic_(web_browser)" title="Mosaic (web browser)">Mosaic</a>, <a href="/wiki/Cello_(web_browser)" title="Cello (web browser)">Cello</a>, or <a href="/wiki/Samba_(web_browser)" class="mw-redirect" title="Samba (web browser)">Samba</a>, and would be sent a bare bones HTML document.\n</p><p>For this reason, most Web browsers use a User-Agent string value as follows:\n</p><p><i>Mozilla/[version] ([system and browser information]) [platform] ([platform details]) [extensions]</i>. For example, Safari on the iPad has used the following:\n</p>\n<pre>Mozilla/5.0 (iPad; U; CPU OS 3_2_1 like Mac OS X; en-us) AppleWebKit/531.21.10 (KHTML, like Gecko) Mobile/7B405\n</pre>\n<p>The components of this string are as follows:\n</p>\n<ul><li><i>Mozilla/5.0</i>: Previously used to indicate compatibility with the Mozilla rendering engine.</li>\n<li><i>(iPad; U; CPU OS 3_2_1 like Mac OS X; en-us)</i>: Details of the system in which the browser is running.</li>\n<li><i>AppleWebKit/531.21.10</i>: The platform the browser uses.</li>\n<li><i>(KHTML, like Gecko)</i>: Browser platform details.</li>\n<li><i>Mobile/7B405</i>: This is used by the browser to indicate specific enhancements that are available directly in the browser or through third parties. An example of this is Microsoft Live Meeting which registers an extension so that the Live Meeting service knows if the software is already installed, which means it can provide a streamlined experience to joining meetings.</li></ul>\n<p>Before migrating to the <a href="/wiki/Chromium_(web_browser)" title="Chromium (web browser)">Chromium</a> code base, <a href="/wiki/Opera_(web_browser)" title="Opera (web browser)">Opera</a> was the most widely used web browser that did not have the User-Agent string with "Mozilla" (instead beginning it with "Opera"). Since July 15, 2013,<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> Opera\'s User-Agent string begins with "Mozilla/5.0" and, to avoid encountering legacy server rules, no longer includes the word "Opera" (instead using the string "OPR" to denote the Opera version).\n</p>\n<h3><span id="Format_for_automated_agents_.28bots.29"></span><span class="mw-headline" id="Format_for_automated_agents_(bots)">Format for automated agents (bots)</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=4" title="Edit section: Format for automated agents (bots)">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Automated web crawling tools can use a simplified form, where an important field is contact information in case of problems.  By convention the word "bot" is included in the name of the agent.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> For example:\n</p>\n<pre>Googlebot/2.1 (+http://www.google.com/bot.html)\n</pre>\n<p>Automated agents are expected to follow rules in a special file called "<a href="/wiki/Robots.txt" class="mw-redirect" title="Robots.txt">robots.txt</a>".\n</p>\n<h3><span class="mw-headline" id="User_agent_spoofing">User agent spoofing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=5" title="Edit section: User agent spoofing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The popularity of various Web browser products has varied throughout the Web\'s history, and this has influenced the design of websites in such a way that websites are sometimes designed to work well only with particular browsers, rather than according to uniform standards by the <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a> (W3C) or the <a href="/wiki/Internet_Engineering_Task_Force" title="Internet Engineering Task Force">Internet Engineering Task Force</a> (IETF). Websites often include code to detect browser version to adjust the page design sent according to the user agent string received. This may mean that less-popular browsers are not sent complex content (even though they might be able to deal with it correctly) or, in extreme cases, refused all content.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup> Thus, various browsers have a feature to <i>cloak</i> or <i>spoof</i> their identification to force certain server-side content. For example, the Android browser identifies itself as <a href="/wiki/Safari_(web_browser)" title="Safari (web browser)">Safari</a> (among other things) in order to aid compatibility.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup><sup id="cite_ref-11" class="reference"><a href="#cite_note-11">&#91;11&#93;</a></sup>\n</p><p>Other HTTP client programs, like <a href="/wiki/Download_manager" title="Download manager">download managers</a> and <a href="/wiki/Offline_browser" class="mw-redirect" title="Offline browser">offline browsers</a>, often have the ability to change the user agent string.\n</p><p><a href="/wiki/Spam_bots" class="mw-redirect" title="Spam bots">Spam bots</a> and <a href="/wiki/Web_scrapers" class="mw-redirect" title="Web scrapers">Web scrapers</a> often use fake user agents.\n</p><p>A result of user agent spoofing may be that collected <a href="/wiki/Usage_share_of_web_browsers" title="Usage share of web browsers">statistics of Web browser usage</a> are inaccurate.\n</p>\n<h3><span class="mw-headline" id="User_agent_sniffing">User agent sniffing</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=6" title="Edit section: User agent sniffing">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/Browser_sniffing" title="Browser sniffing">Browser sniffing</a></div>\n<p><i>User agent sniffing</i> is the practice of websites showing different or adjusted content when viewed with certain user agents. An example of this is <a href="/wiki/Microsoft_Exchange_Server" title="Microsoft Exchange Server">Microsoft Exchange Server</a> 2003\'s Outlook Web Access feature. When viewed with Internet Explorer 6 or newer, more functionality is displayed compared to the same page in any other browsers. User agent sniffing is considered poor practice, since it encourages browser-specific design and penalizes new browsers with unrecognized user agent identifications. Instead, the W3C recommends creating standard HTML markup,<sup id="cite_ref-www.w3.org_12-0" class="reference"><a href="#cite_note-www.w3.org-12">&#91;12&#93;</a></sup>  allowing correct rendering in as many browsers as possible, and to test for specific browser features rather than particular browser versions or brands.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup>\n</p><p>Websites intended for display by mobile phones often rely on user agent sniffing, since <a href="/wiki/Mobile_browser" title="Mobile browser">mobile browsers</a> often differ greatly from each other.\n</p>\n<h3><span class="mw-headline" id="Encryption_strength_notations">Encryption strength notations</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=7" title="Edit section: Encryption strength notations">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Web browsers created in the United States, such as <a href="/wiki/Netscape_Navigator" title="Netscape Navigator">Netscape Navigator</a> and <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a>, previously used the letters U, I, and N to specify the <a href="/wiki/Encryption" title="Encryption">encryption</a> strength in the user agent string. Until 1996, when the United States government disallowed encryption with keys longer than 40&#160;bits to be exported, vendors shipped various browser versions with different encryption strengths. "U" stands for "USA" (for the version with 128-bit encryption), "I" stands for "International"&#160;&#8211;&#32; the browser has 40-bit encryption and can be used anywhere in the world&#160;&#8211;&#32; and "N" stands (<i>de facto</i>) for "None" (no encryption).<sup id="cite_ref-zawinski-old_14-0" class="reference"><a href="#cite_note-zawinski-old-14">&#91;14&#93;</a></sup> Following the lifting of <a href="/wiki/Export_of_cryptography_in_the_United_States#PC_era" class="mw-redirect" title="Export of cryptography in the United States">export restrictions</a>, most vendors supported 256-bit encryption.\n</p>\n<h3><span class="mw-headline" id="Deprecation_of_User-Agent_header">Deprecation of User-Agent header</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=User_agent&amp;action=edit&amp;section=8" title="Edit section: Deprecation of User-Agent header">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>In 2020, <a href="/wiki/Google" title="Google">Google</a> announced that they would be phasing out support for the User-Agent header in their <a href="/wiki/Google_Chrome" title="Google Chrome">Google Chrome</a> browser. They stated that other major web browser vendors were supportive of the move, but that they did not know when other vendors would follow suit.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup> Google stated that a new feature called <a href="/wiki/Client_Hints" class="mw-redirect" title="Client Hints">Client Hints</a> would replace the functionality of the User-Agent string.<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup>\n</p>\n<h2><span class="mw-headline" id="See_also">See also</span><span
