href="#cite_note-2">&#91;2&#93;</a></sup></td></tr></tbody></table>\n<p><b>archive.today</b> (formerly <b>archive.is</b>) is an <a href="/wiki/Archive_site" title="Archive site">archive site</a> which stores snapshots of web pages.<sup id="cite_ref-cnet_3-0" class="reference"><a href="#cite_note-cnet-3">&#91;3&#93;</a></sup> It retrieves one page at a time similar to <a href="/wiki/WebCite" title="WebCite">WebCite</a>, smaller than 50MB each, but with support for <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>-heavy sites such as <a href="/wiki/Google_Maps" title="Google Maps">Google Maps</a> and <a href="/wiki/Progressive_web_application" title="Progressive web application">progressive web applications</a> such as <a href="/wiki/Twitter" title="Twitter">Twitter</a>.\n</p><p>Archive.today records simultaneously two different \'snapshots\' of a web-page. One is "Webpage" which includes any functional live links that are in the original. The other is "Screenshot" which provides a static and non-interactive visualization of the representation.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" role="button" id="toctogglecheckbox" class="mw-editsection-bracket">]</span></span></h2>\n<h3><span class="mw-headline" id="Functionality">Functionality</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=2" title="Edit section: Functionality">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Archive.today can capture individual pages in response to explicit user requests.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup><sup id="cite_ref-vice_stealing_6-0" class="reference"><a href="#cite_note-vice_stealing-6">&#91;6&#93;</a></sup><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> Since its beginning, Archive.Today supports <a href="/wiki/Web_crawler" title="Web crawler">crawling</a> pages with <a href="/wiki/URL" title="URL">URLs</a> containing a now-deprecated <a href="/wiki/Hash-bang_fragment" class="mw-redirect" title="Hash-bang fragment">hash-bang fragment</a> (<style data-mw-deduplicate="TemplateStyles:r886049734">.mw-parser-output .monospaced{font-family:monospace,monospace}</style><span class="monospaced">#!</span>).<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup>\n</p><p>Archive.today records only text and images, excluding video, <a href="/wiki/XML" title="XML">XML</a>, <a href="/wiki/Rich_Text_Format" title="Rich Text Format">RTF</a>, <a href="/wiki/Spreadsheet" title="Spreadsheet">spreadsheet</a> (<a href="/wiki/Microsoft_Excel_file_format" class="mw-redirect" title="Microsoft Excel file format">xls</a> or <a href="/wiki/OpenDocument" title="OpenDocument">ods</a>) and other <a href="/wiki/Static_web_page" title="Static web page">non-static</a> content. It keeps track of the history of snapshots saved, returning to the user a request for confirmation before adding a new snapshot of an already saved Internet address.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup>\n</p><p>Pages are captured with 1024 pixels of browser width. <a href="/wiki/CSS" title="CSS">CSS</a> is converted to <a href="/wiki/CSS#Sources" title="CSS">inline</a> CSS, removing <a href="/wiki/Responsive_web_design" title="Responsive web design">responsive web design</a> and selectors such as <code>:hover</code> and <code>:active</code>. Content generated using <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a> during the crawling process appears in a frozen state.<sup id="cite_ref-frozen_10-0" class="reference"><a href="#cite_note-frozen-10">&#91;10&#93;</a></sup>\n<a href="/wiki/HTML_class" class="mw-redirect" title="HTML class">HTML class</a> names are preserved inside the <code>old-class</code> <a href="/wiki/HTML_attribute" title="HTML attribute">attribute</a>.\n</p><p>When <a href="/wiki/Selection_(user_interface)" title="Selection (user interface)">text is selected</a>, a JavaScript applet<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (December 2020)">clarification needed</span></a></i>&#93;</sup> generates a <a href="/wiki/URL_fragment" class="mw-redirect" title="URL fragment">URL fragment</a> seen in the browser\'s <a href="/wiki/URL_bar" class="mw-redirect" title="URL bar">URL bar</a> that automatically highlights that portion of the text when visited again.\n</p><p>Web pages cannot be <a href="/wiki/Redundancy_(engineering)" title="Redundancy (engineering)">duplicated</a> from <i>archive.is</i> to <i><a href="/wiki/Web.archive.org" class="mw-redirect" title="Web.archive.org">web.archive.org</a></i> as <a href="/wiki/Remote_backup_service" title="Remote backup service">second-level backup</a>, as archive.is places an exclusion for Wayback Machine and does not save its snapshots in WARC format. The reverse\xe2\x80\x94from <i>web.archive.org</i> to <i>archive.is</i>\xe2\x80\x94is possible,<sup id="cite_ref-Wayback_Mach_to_archive.is_11-0" class="reference"><a href="#cite_note-Wayback_Mach_to_archive.is-11">&#91;11&#93;</a></sup><sup class="noprint Inline-Template noprint Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Verifiability#Wikipedia_and_sources_that_mirror_or_use_it" title="Wikipedia:Verifiability"><span title="This claim cites another Wikipedia article. Articles need references to reliable third-party sources. (February 2021)">circular reference</span></a></i>&#93;</sup> but the copy usually takes more time than a direct capture. Some web sites get deleted from <a href="/wiki/Internet_Archive" title="Internet Archive">Internet Archive</a>\'s listings retroactively or blocked from being saved due to their <a href="/wiki/Robots.txt" class="mw-redirect" title="Robots.txt">robots.txt</a> file, but Archive.today does not use this.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2020)">citation needed</span></a></i>&#93;</sup>\n</p><p>The research toolbar enables advanced keywords operators, using <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">*</code> as the <a href="/wiki/Wildcard_character#Computing" title="Wildcard character">wildcard character</a>. A couple of <a href="/wiki/Quotation_mark" title="Quotation mark">quotation marks</a> address the search to an exact sequence of keywords present in the title or in the body of the webpage, whereas the <i>insite</i> operator restricts it to a specific Internet domain.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">&#91;12&#93;</a></sup>\n</p><p>Once a web page is archived, it cannot be deleted directly by any Internet user.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup> \n</p><p>While saving a <a href="/wiki/Linked_list#Basic_concepts_and_nomenclature" title="Linked list">dynamic list</a>, archive.today searchbox shows only a result that links the previous and the following section of the list (e.g. 20 links for page).<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup> The other web pages saved are filtered, and sometimes may be found by one of their occurrences.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2020)">citation needed</span></a></i>&#93;</sup>\n</p><p>The search feature is backed by Google CustomSearch. If it delivers no results, archive.is attempts to utilize <a href="/wiki/Yandex_Search" title="Yandex Search">Yandex Search</a>.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2020)">citation needed</span></a></i>&#93;</sup>\n</p><p>If a page has already been archived, archive.is asks the user to confirm archiving a new revision, instead of immediately archiving it.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2020)">citation needed</span></a></i>&#93;</sup>\n</p><p>While loading a page, a list of URLs to individual page elements among their content sizes, <a href="/wiki/List_of_HTTP_status_codes" title="List of HTTP status codes">HTTP statuses</a> and <a href="/wiki/MIME_type" class="mw-redirect" title="MIME type">MIME types</a> is shown. This list can only be viewed during the crawling process.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (July 2020)">citation needed</span></a></i>&#93;</sup>\n</p><p>One can download archived pages as a ZIP file, except pages archived since 29 November 2019, when Archive.Today changed their browser engine from <a href="/wiki/PhantomJS" title="PhantomJS">PhantomJS</a> to <a href="/wiki/Chromium_(web_browser)" title="Chromium (web browser)">Chromium</a>.<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup>\n</p><p>Since July 2013, archive.today supports the <a href="/wiki/Memento_Project" title="Memento Project">Memento Project</a> <a href="/wiki/Application_programming_interface" class="mw-redirect" title="Application programming interface">application programming interface</a> (API).<sup id="cite_ref-WSRG_16-0" class="reference"><a href="#cite_note-WSRG-16">&#91;16&#93;</a></sup><sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup>\n</p>\n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=3" title="Edit section: History">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Archive.today was founded in 2012. The site originally branded itself as archive.today, but in May 2015, changed the primary mirror to archive.is.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup>\n</p><p>In January 2019, it began to deprecate the archive.is domain in favor of the archive.today mirror.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup>\n</p>\n<h2><span class="mw-headline" id="Worldwide_availability">Worldwide availability</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=4" title="Edit section: Worldwide availability">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<h3><span class="mw-headline" id="Australia">Australia</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=5" title="Edit section: Australia">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/Internet_censorship_in_Australia" title="Internet censorship in Australia">Internet censorship in Australia</a></div>\n<p>In March 2019, the site was blocked for six months by several <a href="/wiki/Australia" title="Australia">Australian</a> internet providers in the aftermath of the <a href="/wiki/Christchurch_mosque_shootings" title="Christchurch mosque shootings">Christchurch mosque shootings</a> in an attempt to limit distribution of the footage of the attack.<sup id="cite_ref-20" class="reference"><a href="#cite_note-20">&#91;20&#93;</a></sup><sup id="cite_ref-21" class="reference"><a href="#cite_note-21">&#91;21&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="China">China</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=6" title="Edit section: China">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/Internet_censorship_in_China" title="Internet censorship in China">Internet censorship in China</a></div>\n<p>According to <a href="/wiki/GreatFire" title="GreatFire">GreatFire</a>.org, archive.today has been blocked in China since March 2016,<sup id="cite_ref-greatfire-archive-is_22-0" class="reference"><a href="#cite_note-greatfire-archive-is-22">&#91;22&#93;</a></sup> archive.li since September 2017,<sup id="cite_ref-23" class="reference"><a href="#cite_note-23">&#91;23&#93;</a></sup> and archive.fo since July 2018.<sup id="cite_ref-24" class="reference"><a href="#cite_note-24">&#91;24&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Finland">Finland</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=7" title="Edit section: Finland">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>On 21 July 2015, the operators blocked access to the service from all <a href="/wiki/Finland" title="Finland">Finnish</a> <a href="/wiki/IP_address" title="IP address">IP addresses</a>, stating on <a href="/wiki/Twitter" title="Twitter">Twitter</a> that they did this in order to avoid escalating a dispute they allegedly had with the Finnish government.<sup id="cite_ref-ilta_Suom_25-0" class="reference"><a href="#cite_note-ilta_Suom-25">&#91;25&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Russia">Russia</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=8" title="Edit section: Russia">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/Internet_censorship_in_Russia" title="Internet censorship in Russia">Internet censorship in Russia</a></div>\n<p>In <a href="/wiki/Russia" title="Russia">Russia</a>, only HTTP access is possible; HTTPS connections are blocked.<sup id="cite_ref-26" class="reference"><a href="#cite_note-26">&#91;26&#93;</a></sup><sup id="cite_ref-27" class="reference"><a href="#cite_note-27">&#91;27&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Worldwide">Worldwide</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Archive.today&amp;action=edit&amp;section=9" title="Edit section: Worldwide">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Archive.today currently blocks requests from <a href="/wiki/Cloudflare" title="Cloudflare">Cloudflare</a>\'s recursive DNS resolver, <a href="/wiki/1.1.1.1" title="1.1.1.1">1.1.1.1</a>.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28">&#91;28&#93;</a></sup> Archive.today insists that recursive DNS resolvers include the geolocation of the user making the DNS lookup. For privacy reasons, Cloudflare specifically does not include the geolocation of the user making the request. As a result, the archive.today DNS servers intentionally return invalid responses when queried by a Cloudflare recursive DNS resolver.<sup id="cite_ref-29" class="reference"><a href="#cite_note-29">&#91;29&#93;</a></sup><sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items" title="Wikipedia:Manual of Style/Dates and numbers"><span title="Doesn&#39;t seem to be true any longer. I can access archive.today with CloudFlare 1.1.1.1 DNS. (April 2021)">needs update</span></a></i>&#93;</sup>\n</p><p>Additionally, since late 2018, Archive.today has implemented a <a href="/wiki/Data_cap" title="Data cap">data cap</a> limitation, presumably to help protect against <a href="/wiki/Denial-of-service_attacks" class="mw-redirect" title="Denial-of-service attacks">denial-of-service attacks</a>. Individual users can only archive and/or retrieve approximately 10 to 20 megabytes of data per day. After that limitation is reached, their web server blocks the individual user\'s <a href="/wiki/IP_address" title="IP address">IP address</a> by no longer responding.<sup class="noprint Inline-Template Template-Fact" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Citation_needed" title="Wikipedia:Citation needed"><span title="This claim needs references to reliable sources. (October 2019)">citation needed</span></a></i>&#93;</sup>\n</p>\n<h2><span class="mw-headline" id="See_also">See also</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a
