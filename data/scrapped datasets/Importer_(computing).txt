An importer often is not an entire program by itself, but an extension to another program, implemented as a <a href="/wiki/Plug-in_(computing)" title="Plug-in (computing)">plug-in</a>.  When implemented in this way, the importer reads the data from the file and converts it into the hosting application\'s native format.\n</p><p>For example, the data file for a <a href="/wiki/3D_model" class="mw-redirect" title="3D model">3D model</a> may be written from a modeler, such as <a href="/wiki/3D_Studio_Max" class="mw-redirect" title="3D Studio Max">3D Studio Max</a>.  A <a href="/wiki/Video_game_developer" title="Video game developer">game developer</a> may then want to use that model in their <a href="/wiki/Video_game" title="Video game">game\'s</a>  <a href="/wiki/Level_editor" class="mw-redirect" title="Level editor">editor</a>.  An importer, part of the editor, may read in the 3D Studio Max model and convert it to the game\'s native format so it can be used in <a href="/wiki/Level_(computer_and_video_games)" class="mw-redirect" title="Level (computer and video games)">game levels</a>.\n</p><p>Importers are important <a href="/wiki/Game_development_tool" title="Game development tool">tools</a> in the <a href="/wiki/Video_game_industry" title="Video game industry">video game industry</a>. A plug-in or application that does the converse of an importer is called an <a href="/wiki/Exporter_(computing)" title="Exporter (computing)">exporter</a>.\n</p>\n<h2><span class="mw-headline"
