It uses automated methods to extract both structured and unstructured data from web pages, server logs and link structures. There are three main sub-categories of web mining. <a href="/wiki/Web_content" title="Web content">Web content</a> mining extracts information from within a page. Web structure mining discovers the structure of the <a href="/wiki/Hyperlink" title="Hyperlink">hyperlinks</a> between documents, categorizing sets of web pages and measuring the similarity and relationship between different sites. Web usage mining finds patterns of usage of web pages.\n</p>\n<div id="toc" class="toc" role="navigation" aria-labelledby="mw-toc-heading"><input type="checkbox" role="button" id="toctogglecheckbox" management</li></ul>\n</td></tr></tbody></table>\n<h2><span class="mw-headline" id="Web_usage_mining">Web usage mining</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_mining&amp;action=edit&amp;section=2" title="Edit section: Web usage mining">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Web usage mining is the application of <a href="/wiki/Data_mining" title="Data mining">data mining</a> techniques to discover interesting usage patterns from Web data in order to understand and better serve the needs of <a href="/wiki/Web-based_application" class="mw-redirect" title="Web-based application">Web-based applications</a>.\nUsage data captures the identity or origin of Web users along with their browsing behavior at a Web site.\n</p><p>Web usage mining itself can be classified further depending on the kind of usage data considered:\n</p>\n<ul><li><b>Web server data</b>: The user logs are collected by the <a href="/wiki/Web_server" title="Web server">Web server</a>. Typical data includes IP address, page reference and access time.</li>\n<li><b>Application server data</b>: Commercial application servers have significant features to enable <a href="/wiki/E-commerce" title="E-commerce">e-commerce</a> applications to be built on top of them with little effort. A key feature is the ability to track various kinds of business events and log them in application server logs.</li>\n<li><b>Application level data</b>: New kinds of events can be defined in an application, and logging can be turned on for them thus generating histories of these specially defined events. Many end applications require a combination of one or more of the techniques applied in the categories above.</li></ul>\n<p>Studies related to work<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> are concerned with two areas: constraint-based data mining algorithms applied in Web usage mining and developed software tools (systems). Costa and Seco demonstrated that web log mining can be used to extract <a href="/wiki/Semantic" class="mw-redirect" title="Semantic">semantic</a> information (<a href="/wiki/Hyponymy" class="mw-redirect" title="Hyponymy">hyponymy</a> relationships in particular) about the user and a given community.\n</p>\n<h3><span class="mw-headline" id="Pros">Pros</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_mining&amp;action=edit&amp;section=3" title="Edit section: Pros">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Web usage mining essentially has many advantages which makes this technology attractive to corporations including government agencies. This technology has enabled <a href="/wiki/E-commerce" title="E-commerce">e-commerce</a> to do <a href="/wiki/Personalized_marketing" title="Personalized marketing">personalized marketing</a>, which eventually results in higher trade volumes. Government agencies are using this technology to classify threats and <a href="/wiki/Fight_against_terrorism" class="mw-redirect" title="Fight against terrorism">fight against terrorism</a>. The predicting capability of mining applications can benefit society by identifying criminal activities. Companies can establish better <a href="/wiki/Customer_relationship" class="mw-redirect" title="Customer relationship">customer relationship</a> by understanding the needs of the customer better and reacting to customer needs faster. Companies can find, attract and retain customers; they can save on production costs by utilizing the acquired insight of customer requirements. They can increase <a href="/wiki/Profitability" class="mw-redirect" title="Profitability">profitability</a> by <a href="/wiki/Target_pricing" class="mw-redirect" title="Target pricing">target pricing</a> based on the <a href="/wiki/User_profiling" class="mw-redirect" title="User profiling">profiles</a> created. They can even find customers who might default to a competitor the company will try to retain the customer by providing promotional offers to the specific customer, thus reducing the risk of losing a customer or customers.\n</p><p>More benefits of web usage mining, particularly in the area of <a href="/wiki/Personalization" title="Personalization">personalization</a>, are outlined in specific frameworks such as the <a href="/wiki/Probabilistic_latent_semantic_analysis" title="Probabilistic latent semantic analysis">probabilistic latent semantic analysis</a> model, which offer additional features to the <a href="/wiki/User_behavior_analytics" title="User behavior analytics">user behavior</a> and access pattern.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup> This is because the process provides the user with more relevant content through collaborative recommendation. These models also demonstrate a capability in web usage mining technology to address problems associated with traditional techniques such as biases and questions regarding validity since the data and patterns obtained are not subjective and do not degrade over time.<sup id="cite_ref-3" class="reference"><a href="#cite_note-3">&#91;3&#93;</a></sup> There are also elements unique to web usage mining that can show the technology\'s benefits and these include the way semantic knowledge is applied when interpreting, analyzing, and reasoning about usage patterns during the mining phase.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Cons">Cons</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_mining&amp;action=edit&amp;section=4" title="Edit section: Cons">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Web usage mining by itself does not create issues, but this technology when used on <a href="/wiki/Personal_data" title="Personal data">data of personal nature</a> might cause concerns. The most criticized ethical issue involving web usage mining is the invasion of <a href="/wiki/Internet_privacy" title="Internet privacy">privacy</a>. Privacy is considered lost when information concerning an individual is obtained, used, or disseminated, especially if this occurs without the individual\'s knowledge or consent.<sup id="cite_ref-Lita_van_Wel_and_Lamb\xc3\xa8r_Royakkers_2004_5-0" class="reference"><a href="#cite_note-Lita_van_Wel_and_Lamb\xc3\xa8r_Royakkers_2004-5">&#91;5&#93;</a></sup> The obtained data will be analyzed, made <a href="/wiki/Anonymization" class="mw-redirect" title="Anonymization">anonymous</a>, then <a href="/wiki/Data_clustering" class="mw-redirect" title="Data clustering">clustered</a> to form anonymous profiles.<sup id="cite_ref-Lita_van_Wel_and_Lamb\xc3\xa8r_Royakkers_2004_5-1" class="reference"><a href="#cite_note-Lita_van_Wel_and_Lamb\xc3\xa8r_Royakkers_2004-5">&#91;5&#93;</a></sup> These applications de-<a href="/wiki/Personalization" title="Personalization">individualize</a> users by judging them by their mouse clicks rather than by identifying information. De-individualization in general can be defined as a tendency of judging and treating people on the basis of group characteristics instead of on their own individual characteristics and merits.<sup id="cite_ref-Lita_van_Wel_and_Lamb\xc3\xa8r_Royakkers_2004_5-2" class="reference"><a href="#cite_note-Lita_van_Wel_and_Lamb\xc3\xa8r_Royakkers_2004-5">&#91;5&#93;</a></sup>\n</p><p>Another important concern is that the companies collecting the data for a specific purpose might use the data for totally different purposes, and this essentially violates the user\'s interests.\n</p><p>The growing trend of selling personal data as a commodity encourages website owners to trade personal data obtained from their site. This trend has increased the amount of data being captured and traded increasing the likeliness of one\'s privacy being invaded. The companies which buy the data are obliged make it anonymous and these companies are considered authors of any specific release of mining patterns. They are legally responsible for the contents of the release; any inaccuracies in the release will result in serious lawsuits, but there is no law preventing them from trading the data.\n</p><p>Some mining algorithms might use controversial attributes like sex, race, religion, or sexual orientation to categorize individuals. These practices might be against the anti-discrimination legislation.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> The applications make it hard to identify the use of such controversial attributes, and there is no strong rule against the usage of such algorithms with such attributes. This process could result in denial of service or a privilege to an individual based on his race, religion or sexual orientation. This situation can be avoided by the high ethical standards maintained by the data mining company. The collected data is being made anonymous so that, the obtained data and the obtained patterns cannot be traced back to an individual. It might look as if this poses no threat to one\'s privacy, however additional information can be inferred by the application by combining two separate unscrupulous data from the user.\n</p>\n<h2><span class="mw-headline" id="Web_structure_mining">Web structure mining</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a title="Graph theory">graph theory</a> to analyze the node and connection structure of a web site. According to the type of web structural data, web structure mining can be divided into two kinds:\n</p>\n<ol><li>Extracting patterns from hyperlinks in the web: a <a href="/wiki/Hyperlink" title="Hyperlink">hyperlink</a> is a structural component that connects the web page to a different location.</li>\n<li>Mining the <a href="/wiki/Web_document" class="mw-redirect" title="Web document">document</a> structure: analysis of the tree-like structure of page structures to describe <a href="/wiki/HTML" title="HTML">HTML</a> or <a href="/wiki/XML" title="XML">XML</a> tag usage.</li></ol>\n<p>Web structure mining terminology:\n</p>\n<ul><li>Web graph: <a href="/wiki/Directed_graph" title="Directed graph">directed graph</a> representing web.</li>\n<li>Node: web page in graph.</li>\n<li>Edge: hyperlinks.</li>\n<li>In degree: number of links pointing to particular node.</li>\n<li>Out degree: number of links generated  from particular node.</li></ul>\n<p>An example of a techniques of web structure mining is the <a href="/wiki/PageRank" title="PageRank">PageRank</a> algorithm used by <a href="/wiki/Google" title="Google">Google</a> to rank search results. The rank of a page is decided by the number and quality of links pointing to the target node.\n</p>\n<h2><span class="mw-headline" id="Web_content_mining">Web content mining</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_mining&amp;action=edit&amp;section=6" title="Edit section: Web content mining">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Web content mining is the mining, extraction and integration of useful data, information and knowledge from Web page content. The heterogeneity and the <a href="/wiki/Unstructured_data" title="Unstructured data">lack of structure</a> that permits much of the ever-expanding <a href="/wiki/Source_text" title="Source text">information sources</a> on the World Wide Web, such as <a href="/wiki/Hypertext" title="Hypertext">hypertext</a> documents, makes automated discovery, organization, and search and indexing tools of the Internet and the World Wide Web such as <a href="/wiki/Lycos" title="Lycos">Lycos</a>, <a href="/wiki/Alta_Vista" class="mw-redirect" title="Alta Vista">Alta Vista</a>, <a href="/wiki/WebCrawler" title="WebCrawler">WebCrawler</a>, <a href="/wiki/Aliweb" title="Aliweb">Aliweb</a>, <a href="/wiki/MetaCrawler" title="MetaCrawler">MetaCrawler</a>, and others provide some comfort to users, but they do not generally provide structural information nor categorize, filter, or interpret documents. These factors have prompted researchers to develop more intelligent tools for <a href="/wiki/Information_retrieval" title="Information retrieval">information retrieval</a>, such as intelligent <a href="/wiki/Web_agent" class="mw-redirect" title="Web agent">web agents</a>, as well as to extend database and data mining techniques to provide a higher level of organization for <a href="/wiki/Semi-structured_data" title="Semi-structured data">semi-structured data</a> available on the web. The agent-based approach to web mining involves the development of sophisticated <a href="/wiki/Artificial_intelligence" title="Artificial intelligence">AI systems</a> that can act <a href="/wiki/Autonomous_agent" title="Autonomous agent">autonomously</a> or semi-autonomously on behalf of a particular user, to discover and organize web-based information.\n</p><p>Web content mining is differentiated from two different points of view:<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> Information Retrieval View and Database View.<sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup> summarized the research works done for unstructured data and semi-structured data from information retrieval view. It shows that most of the researches use bag of words, which is based on the statistics about single words in isolation, to represent unstructured text and take single word found in the training corpus as features. For the semi-structured data, all the works utilize the HTML structures inside the documents and some utilized the hyperlink structure between the documents for document representation. As for the database view, in order to have the better <a href="/wiki/Information_management" title="Information management">information management</a> and querying on the web, the mining always tries to infer the structure of the web site to transform a web site to become a database.\n</p><p>There are several ways to represent documents; <a href="/wiki/Vector_space_model" title="Vector space model">vector space model</a> is typically used. The documents constitute the whole vector space.  This representation does not realize the importance of words in a document. To resolve this, <a href="/wiki/Tf-idf" class="mw-redirect" title="Tf-idf">tf-idf</a> (Term Frequency Times Inverse Document Frequency) is introduced.\n</p><p>By multi-scanning the document, we can implement feature selection. Under the condition that the category result is rarely affected, the extraction of feature subset is needed. The general algorithm is to construct an evaluating function to evaluate the features. As feature set, <a href="/wiki/Information_gain" class="mw-redirect" title="Information gain">information gain</a>, <a href="/wiki/Cross_entropy" title="Cross entropy">cross entropy</a>, <a href="/wiki/Mutual_information" title="Mutual information">mutual information</a>, and <a href="/wiki/Odds_ratio" title="Odds ratio">odds ratio</a> are usually used.\nThe classifier and <a href="/wiki/Pattern_analysis" class="mw-redirect" title="Pattern analysis">pattern analysis</a> methods of text data mining are very similar to traditional data mining techniques. The usual evaluative merits are <a href="/w/index.php?title=Classification_accuracy&amp;action=edit&amp;redlink=1" class="new" title="Classification accuracy (page does not exist)">classification accuracy</a>, <a href="/wiki/Precision_and_recall" title="Precision and recall">precision and recall</a> and <a href="/w/index.php?title=Information_score&amp;action=edit&amp;redlink=1" class="new" title="Information score (page does not exist)">information score</a>.\n</p><p>Web mining can complement the retrieval of <a href="/wiki/Structured_data" class="mw-redirect" title="Structured data">structured data</a> transmitted with <a href="/wiki/Open_protocol" class="mw-redirect" title="Open protocol">open protocols</a> like <a href="/wiki/OAI-PMH" class="mw-redirect" title="OAI-PMH">OAI-PMH</a>: an example is the aggregation of works from academic publications,<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup> which are mined to identify <a href="/wiki/Open_access" title="Open access">open access</a> versions through a mix of <a href="/wiki/Open_source" title="Open source">open source</a> and <a href="/wiki/Open_data" title="Open data">open data</a> methods by <a href="/wiki/Academic_database" class="mw-redirect" title="Academic database">academic databases</a> like <a href="/wiki/Unpaywall" class="mw-redirect" title="Unpaywall">Unpaywall</a>.<sup id="cite_ref-10" class="reference"><a href="#cite_note-10">&#91;10&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Web_content_mining_in_foreign_languages">Web content mining in foreign languages</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Web_mining&amp;action=edit&amp;section=7" title="Edit section: Web content mining in foreign languages">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<h4><span class="mw-headline" id="Chinese">Chinese</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a title="Language code">language code</a> of <a href="/wiki/Chinese_writing" class="mw-redirect" title="Chinese writing">Chinese</a> words is very complicated compared to that of English. The <a href="/wiki/GB_18030" title="GB 18030">GB</a>, <a href="/wiki/Big5" title="Big5">Big5</a> and <a href="/wiki/HZ_(character_encoding)" title="HZ (character encoding)">HZ code</a> are common Chinese word codes in web documents.
