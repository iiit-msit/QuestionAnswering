links</span></a></li>\n</ul>\n</div>\n\n<h2><span id="Specifying_the_document.27s_character_encoding"></span><span class="mw-headline" id="Specifying_the_document\'s_character_encoding">Specifying the document\'s character encoding</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Character_encodings_in_HTML&amp;action=edit&amp;section=1" title="Edit section: Specifying the document&#039;s character encoding">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>There are several ways to specify which character encoding is used in the document. First, the <a href="/wiki/Web_server" title="Web server">web server</a> can include the character encoding or "<code>charset</code>" in the <a href="/wiki/Hypertext_Transfer_Protocol" title="Hypertext Transfer Protocol">Hypertext Transfer Protocol</a> (HTTP) <code>Content-Type</code> header, which would typically look like this:<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup>\n</p>\n<pre>Content-Type: text/html; charset=ISO-8859-4\n</pre>\n<p>This method gives the HTTP server a convenient way to alter document\'s encoding according to <a href="/wiki/Content_negotiation" title="Content negotiation">content negotiation</a>; certain HTTP server software can do it, for example Apache with the <a href="/wiki/List_of_Apache_modules" title="List of Apache modules">module</a> <code>mod_charset_lite</code>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup>\n</p><p>For HTML it is possible to include this information inside the <code>head</code> element near the top of the document:<sup id="cite_ref-html5charset_3-0" class="reference"><a href="#cite_note-html5charset-3">&#91;3&#93;</a></sup>\n</p>\n<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">http-equiv</span><span class="o">=</span><span class="s">&quot;Content-Type&quot;</span> <span class="na">content</span><span class="o">=</span><span class="s">&quot;text/html; charset=utf-8&quot;</span><span class="p">&gt;</span>\n</pre></div>\n<p><a href="/wiki/HTML5" title="HTML5">HTML5</a> also allows the following syntax to mean exactly the same:<sup id="cite_ref-html5charset_3-1" class="reference"><a href="#cite_note-html5charset-3">&#91;3&#93;</a></sup>\n</p>\n<div class="mw-highlight mw-highlight-lang-html mw-content-ltr" dir="ltr"><pre><span></span><span class="p">&lt;</span><span class="nt">meta</span> <span class="na">charset</span><span class="o">=</span><span class="s">&quot;utf-8&quot;</span><span class="p">&gt;</span>\n</pre></div>\n<p><a href="/wiki/XHTML" title="XHTML">XHTML</a> documents have a third option: to express the character encoding via <a href="/wiki/XML" title="XML">XML</a> declaration, as follows:<sup id="cite_ref-4" class="reference"><a href="#cite_note-4">&#91;4&#93;</a></sup>\n</p>\n<div class="mw-highlight mw-highlight-lang-xml mw-content-ltr" dir="ltr"><pre><span></span><span class="cp">&lt;?xml version=&quot;1.0&quot; encoding=&quot;ISO-8859-1&quot;?&gt;</span>\n</pre></div>\n<p>As the character encoding cannot be known until this<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (October 2019)">clarification needed</span></a></i>&#93;</sup> declaration is parsed, there can be a problem knowing which encoding is used for the declaration itself. The main principle is that the declaration shall be encoded in pure ASCII, and therefore (if the declaration is inside the file) the encoding needs to be an <a href="/wiki/ASCII_extension" class="mw-redirect" title="ASCII extension">ASCII extension</a>. In order to allow encodings not backwards compatible with ASCII, browsers must be able to parse declarations in such encodings. Examples of such encodings are <a href="/wiki/UTF-16BE" class="mw-redirect" title="UTF-16BE">UTF-16BE</a> and <a href="/wiki/UTF-16LE" class="mw-redirect" title="UTF-16LE">UTF-16LE</a>.\n</p><p>As of HTML5 the recommended charset is <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a>.<sup id="cite_ref-html5charset_3-2" class="reference"><a href="#cite_note-html5charset-3">&#91;3&#93;</a></sup> An "encoding sniffing algorithm" is defined in the specification to determine the character encoding of the document based on multiple sources of input, including:\n</p>\n<ol><li>Explicit user instruction</li>\n<li>An explicit meta tag within the first 1024 bytes of the document</li>\n<li>A <a href="/wiki/Byte_order_mark" title="Byte order mark">Byte order mark</a> within the first three bytes of the document</li>\n<li>The HTTP Content-Type or other transport layer information</li>\n<li>Analysis of the document bytes looking for specific sequences or ranges of byte values,<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup> and other tentative detection mechanisms.</li></ol>\n<p>For ASCII-compatible character encodings the consequence of choosing incorrectly is that characters outside the printable ASCII range (32 to 126) usually appear incorrectly. This presents few problems for <a href="/wiki/English_language" title="English language">English</a>-speaking users, but other languages regularly\xe2\x80\x94in some cases, always\xe2\x80\x94require characters outside that range. In <a href="/wiki/CJK" class="mw-redirect" title="CJK">CJK</a> environments where there are several different multi-byte encodings in use, auto-detection is also often employed. Finally, browsers usually permit the user to override <i>incorrect</i> charset label manually as well.\n</p><p>It is increasingly common for multilingual websites and websites in non-Western languages to use <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a>, which allows use of the same encoding for all languages. <a href="/wiki/UTF-16" title="UTF-16">UTF-16</a> or <a href="/wiki/UTF-32" title="UTF-32">UTF-32</a>, which can be used for all languages as well, are less widely used because they can be harder to handle in programming languages that assume a <a href="/wiki/Byte-oriented" class="mw-redirect" title="Byte-oriented">byte-oriented</a> ASCII superset encoding, and they are less efficient for text with a high frequency of ASCII characters, which is usually the case for HTML documents.\n</p><p>Successful viewing of a page is not necessarily an indication that its encoding is specified correctly. If the page\'s creator and reader are both assuming some platform-specific character encoding, and the server does not send any identifying information, then the reader will nonetheless see the page as the creator intended, but other readers on different platforms or with different native languages will not see the page as intended.\n</p>\n<h2><span class="mw-headline" id="Permitted_encodings">Permitted encodings</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Character_encodings_in_HTML&amp;action=edit&amp;section=2" title="Edit section: Permitted encodings">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>The <a href="/wiki/WHATWG" title="WHATWG">WHATWG</a> Encoding Standard, referenced by recent HTML standards (the current WHATWG HTML Living Standard, as well as the formerly competing <a href="/wiki/W3C" class="mw-redirect" title="W3C">W3C</a> HTML 5.0 and 5.1) specifies a list of encodings which browsers must support. The HTML standards forbid support of other encodings.<sup id="cite_ref-html51_6-0" class="reference"><a href="#cite_note-html51-6">&#91;6&#93;</a></sup><sup id="cite_ref-html50_7-0" class="reference"><a href="#cite_note-html50-7">&#91;7&#93;</a></sup><sup id="cite_ref-html5living_8-0" class="reference"><a href="#cite_note-html5living-8">&#91;8&#93;</a></sup> The Encoding Standard further stipulates that new formats, new protocols (even when existing formats are used) and authors of new documents are required to use <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> exclusively.<sup id="cite_ref-namesandlabels_9-0" class="reference"><a href="#cite_note-namesandlabels-9">&#91;9&#93;</a></sup>\n</p><p>Besides UTF-8, the following encodings are explicitly listed in the HTML standard id="Character_references">Character references</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Character_encodings_in_HTML&amp;action=edit&amp;section=3" title="Edit section: Character references">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<div role="note" class="hatnote navigation-not-searchable">Main articles: <a href="/wiki/Character_entity_reference" class="mw-redirect" title="Character entity reference">Character entity reference</a> and <a href="/wiki/Numeric_character_reference" title="Numeric character reference">Numeric character reference</a></div>\n<p>In addition to native character encodings, characters can also be encoded as <i>character references</i>, which can be <i>numeric character references</i> (<a href="/wiki/Decimal" title="Decimal">decimal</a> or <a href="/wiki/Hexadecimal" title="Hexadecimal">hexadecimal</a>) or <i>character entity references</i>. Character entity references are also sometimes referred to as <i>named entities</i>, or <i>HTML entities</i> for HTML. HTML\'s usage of character references derives from <a href="/wiki/SGML" class="mw-redirect" title="SGML">SGML</a>.\n</p>\n<h3><span class="mw-headline" id="HTML_character_references">HTML character references</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Character_encodings_in_HTML&amp;action=edit&amp;section=4" title="Edit section: HTML character references">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>A <i>numeric character reference</i> in HTML refers to a character by its <a href="/wiki/Universal_Character_Set" class="mw-redirect" title="Universal Character Set">Universal Character Set</a>/<a href="/wiki/Unicode" title="Unicode">Unicode</a> <i>code point</i>, and uses the format\n</p>\n<dl><dd><code>&amp;#<i>nnnn</i>;</code></dd></dl>\n<p>or\n</p>\n<dl><dd><code>&amp;#x<i>hhhh</i>;</code></dd></dl>\n<p>where <i>nnnn</i> is the code point in <a href="/wiki/Decimal" title="Decimal">decimal</a> form, and <i>hhhh</i> is the code point in <a href="/wiki/Hexadecimal" title="Hexadecimal">hexadecimal</a> form. The <i>x</i> must be lowercase in XML documents. The <i>nnnn</i> or <i>hhhh</i> may be any number of digits and may include leading zeros. The <i>hhhh</i> may mix uppercase and lowercase, though uppercase is the usual style.\n</p><p>Not all <a href="/wiki/Web_browser" title="Web browser">web browsers</a> or <a href="/wiki/Email_client" title="Email client">email clients</a> used by receivers of HTML documents, or <a href="/wiki/Text_editor" title="Text editor">text editors</a> used by authors of HTML documents, will be able to render all HTML characters. Most modern software is able to display most or all of the characters for the user\'s language, and will draw a box or other clear indicator for characters they cannot render.\n</p><p>For codes from 0 to 127, the original 7-bit <a href="/wiki/ASCII" title="ASCII">ASCII</a> standard set, most of these characters can be used without a character reference. Codes from 160 to 255 can all be created using <a href="/wiki/List_of_XML_and_HTML_character_entity_references" title="List of XML and HTML character entity references">character entity names</a>. Only a few higher-numbered codes can be created using entity names, but all can be created by decimal number character reference.\n</p><p>Character entity references can also have the format <code>&amp;<i>name</i>;</code> where <i>name</i> is a case-sensitive alphanumeric string. For example, "\xce\xbb" can also be encoded as <code>&amp;lambda;</code> in an HTML document. The character entity references <code>&amp;lt;</code>, <code>&amp;gt;</code>, <code>&amp;quot;</code> and <code>&amp;amp;</code> are predefined in HTML and SGML, because <code>&lt;</code>, <code>&gt;</code>, <code>"</code>  and <code>&amp;</code> are already used to delimit markup. This notably did not include XML\'s <code>&amp;apos;</code> (\') entity prior to <a href="/wiki/HTML5" title="HTML5">HTML5</a>. For a list of all named HTML character entity references along with the versions in which they were introduced, see <a href="/wiki/List_of_XML_and_HTML_character_entity_references" title="List of XML and HTML character entity references">List of XML and HTML character entity references</a>.\n</p><p>Unnecessary use of HTML character references may significantly reduce HTML readability. If the character encoding for a web page is chosen appropriately, then HTML character references are usually only required for markup delimiting characters as mentioned above, and for a few special characters (or none at all if a native <a href="/wiki/Unicode" title="Unicode">Unicode</a> encoding like <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> is used). Incorrect HTML entity escaping may also open up security vulnerabilities for injection attacks such as <a href="/wiki/Cross-site_scripting" title="Cross-site scripting">cross-site scripting</a>. If HTML attributes are left unquoted, certain characters, most importantly <a href="/wiki/Whitespace_character" title="Whitespace character">whitespace</a>, such as space and tab, must be escaped using entities. Other languages related to HTML have their own methods of escaping characters.\n</p>\n<h3><span class="mw-headline" id="XML_character_references">XML character references</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Character_encodings_in_HTML&amp;action=edit&amp;section=5" title="Edit section: XML character references">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Unlike traditional HTML with its large range of character entity references, in <a href="/wiki/XML" title="XML">XML</a> there are only five predefined character entity references. These are used to escape characters that are markup sensitive in certain contexts:<sup id="cite_ref-45" class="reference"><a href="#cite_note-45">&#91;30&#93;</a></sup>\n</p>\n<ul><li><code>&amp;amp;</code>  \xe2\x86\x92 &amp; (<a href="/wiki/Ampersand" title="Ampersand">ampersand</a>, U+0026)</li>\n<li><code>&amp;lt;</code>   \xe2\x86\x92 &lt; (less-than sign, U+003C)</li>\n<li><code>&amp;gt;</code>   \xe2\x86\x92 &gt; (greater-than sign, U+003E)</li>\n<li><code>&amp;quot;</code> \xe2\x86\x92 " (quotation mark, U+0022)</li>\n<li><code>&amp;apos;</code> \xe2\x86\x92 \' (apostrophe, U+0027)</li></ul>\n<p>All other character entity references have to be defined before they can be used. For example, use of <code>&amp;eacute;</code> (which gives \xc3\xa9, Latin lower-case E with acute accent, U+00E9 in Unicode) in an XML document will generate an error unless the entity has already been defined. XML also requires that the <code>x</code> in hexadecimal numeric references be in lowercase: for example <code>&amp;#xA1b</code> rather than <code>&amp;#XA1b</code>. <a href="/wiki/XHTML" title="XHTML">XHTML</a>, which is an XML application, supports the HTML entity set, along with XML\'s predefined entities.\n</p>\n<h2><span class="mw-headline" id="See_also">See also</span><span
