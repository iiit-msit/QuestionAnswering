href="/w/index.php?title=Media_type&amp;action=edit&amp;section=1" title="Edit section: Naming">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>A media type consists of a <i>type</i> and a <i>subtype</i>, which is further structured into a <i>tree</i>. A media type can optionally define a <i>suffix</i> and <i>parameters</i>:\n</p>\n<dl><dd><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">type &quot;/&quot; [tree &quot;.&quot;] subtype [&quot;+&quot; suffix]* [&quot;;&quot; parameter]</code></dd></dl>\n<p>As of November 1996, the registered types were: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">application</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">audio</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">image</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">message</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">multipart</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">text</code> and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">video</code>.<sup id="cite_ref-&#82;FC2045_2-1" class="reference"><a href="#cite_note-RFC2045-2">&#91;2&#93;</a></sup> By December 2020, the registered types included the foregoing, plus <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">font</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">example</code>, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">model</code>.<sup id="cite_ref-iana-2020-12-09_5-0" class="reference"><a href="#cite_note-iana-2020-12-09-5">&#91;5&#93;</a></sup>\n</p><p>An unofficial top-level name in common use is <a href="/wiki/Chemical_file_format#The_Chemical_MIME_Project" title="Chemical file format"><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">chemical</code></a>.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup><sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup><sup id="cite_ref-8" class="reference"><a href="#cite_note-8">&#91;8&#93;</a></sup>\n</p><p>As an example, an HTML file might be designated <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">text/html; charset=UTF-8</code>. In this example, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">text</code> is the type, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">html</code> is the subtype, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">charset=UTF-8</code> is an optional parameter indicating the character encoding.\n</p><p>A subtype typically consists of a media format, but it may or must also contain other content, such as a tree prefix, producer, product or suffix, according to the different rules in registration trees.\n</p><p>Types, subtypes, and parameter names are case-insensitive. Parameter values are usually case-sensitive, but may be interpreted in a case-insensitive fashion depending on the intended use.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup>\n</p>\n<h3><span id="Common_examples_.5B10.5D"></span><span class="mw-headline" id="Common_examples_[10]">Common examples <sup id="cite_ref-10" dir="ltr">text/xml</code></li></ul>\n<h3><span class="mw-headline" id="Registration_trees">Registration trees</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=3" title="Edit section: Registration trees">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>All media types should be registered using the IANA registration procedures. For the efficiency and flexibility of the media type registration process, different structures of subtypes can be registered in registration trees that are distinguished by the use of tree prefixes. Currently the following trees are created: standard (no prefix), vendor (<code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">vnd.</code> prefix), personal or vanity (<code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">prs.</code> prefix), unregistered (<code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">x.</code> prefix). These registration trees were first defined in November 1996 (obsoleted RFC 2048 - currently RFC 6838). New registration trees may be created by <a href="/wiki/Internet_Engineering_Task_Force" title="Internet Engineering Task Force">IETF</a> Standards Action for external registration and management by well-known permanent organizations (e.g. scientific societies).\n</p>\n<h4><span class="mw-headline" id="Standards_tree">Standards tree</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=4" title="Edit section: Standards tree">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>The standards tree does not use any tree prefix:<sup id="cite_ref-&#82;FC6838_12-0" class="reference"><a href="#cite_note-RFC6838-12">&#91;12&#93;</a></sup>\n</p>\n<dl><dd><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">type &quot;/&quot; subtype [&quot;+&quot; suffix] *[&quot;;&quot; parameter]</code></dd></dl>\n<p>Examples: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">application/javascript</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">image/png</code>.\n</p><p>Registrations in the standards tree must be either associated with IETF specifications approved directly by the IESG, or registered by an IANA recognized standards-related organization.\n</p>\n<h4><span class="mw-headline" id="Vendor_tree">Vendor tree</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=5" title="Edit section: Vendor tree">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>The vendor tree includes media types associated with publicly available products. It uses the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">vnd.</code> tree prefix:<sup id="cite_ref-&#82;FC6838_12-1" class="reference"><a href="#cite_note-RFC6838-12">&#91;12&#93;</a></sup>\n</p>\n<dl><dd><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">type &quot;/&quot; &quot;vnd.&quot; subtype [&quot;+&quot; suffix] *[&quot;;&quot; parameter]</code></dd></dl>\n<p>Examples: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">application/vnd.ms-excel</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">application/vnd.oasis.opendocument.text</code>.\n</p><p>The terms "vendor" and "producer" are considered equivalent in the context. Industry consortia as well as non-commercial entities can register media types in the vendor tree. A registration in the vendor tree may be created by anyone who needs to interchange files associated with some software product or set of products. However, the registration belongs to the vendor or organization producing the software that employs the type being registered, and that vendor or organization can at any time elect to assert ownership of a registration done by a third party.\n</p>\n<h4><span class="mw-headline" id="Personal_or_vanity_tree">Personal or vanity tree</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=6" title="Edit section: Personal or vanity tree">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>The personal or vanity tree includes media types associated with non publicly available products or experimental media types. It uses the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">prs.</code> tree prefix:\n</p>\n<dl><dd><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">type &quot;/&quot; &quot;prs.&quot; subtype [&quot;+&quot; suffix] *[&quot;;&quot; parameter]</code></dd></dl>\n<p>Examples: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">audio/prs.sid</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">image/prs.btif</code>.\n</p>\n<h4><span class="mw-headline" id="Unregistered_tree">Unregistered tree</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=7" title="Edit section: Unregistered tree">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>The unregistered tree includes media types intended exclusively for use in private environments and only with the active agreement of the parties exchanging them. It uses the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">x.</code> tree prefix:\n</p>\n<dl><dd><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">type &quot;/&quot; &quot;x.&quot; subtype [&quot;+&quot; suffix] *[&quot;;&quot; parameter]</code></dd></dl>\n<p>Examples: <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">application/x.foo</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">video/x.bar</code>.\n</p><p>Media types in this tree cannot be registered. According to RFC 6838 (published in January 2013), any use of types in the unregistered tree is strongly discouraged. In addition, subtypes prefixed with <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">x-</code> or <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">X-</code> are no longer considered to be members of this tree.\n</p><p>According to obsoleted RFC 2048 (published in November 1996)\xe2\x80\x94it should rarely, if ever, be necessary to use unregistered types, and as such use of any <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">x.</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">x-</code> or <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">X-</code> prefixes is discouraged. Obsoleted RFC 1590 (published in September 1993) stated that the use of the <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">x-</code> or <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">X-</code> prefix may be used for unregistered subtypes.\n</p><p>Media types that have been widely deployed (with a subtype prefixed with <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">x-</code> or <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">X-</code>) without being registered, should be, if possible, re-registered with a proper prefixed subtype. If this is not possible, the media type can, after an approval by both the media types reviewer and the IESG, be registered in the standards tree with its unprefixed subtype.\n</p>\n<h3><span class="mw-headline" id="Suffix">Suffix</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=8" title="Edit section: Suffix">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Suffix is an augmentation to the media type definition to additionally specify the underlying structure of that media type, allowing for generic processing based on that structure and independent of the exact type\'s particular semantics. Media types that make use of a named structured syntax should use the appropriate IANA registered <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">&quot;+&quot;suffix</code> for that structured syntax when they are registered. Unregistered suffixes should not be used (since January 2013). Structured syntax suffix registration procedures are defined in RFC 6838.<sup id="cite_ref-&#82;FC6838_12-2" class="reference"><a href="#cite_note-RFC6838-12">&#91;12&#93;</a></sup>\n</p><p>The <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+xml</code> suffix has been defined since January 2001 (RFC 3023<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup>), and was formally included in the initial contents of the Structured Syntax Suffix Registry along with <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+json</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+ber</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+der</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+fastinfoset</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+wbxml</code>, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+zip</code> in January 2013 (RFC 6839). Subsequent additions include <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+gzip</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+cbor</code>, <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+json-seq</code>, and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">+cbor-seq</code>.<sup id="cite_ref-14" class="reference"><a href="#cite_note-14">&#91;14&#93;</a></sup>\n</p>\n<h2><span class="mw-headline" id="Mailcap">Mailcap</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=9" title="Edit section: Mailcap">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Mailcap (derived from the phrase "mail capability") is a type of meta file used to configure how MIME-aware applications such as mail clients and web browsers render files of different MIME-types. The mailcap format is defined by RFC 1524 "A User Agent Configuration Mechanism for Multimedia Mail Format Information" but is not defined as an Internet standard. It is supported by most Unix systems.\n</p><p>Lines can be comments starting with the # character, or a mime-type followed by how to handle that mime type.\n</p>\n<h2><span class="mw-headline" id="Mime.types">Mime.types</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=10" title="Edit section: Mime.types">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>An associated file is the <b>mime.types</b> file, which associates filename extensions with a <i>MIME type</i>. If the MIME type is properly set, this is unnecessary, but MIME types may be incorrectly set, or set to a generic type such as <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">application/octet-stream</code>, and <b>mime.types</b> allows one to fall back on the extension in these cases. Similarly, since many file systems do not store MIME type information, but instead rely on the filename extension, a mime.types file is frequently used by web servers to determine MIME type.\n</p><p>When <i>viewing</i> a file, these two work together as follows: <b><code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">mime.types</code></b> associates an extension with a MIME type, while <code><a href="/wiki/Mailcap" class="mw-redirect" title="Mailcap">mailcap</a></code> associates a MIME type with a program.\n</p><p>In UNIX-type systems, the mime.types file is usually located at <code>/etc/<b>mime.types</b></code> and/or <code>$HOME/<b>.mime.types</b></code> and the format is simply that each line is a space-delimited list of a MIME type, followed by zero or more extensions. For example, the HTML type can be associated with the extensions <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">.htm</code> and <code class="mw-highlight mw-highlight-lang-text mw-content-ltr" id="" style="" dir="ltr">.html</code> by the following line:\n</p>\n<pre>text/html     htm   html\n</pre>\n<h3><span class="mw-headline" id="Netscape_use">Netscape use</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=Media_type&amp;action=edit&amp;section=11" title="Edit section: Netscape use">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The mime.types file dates to <a href="/wiki/Netscape" title="Netscape">Netscape</a>, where it used a different format;<sup id="cite_ref-15" class="reference"><a href="#cite_note-15">&#91;15&#93;</a></sup> it used <a href="/wiki/Attribute%E2%80%93value_pair" title="Attribute\xe2\x80\x93value pair">key\xe2\x80\x93value pairs</a> and a comma-separated list of extensions, together with a <a href="/wiki/File_format#Magic_number" title="File format">standard header</a> consisting of a specific comment that identifies the file as a mime.types file, as follows.\n</p>\n<pre>#--Netscape Communications Corporation MIME Information\n# Do not delete the above line. It is used to identify the file type.\n\ntype=text/html exts=htm,html\n</pre>\n<table class="box-Missing_information plainlinks metadata ambox ambox-content" role="presentation"><tbody><tr><td class="mbox-image"><div style="width:52px"><a href="/wiki/File:Wiki_letter_w.svg" class="image"><img alt="Wiki letter w.svg" src="//upload.wikimedia.org/wikipedia/en/thumb/6/6c/Wiki_letter_w.svg/44px-Wiki_letter_w.svg.png" decoding="async" width="44" height="44" srcset="//upload.wikimedia.org/wikipedia/en/thumb/6/6c/Wiki_letter_w.svg/66px-Wiki_letter_w.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/6/6c/Wiki_letter_w.svg/88px-Wiki_letter_w.svg.png 2x" data-file-width="44" data-file-height="44" /></a></div></td><td class="mbox-text"><div class="mbox-text-span">This article <b>is missing information</b> about .<span class="hide-when-compact"> Please expand the article to include this information.
