title="Comparison of layout engines (XHTML 1.1)">1.1</a></li></ul></li></ul></td>\n</tr></tbody></table>\n<p><b>Extensible HyperText Markup Language</b> (<b>XHTML</b>) is part of the family of <a href="/wiki/XML" title="XML">XML</a> <a href="/wiki/Markup_language" title="Markup language">markup languages</a>. It mirrors or extends versions of the widely used <a href="/wiki/HyperText_Markup_Language" class="mw-redirect" title="HyperText Markup Language">HyperText Markup Language</a> (HTML), the language in which <a href="/wiki/Web_page" title="Web page">Web pages</a> are formulated.\n</p><p>While HTML, prior to <a href="/wiki/HTML5" title="HTML5">HTML5</a>, was defined as an application of <a href="/wiki/Standard_Generalized_Markup_Language" title="Standard Generalized Markup Language">Standard Generalized Markup Language</a> (SGML), a flexible markup language framework, XHTML is an application of <a href="/wiki/XML" title="XML">XML</a>, a more restrictive subset of SGML. XHTML documents are <a href="/wiki/Well-formed_document" title="Well-formed document">well-formed</a> and may therefore be parsed using standard XML parsers, unlike HTML, which requires a lenient HTML-specific parser.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup>\n</p><p>XHTML 1.0 became a <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium (W3C)</a> <a href="/wiki/W3C_recommendation" class="mw-redirect" title="W3C recommendation">recommendation</a> on January 26, 2000. XHTML 1.1 became a W3C recommendation on May 31, 2001. The standard known as XHTML5 is being developed as an XML adaptation of the HTML5 specification.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2">&#91;2&#93;</a></sup><sup id="cite_ref-3" Overview">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>XHTML 1.0 is "a reformulation of the three HTML 4 document types as applications of XML 1.0".<sup id="cite_ref-W3cWhatIsXhtml_4-0" class="reference"><a href="#cite_note-W3cWhatIsXhtml-4">&#91;4&#93;</a></sup> The <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a> (W3C) also continues to maintain the HTML 4.01 Recommendation, and the specifications for <a href="/wiki/HTML5" title="HTML5">HTML5</a> and XHTML5 are being actively developed. In the current XHTML 1.0 Recommendation document, as published and revised to August 2002, the W3C commented that, "The XHTML family is the next step in the evolution of the Internet. By migrating to XHTML today, content developers can enter the XML world with all of its attendant benefits, while still remaining confident in their content\'s backward and future compatibility."<sup id="cite_ref-W3cWhatIsXhtml_4-1" class="reference"><a href="#cite_note-W3cWhatIsXhtml-4">&#91;4&#93;</a></sup>\n</p><p>However, in 2005, the <a href="/wiki/Web_Hypertext_Application_Technology_Working_Group" class="mw-redirect" title="Web Hypertext Application Technology Working Group">Web Hypertext Application Technology Working Group</a> (WHATWG) formed, independently of the W3C, to work on advancing ordinary HTML not based on XHTML. The WHATWG eventually began working on a standard that supported both XML and non-XML <a href="/wiki/Serialization" title="Serialization">serializations</a>, <a href="/wiki/HTML5" title="HTML5">HTML5</a>, in parallel to W3C standards such as XHTML 2. In 2007, the W3C\'s HTML working group voted to officially recognize HTML5 and work on it as the next-generation HTML standard.<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup>  In 2009, the W3C allowed the XHTML 2 Working Group\'s charter to expire, acknowledging that HTML5 would be the sole next-generation HTML standard, including both XML and non-XML serializations.<sup id="cite_ref-XHTML_FAQ_6-0" class="reference"><a href="#cite_note-XHTML_FAQ-6">&#91;6&#93;</a></sup>  Of the two serializations, the W3C suggests that most authors use the HTML syntax, rather than the XHTML syntax.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Motivation">Motivation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=2" title="Edit section: Motivation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>XHTML was developed to make HTML more <a href="/wiki/Extensible" class="mw-redirect" title="Extensible">extensible</a> and increase <a href="/wiki/Interoperability" title="Interoperability">interoperability</a> with other data formats.<sup id="cite_ref-xhtml-1.0-w3c-rec_8-0" class="reference"><a href="#cite_note-xhtml-1.0-w3c-rec-8">&#91;8&#93;</a></sup> In addition, browsers were forgiving of errors in HTML, and most websites were displayed despite technical errors in the markup; XHTML introduced stricter error handling.<sup id="cite_ref-9" class="reference"><a href="#cite_note-9">&#91;9&#93;</a></sup> HTML 4 was ostensibly an application of <a href="/wiki/Standard_Generalized_Markup_Language" title="Standard Generalized Markup Language">Standard Generalized Markup Language</a> (SGML); however the specification for SGML was complex, and neither web browsers nor the HTML 4 Recommendation were fully conformant to it.<sup id="cite_ref-normative-ref-to-sgml_10-0" class="reference"><a href="#cite_note-normative-ref-to-sgml-10">&#91;10&#93;</a></sup> The XML standard, approved in 1998, provided a simpler data format closer in simplicity to HTML 4.<sup id="cite_ref-xhtml-myths-reality_11-0" class="reference"><a href="#cite_note-xhtml-myths-reality-11">&#91;11&#93;</a></sup> By shifting to an XML format, it was hoped HTML would become compatible with common XML tools;<sup id="cite_ref-xml-tools_12-0" class="reference"><a href="#cite_note-xml-tools-12">&#91;12&#93;</a></sup> servers and proxies would be able to transform content, as necessary, for constrained devices such as mobile phones.<sup id="cite_ref-wireless-content-using-xhtml_13-0" class="reference"><a href="#cite_note-wireless-content-using-xhtml-13">&#91;13&#93;</a></sup>\nBy using <a href="/wiki/XML_namespace" title="XML namespace">namespaces</a>, XHTML documents could provide extensibility by including fragments from other XML-based languages such as <a href="/wiki/Scalable_Vector_Graphics" title="Scalable Vector Graphics">Scalable Vector Graphics</a> and <a href="/wiki/MathML" title="MathML">MathML</a>.<sup id="cite_ref-namespaces-course_14-0" class="reference"><a href="#cite_note-namespaces-course-14">&#91;14&#93;</a></sup> Finally, the renewed work would provide an opportunity to divide HTML into reusable components (<a href="/wiki/XHTML_Modularization" title="XHTML Modularization">XHTML Modularization</a>) and clean up untidy parts of the language.<sup id="cite_ref-xhtml-faq_15-0" class="reference"><a href="#cite_note-xhtml-faq-15">&#91;15&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Relationship_to_HTML"><span id="Relationship_to_HTML">Relationship to HTML</span></span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=3" title="Edit section: Relationship to HTML">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>There are various differences between XHTML and HTML. The <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a> (DOM) is a tree structure that represents the page internally in applications, and XHTML and HTML are two different ways of representing that in markup. Both are less expressive than the DOM \xe2\x80\x93 for example, "--" may be placed in comments in the DOM, but cannot be represented in a comment in either XHTML or HTML \xe2\x80\x93 and generally XHTML\'s XML syntax is more expressive than HTML (for example, arbitrary namespaces are not allowed in HTML). XHTML uses an XML syntax, while HTML uses a pseudo-<a href="/wiki/SGML" class="mw-redirect" title="SGML">SGML</a> syntax (officially SGML for HTML&#160;4 and under, but never in practice, and standardised away from SGML in HTML5). Because the expressible contents of the DOM in syntax are slightly different, there are some changes in actual behavior between the two models. Syntax differences, however, can be overcome by implementing alternate translational framework within the markup.\n</p><p>First, there are some differences in syntax:<sup id="cite_ref-16" class="reference"><a href="#cite_note-16">&#91;16&#93;</a></sup>\n</p>\n<ul><li>Broadly, the XML rules require that all <a href="/wiki/HTML_element" title="HTML element">elements</a> be closed, either by a separate closing tag or using self-closing syntax (e.g. <code>&lt;br/&gt;</code>), while HTML syntax permits some elements to be unclosed because either they are always empty (e.g. <code>&lt;input&gt;</code>) or their end can be determined implicitly ("omissibility", e.g. <code>&lt;p&gt;</code>).</li>\n<li>XML is case-sensitive for element and <a href="/wiki/HTML_attribute" title="HTML attribute">attribute</a> names, while HTML is not.</li>\n<li>Some shorthand features in HTML are omitted in XML, such as (1) <i>attribute minimization</i>, where attribute values or their quotes may be omitted (e.g. <code>&lt;option selected&gt;</code> or <code>&lt;option selected=selected&gt;</code>, while in XML this must be expressed as <code>&lt;option selected="selected"&gt;</code>); (2) <i>element minimization</i> may be used to remove elements entirely (such as <code>&lt;tbody&gt;</code> inferred in a table if not given); and (3) the rarely used SGML syntax for element minimization ("shorttag"), which most browsers do not implement.<sup id="cite_ref-17" class="reference"><a href="#cite_note-17">&#91;17&#93;</a></sup></li>\n<li>There are numerous other technical requirements surrounding namespaces and precise parsing of whitespace and certain characters and elements. The exact parsing of HTML in practice has been undefined until recently; see the HTML5 specification (<a rel="nofollow" class="external text" href="https://www.whatwg.org/specs/web-apps/current-work/multipage/">[HTML5]</a>) for full details, or the working summary (<a rel="nofollow" class="external text" href="https://wiki.whatwg.org/wiki/HTML_vs._XHTML">HTML vs. XHTML</a>).</li></ul>\n<p>In addition to the syntactical differences, there are some behavioral differences, mostly arising from the underlying differences in serialization. For example:\n</p>\n<ul><li>Behavior on parse errors differ. A fatal parse error in XML (such as an incorrect tag structure) causes document processing to be aborted.</li>\n<li>Most content requiring namespaces will not work in HTML, except the built-in support for SVG and MathML in the HTML5 parser along with certain magic prefixes such as <code>xlink</code>.</li>\n<li>JavaScript processing is different in XHTML, with minor changes in case sensitivity to some functions, and further precautions to restrict processing to well-formed content. Scripts must not use the <code class="mw-highlight mw-highlight-lang-javascript mw-content-ltr" id="" style="" dir="ltr"><span class="nb">document</span><span class="p">.</span><span class="nx">write</span><span class="p">()</span></code> method; it is not available for XHTML. The <code>innerHTML</code> property is available, but will not insert non-well-formed content. On the other hand, it can be used to insert well-formed namespaced content into XHTML.</li>\n<li><a href="/wiki/Cascading_Style_Sheets" class="mw-redirect" title="Cascading Style Sheets">Cascading Style Sheets</a> (CSS) are also applied differently. Due to XHTML\'s case-sensitivity, all CSS selectors become case-sensitive for XHTML documents.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup> Some CSS properties, such as backgrounds, set on the <code>&lt;body&gt;</code> element in HTML are \'inherited upwards\' into the <code>&lt;html&gt;</code> element; this appears<sup class="noprint Inline-Template" style="margin-left:0.1em; white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Please_clarify" title="Wikipedia:Please clarify"><span title="The text near this tag may need clarification or removal of jargon. (December 2018)">clarification needed</span></a></i>&#93;</sup> not to be the case for XHTML.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup></li></ul>\n<h3><span class="mw-headline" id="Adoption">Adoption</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=4" title="Edit section: Adoption">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The similarities between HTML 4.01 and XHTML 1.0 led many web sites and <a href="/wiki/Content_management_system" title="Content management system">content management systems</a> to adopt the initial W3C XHTML 1.0 Recommendation. To aid authors in the transition, the W3C provided guidance on how to publish XHTML 1.0 documents in an HTML-compatible manner, and serve them to browsers that were not designed for XHTML.<sup id="cite_ref-appendix_c_20-0" class="reference"><a href="#cite_note-appendix_c-20">&#91;20&#93;</a></sup><sup id="cite_ref-media_types_21-0" class="reference"><a href="#cite_note-media_types-21">&#91;21&#93;</a></sup>\n</p><p>Such "HTML-compatible" content is sent using the HTML media type (<code>text/html</code>) rather than the official Internet media type for XHTML (<code>application/xhtml+xml</code>). When measuring the adoption of XHTML to that of regular HTML, therefore, it is important to distinguish whether it is media type usage or actual document contents that is being compared.<sup id="cite_ref-22" class="reference"><a href="#cite_note-22">&#91;22&#93;</a></sup><sup id="cite_ref-23" class="reference"><a href="#cite_note-23">&#91;23&#93;</a></sup>\n</p><p>Most web browsers have mature support<sup id="cite_ref-mature_support_24-0" class="reference"><a href="#cite_note-mature_support-24">&#91;24&#93;</a></sup> for all of the possible XHTML media types.<sup id="cite_ref-25" class="reference"><a href="#cite_note-25">&#91;25&#93;</a></sup> The notable exception is <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> versions 8 and earlier by <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a>; rather than rendering <code>application/xhtml+xml</code> content, a dialog box invites the user to save the content to disk instead. Both Internet Explorer 7 (released in 2006) and Internet Explorer 8 (released in March 2009) exhibit this behavior.<sup id="cite_ref-wilson-XML_26-0" class="reference"><a href="#cite_note-wilson-XML-26">&#91;26&#93;</a></sup> Microsoft developer <a href="/w/index.php?title=Chris_Wilson_(open_web_advocate)&amp;action=edit&amp;redlink=1" class="new" title="Chris Wilson (open web advocate) (page does not exist)">Chris Wilson</a> explained in 2005 that IE7\xe2\x80\x99s priorities were improved <a href="/wiki/Browser_security" title="Browser security">browser security</a> and <a href="/wiki/Cascading_Style_Sheet" class="mw-redirect" title="Cascading Style Sheet">CSS</a> support, and that proper XHTML support would be difficult to graft onto IE\xe2\x80\x99s compatibility-oriented HTML parser;<sup id="cite_ref-wilson_27-0" class="reference"><a href="#cite_note-wilson-27">&#91;27&#93;</a></sup> however,  <a href="/wiki/Microsoft" title="Microsoft">Microsoft</a> added support for true XHTML in <a href="/wiki/Internet_Explorer_9" title="Internet Explorer 9">IE9</a>.<sup id="cite_ref-28" class="reference"><a href="#cite_note-28">&#91;28&#93;</a></sup>\n</p><p>As long as support is not widespread, most web developers avoid using XHTML that is not HTML-compatible,<sup id="cite_ref-the_x-philes_29-0" class="reference"><a href="#cite_note-the_x-philes-29">&#91;29&#93;</a></sup> so advantages of XML such as namespaces, faster parsing and smaller-footprint browsers do not benefit the user.<sup id="cite_ref-30" class="reference"><a href="#cite_note-30">&#91;30&#93;</a></sup><sup id="cite_ref-31" class="reference"><a href="#cite_note-31">&#91;31&#93;</a></sup><sup id="cite_ref-32" class="reference"><a href="#cite_note-32">&#91;32&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Criticism">Criticism</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=5" title="Edit section: Criticism">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>In the early 2000s, some Web developers began to question why Web authors ever made the leap into authoring in XHTML.<sup id="cite_ref-hickson_33-0" class="reference"><a href="#cite_note-hickson-33">&#91;33&#93;</a></sup><sup id="cite_ref-avk_34-0" class="reference"><a href="#cite_note-avk-34">&#91;34&#93;</a></sup><sup id="cite_ref-stachowiak_35-0" class="reference"><a href="#cite_note-stachowiak-35">&#91;35&#93;</a></sup> Others countered that the problems ascribed to the use of XHTML could mostly be attributed to two main sources: the production of invalid XHTML documents by some Web authors and the lack of support for XHTML built into <a href="/wiki/Internet_Explorer_6" title="Internet Explorer 6">Internet Explorer 6</a>.<sup id="cite_ref-36" class="reference"><a href="#cite_note-36">&#91;36&#93;</a></sup><sup id="cite_ref-37" class="reference"><a href="#cite_note-37">&#91;37&#93;</a></sup> They went on to describe the benefits of XML-based Web documents (i.e. XHTML) regarding searching, indexing and parsing as well as future-proofing the Web itself.\n</p><p>In October 2006, HTML inventor and W3C chair <a href="/wiki/Tim_Berners-Lee" title="Tim Berners-Lee">Tim Berners-Lee</a>, introducing a major W3C effort to develop a new HTML specification, posted in his blog that, "The attempt to get the world to switch to XML \xe2\x80\xa6 all at once didn\'t work. The large HTML-generating public did not move \xe2\x80\xa6 Some large communities did shift and are enjoying the fruits of well-formed systems \xe2\x80\xa6 The plan is to charter a completely new HTML group."<sup id="cite_ref-38" class="reference"><a href="#cite_note-38">&#91;38&#93;</a></sup> The current HTML5 working draft says "special attention has been given to defining clear conformance criteria for user agents in an effort to improve interoperability \xe2\x80\xa6 while at the same time updating the HTML specifications to address issues raised in the past few years." <a href="/wiki/Ian_Hickson" title="Ian Hickson">Ian Hickson</a>, editor of the HTML5 specification criticising the improper use of XHTML in 2002,<sup id="cite_ref-hickson_33-1" class="reference"><a href="#cite_note-hickson-33">&#91;33&#93;</a></sup> is a member of the group developing this specification and is listed as one of the co-editors of the current working draft.<sup id="cite_ref-html5-WD_39-0" class="reference"><a href="#cite_note-html5-WD-39">&#91;39&#93;</a></sup>\n</p><p>Simon Pieters researched the XML-compliance of mobile browsers<sup id="cite_ref-zcorpan_40-0" class="reference"><a href="#cite_note-zcorpan-40">&#91;40&#93;</a></sup> and concluded \xe2\x80\x9cthe claim that XHTML would be needed for mobile devices is simply a myth\xe2\x80\x9d.\n</p>\n<h2><span class="mw-headline" id="Versions_of_XHTML">Versions of XHTML</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=6" title="Edit section: Versions of XHTML">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<h3><span class="mw-headline" id="XHTML_1.0">XHTML 1.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=7" title="Edit section: XHTML 1.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div class="thumb tright"><div class="thumbinner" style="width:302px;"><a href="/wiki/File:Wikipedia_main_page_XHTML_source.png" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/e5/Wikipedia_main_page_XHTML_source.png/300px-Wikipedia_main_page_XHTML_source.png" decoding="async" width="300" height="140" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/e5/Wikipedia_main_page_XHTML_source.png/450px-Wikipedia_main_page_XHTML_source.png 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/e5/Wikipedia_main_page_XHTML_source.png/600px-Wikipedia_main_page_XHTML_source.png 2x" data-file-width="1050" data-file-height="491" /></a>  <div class="thumbcaption"><div class="magnify"><a href="/wiki/File:Wikipedia_main_page_XHTML_source.png" class="internal" title="Enlarge"></a></div>In earlier times<sup class="noprint Inline-Template" style="white-space:nowrap;">&#91;<i><a href="/wiki/Wikipedia:Manual_of_Style/Dates_and_numbers#Chronological_items" title="Wikipedia:Manual of Style/Dates and numbers"><span title="The time period mentioned near this tag is ambiguous. (March 2013)">when?</span></a></i>&#93;</sup>, <a href="/wiki/Wikipedia" title="Wikipedia">Wikipedia</a> used the XHTML 1.0 Transitional doctype and syntax, though it was not served as XHTML</div></div></div>\n<p>December 1998 saw the publication of a W3C Working Draft entitled <i>Reformulating HTML in XML</i>. This introduced Voyager, the codename for a new markup language based on HTML 4, but adhering to the stricter syntax rules of XML. By February 1999 the name of the specification had changed to <i>XHTML 1.0: The Extensible HyperText Markup Language</i>, and in January 2000 it was officially adopted as a W3C Recommendation.<sup id="cite_ref-xhtml-1.0_41-0" class="reference"><a href="#cite_note-xhtml-1.0-41">&#91;41&#93;</a></sup> There are three formal <a href="/wiki/Document_Type_Definition" class="mw-redirect" title="Document Type Definition">DTDs</a> for XHTML 1.0, corresponding to the three different versions of HTML 4.01:\n</p>\n<ul><li><b>XHTML 1.0 Strict</b> is the XML equivalent to strict HTML 4.01, and includes elements and attributes that have not been marked deprecated in the HTML 4.01 specification. As of November&#160;2015<sup class="plainlinks noexcerpt noprint asof-tag update" style="display:none;"><a class="external text" href="https://en.wikipedia.org/w/index.php?title=XHTML&amp;action=edit">&#91;update&#93;</a></sup>, XHTML 1.0 Strict is the document type used for the homepage of the website of the <a href="/wiki/World_Wide_Web_Consortium" title="World Wide Web Consortium">World Wide Web Consortium</a>.</li>\n<li><b>XHTML 1.0 Transitional</b> is the XML equivalent of HTML 4.01 Transitional, and includes the presentational elements (such as <code>center</code>, <code>font</code> and <code>strike</code>) excluded from the strict version.</li>\n<li><b>XHTML 1.0 Frameset</b> is the XML equivalent of HTML 4.01 Frameset, and allows for the definition of <a href="/wiki/Frameset_(HTML)" class="mw-redirect" title="Frameset (HTML)">frameset documents</a>\xe2\x80\x94a common Web feature in the late 1990s.</li></ul>\n<p>The second edition of XHTML 1.0 became a W3C Recommendation in August 2002.<sup id="cite_ref-xhtml-1.0-se_42-0" class="reference"><a href="#cite_note-xhtml-1.0-se-42">&#91;42&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Modularization_of_XHTML">Modularization of XHTML</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=8" title="Edit section: Modularization of XHTML">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p><a href="/wiki/XHTML_Modularization" title="XHTML Modularization">Modularization</a> provides an abstract collection of components through which XHTML can be subsetted and extended. The feature is intended to help XHTML extend its reach onto emerging platforms, such as mobile devices and Web-enabled televisions. The initial draft of <i>Modularization of XHTML</i> became available in April 1999, and reached Recommendation status in April 2001.<sup id="cite_ref-modularization-1.0_43-0" class="reference"><a href="#cite_note-modularization-1.0-43">&#91;43&#93;</a></sup>\n</p><p>The first modular XHTML variants were XHTML 1.1 and XHTML Basic 1.0.\n</p><p>In October 2008 <i>Modularization of XHTML</i> was superseded by <i>XHTML Modularization 1.1</i>, which adds an <a href="/wiki/XML_Schema_(W3C)" title="XML Schema (W3C)">XML Schema</a> implementation. It was itself superseded by a second edition in July 2010.<sup id="cite_ref-modularization-1.1_44-0" class="reference"><a href="#cite_note-modularization-1.1-44">&#91;44&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="XHTML_1.1:_Module-based_XHTML">XHTML 1.1: Module-based XHTML</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=9" title="Edit section: XHTML 1.1: Module-based XHTML">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>XHTML 1.1 evolved out of the work surrounding the initial <i>Modularization of XHTML</i> specification. The W3C released a first draft in September 1999; Recommendation status was reached in May 2001.<sup id="cite_ref-xhtml-1.1_45-0" class="reference"><a href="#cite_note-xhtml-1.1-45">&#91;45&#93;</a></sup> The modules combined within XHTML 1.1 effectively recreate XHTML 1.0 Strict, with the addition of <a href="/wiki/Ruby_character" title="Ruby character">ruby annotation</a> elements (<code>ruby</code>, <code>rbc</code>, <code>rtc</code>, <code>rb</code>, <code>rt</code> and <code>rp</code>) to better support East-Asian languages. Other changes include removal of the <code>name</code> attribute from the <code>a</code> and <code>map</code> elements, and (in the first edition of the language) removal of the <code>lang</code> attribute in favour of <code>xml:lang</code>.\n</p><p>Although XHTML 1.1 is largely compatible with XHTML 1.0 and HTML 4, in August 2002 the Working Group issued a formal Note advising that it should not be transmitted with the HTML media type.<sup id="cite_ref-xhtml-media-2002_46-0" class="reference"><a href="#cite_note-xhtml-media-2002-46">&#91;46&#93;</a></sup> With limited browser support for the alternate <code>application/xhtml+xml</code> media type, XHTML 1.1 proved unable to gain widespread use. In January 2009 a second edition of the document (<i>XHTML Media Types \xe2\x80\x93 Second Edition</i>) was issued, relaxing this restriction and allowing XHTML 1.1 to be served as <code>text/html</code>.<sup id="cite_ref-xhtml-media-2009_47-0" class="reference"><a href="#cite_note-xhtml-media-2009-47">&#91;47&#93;</a></sup>\n</p><p>A second edition of XHTML 1.1 was issued on 23 November 2010, which addresses various errata and adds an XML Schema implementation not included in the original specification.<sup id="cite_ref-48" class="reference"><a href="#cite_note-48">&#91;48&#93;</a></sup> (It was first released briefly on 7 May 2009 as a "Proposed Edited Recommendation"<sup id="cite_ref-xhtml-1.1-se_49-0" class="reference"><a href="#cite_note-xhtml-1.1-se-49">&#91;49&#93;</a></sup> before being rescinded on 19 May due to unresolved issues.)\n</p>\n<h3><span class="mw-headline" id="XHTML_Basic">XHTML Basic</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=10" title="Edit section: XHTML Basic">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/XHTML_Basic" title="XHTML Basic">XHTML Basic</a></div>\n<p>Since <a href="/wiki/Information_appliance" title="Information appliance">information appliances</a> may lack the <a href="/wiki/Resource_(computer_science)" class="mw-redirect" title="Resource (computer science)">system resources</a> to implement all XHTML abstract modules, the W3C defined a feature-limited XHTML specification called XHTML Basic. It provides a minimal feature subset sufficient for the most common content-authoring. The specification became a <a href="/wiki/W3C_recommendation" class="mw-redirect" title="W3C recommendation">W3C recommendation</a> on December 2000.<sup id="cite_ref-xhtml-basic-1.0_50-0" class="reference"><a href="#cite_note-xhtml-basic-1.0-50">&#91;50&#93;</a></sup>\n</p><p>Of all the versions of XHTML, XHTML Basic 1.0 provides the fewest features. With XHTML 1.1, it is one of the two first implementations of modular XHTML. In addition to the Core Modules (Structure, Text, Hypertext, and List), it implements the following abstract modules: Base, Basic Forms, Basic Tables, Image, Link, Metainformation, Object, Style Sheet, and Target.<sup id="cite_ref-51" class="reference"><a href="#cite_note-51">&#91;51&#93;</a></sup><sup id="cite_ref-52" class="reference"><a href="#cite_note-52">&#91;52&#93;</a></sup>\n</p><p>XHTML Basic 1.1 replaces the Basic Forms Module with the Forms Module, and adds the Intrinsic Events, Presentation, and Scripting modules. It also supports additional tags and attributes from other modules. This version became a W3C recommendation on 29 July 2008.<sup id="cite_ref-53" class="reference"><a href="#cite_note-53">&#91;53&#93;</a></sup>\n</p><p>The current version of XHTML Basic is 1.1 Second Edition (23 November 2010), in which the language is re-implemented in the W3C\'s <a href="/wiki/XML_Schema_(W3C)" title="XML Schema (W3C)">XML Schema</a> language. This version also supports the <code>lang</code> attribute.<sup id="cite_ref-54" class="reference"><a href="#cite_note-54">&#91;54&#93;</a></sup>\n</p>\n<h4><span class="mw-headline" id="XHTML-Print">XHTML-Print</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=11" title="Edit section: XHTML-Print">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>XHTML-Print, which became a W3C Recommendation in September 2006, is a specialized version of XHTML Basic designed for documents printed from information appliances to low-end <a href="/wiki/Printer_(computing)" title="Printer (computing)">printers</a>.<sup id="cite_ref-xhtml-print_55-0" class="reference"><a href="#cite_note-xhtml-print-55">&#91;55&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="XHTML_Mobile_Profile">XHTML Mobile Profile</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=12" title="Edit section: XHTML Mobile Profile">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/XHTML_Mobile_Profile" title="XHTML Mobile Profile">XHTML Mobile Profile</a></div>\n<p>XHTML Mobile Profile (abbreviated XHTML MP or XHTML-MP) is a third-party variant of the W3C\'s XHTML Basic specification. Like XHTML Basic, XHTML was developed for information appliances with limited system resources.\n</p><p>In October 2001, a <a href="/wiki/Limited_company" title="Limited company">limited company</a> called the Wireless Application Protocol Forum began adapting XHTML Basic for <a href="/wiki/WAP_2.0" class="mw-redirect" title="WAP 2.0">WAP 2.0</a>, the second major version of the <a href="/wiki/Wireless_Application_Protocol" title="Wireless Application Protocol">Wireless Application Protocol</a>. WAP Forum based their DTD on the W3C\'s Modularization of XHTML, incorporating the same modules the W3C used in XHTML Basic 1.0\xe2\x80\x94except for the Target Module. Starting with this foundation, the WAP Forum replaced the Basic Forms Module with a partial implementation of the Forms Module, added partial support for the Legacy and Presentation modules, and added full support for the Style Attribute Module.\n</p><p>In 2002, the WAP Forum was subsumed into the <a href="/wiki/Open_Mobile_Alliance" title="Open Mobile Alliance">Open Mobile Alliance</a> (OMA), which continued to develop XHTML Mobile Profile as a component of their OMA Browsing Specification.\n</p>\n<h4><span class="mw-headline" id="XHTML_Mobile_Profile_1.1">XHTML Mobile Profile 1.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=13" title="Edit section: XHTML Mobile Profile 1.1">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>To this version, finalized in 2004, the OMA added partial support for the Scripting Module, and partial support for Intrinsic Events. XHTML MP 1.1 is part of v2.1 of the OMA Browsing Specification (1 November 2002).<sup id="cite_ref-oma-browsing-archive_56-0" class="reference"><a href="#cite_note-oma-browsing-archive-56">&#91;56&#93;</a></sup>\n</p>\n<h4><span class="mw-headline" id="XHTML_Mobile_Profile_1.2">XHTML Mobile Profile 1.2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=14" title="Edit section: XHTML Mobile Profile 1.2">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>This version, finalized 27 February 2007, expands the capabilities of XHTML MP 1.1 with full support for the Forms Module and OMA Text Input Modes. XHTML MP 1.2 is part of v2.3 of the OMA Browsing Specification (13 March 2007).<sup id="cite_ref-oma-browsing-archive_56-1" class="reference"><a href="#cite_note-oma-browsing-archive-56">&#91;56&#93;</a></sup>\n</p>\n<h4><span class="mw-headline" id="XHTML_Mobile_Profile_1.3">XHTML Mobile Profile 1.3</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=15" title="Edit section: XHTML Mobile Profile 1.3">edit</a><span class="mw-editsection-bracket">]</span></span></h4>\n<p>XHTML MP 1.3 (finalized on 23 September 2008) uses the XHTML Basic 1.1 <a href="/wiki/Document_type_definition" title="Document type definition">document type definition</a>, which includes the Target Module. Events in this version of the specification are updated to DOM Level 3 specifications (i.e., they are platform- and language-neutral).\n</p>\n<h3><span class="mw-headline" id="XHTML_1.2">XHTML 1.2</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=16" title="Edit section: XHTML 1.2">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The XHTML 2 Working Group considered the creation of a new language based on XHTML 1.1.<sup id="cite_ref-57" class="reference"><a href="#cite_note-57">&#91;57&#93;</a></sup> If XHTML 1.2 was created, it would include <a href="/wiki/WAI-ARIA" title="WAI-ARIA">WAI-ARIA</a> and <code>role</code> attributes to better support accessible web applications, and improved <a href="/wiki/Semantic_Web" title="Semantic Web">Semantic Web</a> support through <a href="/wiki/RDFa" title="RDFa">RDFa</a>. The <code>inputmode</code> attribute from XHTML Basic 1.1, along with the <code>target</code> attribute (for specifying <a href="/wiki/Framing_(World_Wide_Web)" class="mw-redirect" title="Framing (World Wide Web)">frame</a> targets) might also be present. The XHTML2 WG had not been chartered to carry out the development of XHTML1.2. Since the W3C announced that it does not intend to recharter the XHTML2 WG,<sup id="cite_ref-XHTML_FAQ_6-1" class="reference"><a href="#cite_note-XHTML_FAQ-6">&#91;6&#93;</a></sup> and closed the WG in December 2010, this means that XHTML 1.2 proposal would not eventuate.\n</p>\n<h3><span class="mw-headline" id="XHTML_2.0">XHTML 2.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=17" title="Edit section: XHTML 2.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Between August 2002 and July 2006, the W3C released eight Working Drafts of XHTML 2.0, a new version of XHTML able to make a clean break from the past by discarding the requirement of backward compatibility. This lack of compatibility with XHTML 1.x and HTML 4 caused some early controversy in the web developer community.<sup id="cite_ref-xhtml-2.0-criticism_58-0" class="reference"><a href="#cite_note-xhtml-2.0-criticism-58">&#91;58&#93;</a></sup> Some parts of the language (such as the <code>role</code> and RDFa attributes) were subsequently split out of the specification and worked on as separate modules, partially to help make the transition from XHTML 1.x to XHTML 2.0 smoother. A ninth draft of XHTML 2.0 was expected to appear in 2009, but on July 2, 2009, the W3C decided to let the XHTML2 Working Group charter expire by that year\'s end, effectively halting any further development of the draft into a standard.<sup id="cite_ref-XHTML_FAQ_6-2" class="reference"><a href="#cite_note-XHTML_FAQ-6">&#91;6&#93;</a></sup>  Instead, XHTML 2.0 and its related documents were released as <a href="/wiki/W3C#Recommendations_and_Certifications" class="mw-redirect" title="W3C">W3C Notes</a> in 2010.<sup id="cite_ref-xhtml-2.0-note_59-0" class="reference"><a href="#cite_note-xhtml-2.0-note-59">&#91;59&#93;</a></sup><sup id="cite_ref-xhtml2-wg-notes_60-0" class="reference"><a href="#cite_note-xhtml2-wg-notes-60">&#91;60&#93;</a></sup>\n</p><p>New features to have been introduced by XHTML 2.0 included:\n</p>\n<ul><li>HTML forms were to be replaced by <a href="/wiki/XForms" title="XForms">XForms</a>, an XML-based user input specification allowing forms to be displayed appropriately for different rendering devices.</li>\n<li>HTML frames were to be replaced by <a href="/wiki/XFrames" title="XFrames">XFrames</a>.</li>\n<li>The <a href="/wiki/DOM_Events" class="mw-redirect" title="DOM Events">DOM Events</a> were to be replaced by <a href="/wiki/XML_Events" title="XML Events">XML Events</a>, which uses the XML <a href="/wiki/Document_Object_Model" title="Document Object Model">Document Object Model</a>.</li>\n<li>A new list element type, the <code>nl</code> element type, were to be included to specifically designate a list as a navigation list. This would have been useful in creating nested menus, which are currently created by a wide variety of means like nested unordered lists or nested definition lists.</li>\n<li>Any element was to be able to act as a <a href="/wiki/Hyperlink" title="Hyperlink">hyperlink</a>, e.&#160;g., <code>&lt;li href="articles.html"&gt;Articles&lt;/li&gt;</code>, similar to <a href="/wiki/XLink" title="XLink">XLink</a>. However, XLink itself is not compatible with XHTML due to design differences.</li>\n<li>Any element was to be able to reference alternative media with the <code>src</code> attribute, e.&#160;g., <code>&lt;p src="lbridge.jpg" type="image/jpeg"&gt;London Bridge&lt;/p&gt;</code> is the same as <code>&lt;object src="lbridge.jpg" type="image/jpeg"&gt;&lt;p&gt;London Bridge&lt;/p&gt;&lt;/object&gt;</code>.</li>\n<li>The <code>alt</code> attribute of the <code>img</code> element was removed: alternative text was to be given in the content of the <code>img</code> element, much like the <code>object</code> element, e.&#160;g., <code>&lt;img src="hms_audacious.jpg"&gt;HMS &lt;span class="italic"&gt;Audacious&lt;/span&gt;&lt;/img&gt;</code>.</li>\n<li>A single heading element (<code>h</code>) was added. The level of these headings was determined by the depth of the nesting. This would have allowed the use of headings to be infinite, rather than limiting use to six levels deep.</li>\n<li>The remaining presentational elements <code>i</code>, <code>b</code> and <code>tt</code>, still allowed in XHTML 1.x (even Strict), were to be absent from XHTML 2.0. The only somewhat presentational elements remaining were to be <code>sup</code> and <code>sub</code> for superscript and subscript respectively, because they have significant non-presentational uses and are required by certain languages. All other tags were meant to be <a href="/wiki/Semantic_HTML" title="Semantic HTML">semantic</a> instead (e.&#160;g. <code>strong</code> for <b>strong emphasis</b>) while allowing the user agent to control the presentation of elements via CSS (e.g. rendered as boldface text in most visual browsers, but possibly rendered with changes of tone in a text-to-speech reader, larger + italic font per rules in a user-end stylesheet, etc.).</li>\n<li>The addition of RDF triple with the <code>property</code> and <code>about</code> attributes to facilitate the conversion from XHTML to RDF/XML.</li></ul>\n<h3><span class="mw-headline" id="XHTML5">XHTML5</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=18" title="Edit section: XHTML5">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/HTML5" title="HTML5">HTML5</a></div>\n<p><a href="/wiki/HTML5" title="HTML5">HTML5</a> grew independently of the W3C, through a loose group of browser manufacturers and other interested parties calling themselves the <a href="/wiki/WHATWG" title="WHATWG">WHATWG</a>, or Web Hypertext Application Technology Working Group. The key motive of the group was to create a platform for dynamic web applications; they considered XHTML 2.0 to be too document-centric, and not suitable for the creation of <a href="/wiki/Internet_forum" title="Internet forum">internet forum</a> sites or online shops.<sup id="cite_ref-html-5.0-relationship_61-0" class="reference"><a href="#cite_note-html-5.0-relationship-61">&#91;61&#93;</a></sup>\n</p><p>HTML5 has both a regular <code>text/html</code> serialization and an XML serialization, which is also known as <a href="/wiki/XHTML5" class="mw-redirect" title="XHTML5">XHTML5</a>.<sup id="cite_ref-w3cxhtml5_62-0" class="reference"><a href="#cite_note-w3cxhtml5-62">&#91;62&#93;</a></sup> The language is more compatible with HTML 4 and XHTML 1.x than XHTML 2.0, due to the decision to keep the existing HTML form elements and events model. It adds many new elements not found in XHTML 1.x, however, such as <code>section</code> and <code>aside</code> tags.\n</p><p>The XHTML5 language, like HTML5, uses a <a href="/wiki/DOCTYPE" class="mw-redirect" title="DOCTYPE">DOCTYPE</a> declaration without a DTD. Furthermore, the specification deprecates earlier XHTML DTDs by asking the browsers to replace them with one containing only entity definitions for named characters during parsing.<sup id="cite_ref-w3cxhtml5_62-1" class="reference"><a href="#cite_note-w3cxhtml5-62">&#91;62&#93;</a></sup>\n</p>\n<h2><span class="mw-headline" id="Semantic_content_in_XHTML">Semantic content in XHTML</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=19" title="Edit section: Semantic content in XHTML">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p><a href="/wiki/XHTML%2BRDFa" title="XHTML+RDFa">XHTML+RDFa</a> is an extended version of the XHTML markup language for supporting <a href="/wiki/Resource_Description_Framework" title="Resource Description Framework">RDF</a> through a collection of attributes and processing rules in the form of <a href="/wiki/Well-formed_XML#Well-formedness_and_error-handling" class="mw-redirect" title="Well-formed XML">well-formed XML</a> documents. This host language is one of the techniques used to develop <a href="/wiki/Semantic_Web" title="Semantic Web">Semantic Web</a> content by embedding rich semantic markup.\n</p>\n<h2><span class="mw-headline" id="Valid_XHTML_documents">Valid XHTML documents</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=20" title="Edit section: Valid XHTML documents">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<table class="box-Unreferenced_section plainlinks metadata ambox ambox-content ambox-Unreferenced" role="presentation"><tbody><tr><td class="mbox-image"><div style="width:52px"><a href="/wiki/File:Question_book-new.svg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/50px-Question_book-new.svg.png" decoding="async" width="50" height="39" srcset="//upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/75px-Question_book-new.svg.png 1.5x, //upload.wikimedia.org/wikipedia/en/thumb/9/99/Question_book-new.svg/100px-Question_book-new.svg.png 2x" data-file-width="512" data-file-height="399" /></a></div></td><td class="mbox-text"><div class="mbox-text-span">This section <b>does not <a href="/wiki/Wikipedia:Citing_sources" title="Wikipedia:Citing sources">cite</a> any <a href="/wiki/Wikipedia:Verifiability" title="Wikipedia:Verifiability">sources</a></b>.<span class="hide-when-compact"> Please help <a class="external text" href="https://en.wikipedia.org/w/index.php?title=XHTML&amp;action=edit">improve this section</a> by <a href="/wiki/Help:Referencing_for_beginners" title="Help:Referencing for beginners">adding citations to reliable sources</a>. Unsourced material may be challenged and <a href="/wiki/Wikipedia:Verifiability#Burden_of_evidence" title="Wikipedia:Verifiability">removed</a>.</span>  <small class="date-container"><i>(<span class="date">July 2013</span>)</i></small><small class="hide-when-compact"><i> (<a href="/wiki/Help:Maintenance_template_removal" title="Help:Maintenance template removal">Learn how and when to remove this template message</a>)</i></small></div></td></tr></tbody></table>\n<p>An XHTML document that conforms to an XHTML specification is said to be <i>valid</i>. Validity assures consistency in document code, which in turn eases processing, but does not necessarily ensure consistent rendering by browsers. A document can be checked for validity with the <a href="/wiki/W3C_Markup_Validation_Service" title="W3C Markup Validation Service">W3C Markup Validation Service</a> (for XHTML5, the Validator.nu Living Validator should be used instead). In practice, many web development programs provide code validation based on the <a href="/wiki/W3C" class="mw-redirect" title="W3C">W3C</a> standards.\n</p>\n<h3><span class="mw-headline" id="Root_element">Root element</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=21" title="Edit section: Root element">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The root element of an XHTML document must be <code>html</code>, and must contain an <code>xmlns</code> attribute to associate it with the XHTML <a href="/wiki/XML_namespace" title="XML namespace">namespace</a>. The namespace URI for XHTML is <code>http://www.w3.org/1999/xhtml</code>. The example tag below additionally features an <code>xml:lang</code> attribute to identify the document with a <a href="/wiki/Natural_language" title="Natural language">natural language</a>:\n</p>\n<div class="mw-highlight mw-highlight-lang-xml mw-content-ltr" dir="ltr"><pre><span></span><span class="nt">&lt;html</span> <span class="na">xmlns=</span><span class="s">&quot;http://www.w3.org/1999/xhtml&quot;</span> <span class="na">xml:lang=</span><span class="s">&quot;ar&quot;</span><span class="nt">&gt;</span>\n</pre></div>\n<h3><span class="mw-headline" id="DOCTYPEs">DOCTYPEs</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=22" title="Edit section: DOCTYPEs">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">Main article: <a href="/wiki/DOCTYPE" class="mw-redirect" title="DOCTYPE">DOCTYPE</a></div>\n<p>In order to validate an XHTML document, a <a href="/wiki/Document_Type_Declaration" class="mw-redirect" title="Document Type Declaration">Document Type Declaration</a>, or <i>DOCTYPE</i>, may be used. A DOCTYPE declares to the browser the <a href="/wiki/Document_Type_Definition" class="mw-redirect" title="Document Type Definition">Document Type Definition</a> (DTD) to which the document conforms. A Document Type Declaration should be placed before the <a href="/wiki/Root_element" title="Root element">root element</a>.\n</p><p>The <a href="/wiki/System_identifier" title="System identifier">system identifier</a> part of the DOCTYPE, which in these examples is the <a href="/wiki/Uniform_Resource_Locator" class="mw-redirect" title="Uniform Resource Locator">URL</a> that begins with <code>http://</code>, need only point to a copy of the DTD to use, if the validator cannot locate one based on the <a href="/wiki/Public_identifier" class="mw-redirect" title="Public identifier">public identifier</a> (the other quoted string). It does not need to be the specific URL that is in these examples; in fact, authors are encouraged to use local copies of the DTD files when possible. The public identifier, however, must be character-for-character the same as in the examples.\n</p>\n<h3><span class="mw-headline" id="XML_declaration">XML declaration</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=23" title="Edit section: XML declaration">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<div role="note" class="hatnote navigation-not-searchable">See also: <a href="/wiki/Processing_Instruction" title="Processing Instruction">Processing Instruction</a></div>\n<p>A <a href="/wiki/Character_encoding" title="Character encoding">character encoding</a> may be specified at the beginning of an XHTML document in the XML declaration when the document is served using the <code>application/xhtml+xml</code> <a href="/wiki/MIME" title="MIME">MIME</a> type. (If an XML document lacks encoding specification, an XML parser assumes that the encoding is <a href="/wiki/UTF-8" title="UTF-8">UTF-8</a> or <a href="/wiki/UTF-16" title="UTF-16">UTF-16</a>, unless the encoding has already been determined by a higher protocol.)\n</p><p>For example:\n</p>\n<dl><dd><code>&lt;?xml version="1.0" encoding="UTF-8"&#160;?&gt;</code></dd></dl>\n<p>The declaration may be optionally omitted because it declares as its encoding the default encoding. However, if the document instead makes use of XML 1.1 or another character encoding, a declaration is necessary. <a href="/wiki/Internet_Explorer" title="Internet Explorer">Internet Explorer</a> prior to version 7 enters <a href="/wiki/Quirks_mode" title="Quirks mode">quirks mode</a>, if it encounters an XML declaration in a document served as <code>text/html</code>.\n</p>\n<h2><span class="mw-headline" id="Backward_compatibility">Backward compatibility</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=24" title="Edit section: Backward compatibility">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>XHTML 1.x documents are mostly backward compatible with HTML 4 user agents when the appropriate guidelines are followed. XHTML 1.1 is essentially compatible, although the elements for <a href="/wiki/Ruby_character" title="Ruby character">ruby annotation</a> are not part of the HTML 4 specification and thus generally ignored by HTML 4 browsers. Later XHTML 1.x modules such as those for the <code>role</code> attribute, <a href="/wiki/RDFa" title="RDFa">RDFa</a> and <a href="/wiki/WAI-ARIA" title="WAI-ARIA">WAI-ARIA</a> degrade gracefully in a similar manner.\n</p><p>XHTML 2.0 is significantly less compatible, although this can be mitigated to some degree through the use of scripting. (This can be simple one-liners, such as the use of \xe2\x80\x9c<code>document.createElement()</code>\xe2\x80\x9d to register a new HTML element within Internet Explorer, or complete JavaScript frameworks, such as the <a href="/wiki/FormFaces" class="mw-redirect" title="FormFaces">FormFaces</a> implementation of <a href="/wiki/XForms" title="XForms">XForms</a>.)\n</p>\n<h3><span class="mw-headline" id="Examples">Examples</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=XHTML&amp;action=edit&amp;section=25" title="Edit section: Examples">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>The following are examples of XHTML 1.0 Strict, with both having the same visual output.
