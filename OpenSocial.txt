href="/wiki/Application_programming_interface" class="mw-redirect" title="Application programming interface">application programming interfaces</a> (APIs) for <a href="/wiki/World_Wide_Web" title="World Wide Web">Web</a>-based applications. Initially, it was designed for <a href="/wiki/Social_network_service" class="mw-redirect" title="Social network service">social network applications</a> and developed by <a href="/wiki/Google" title="Google">Google</a> along with <a href="/wiki/MySpace" class="mw-redirect" title="MySpace">MySpace</a> and a number of other social networks. More recently, it has become adopted as a general use <a href="/wiki/Runtime_environment" class="mw-redirect" title="Runtime environment">runtime environment</a> for allowing untrusted and partially trusted components from third parties to run in an existing web application. The OpenSocial Foundation moved to integrate or support numerous other <a href="/wiki/Open_Web" class="mw-redirect" title="Open Web">Open Web</a> technologies. This includes <a href="/wiki/OAuth" title="OAuth">OAuth</a> and OAuth 2.0, <a href="/wiki/Activity_Streams_(format)" title="Activity Streams (format)">Activity Streams</a>, and <a href="/wiki/Portable_Contacts" title="Portable Contacts">Portable Contacts</a>, among others.\n</p><p>It was released on November 1, 2007.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1">&#91;1&#93;</a></sup> \nApplications implementing the OpenSocial APIs are <a href="/wiki/Interoperability" title="Interoperability">interoperable</a> with any social network system that supports them. At launch, OpenSocial took a one-size-fits-all approach to development. As it became more robust and the user-base expanded, OpenSocial modularized the platform to allow developers to only include the parts of the platform it needed.<sup id="cite_ref-OS100_2-0" class="reference"><a href="#cite_note-OS100-2">&#91;2&#93;</a></sup>\n</p><p>On December 16, 2014 the <a href="/wiki/W3C" class="mw-redirect" title="W3C">W3C</a> issued a press release, "OpenSocial Foundation Moving Standards Work to W3C Social Web Activity",<sup id="cite_ref-press_release-move_3-0" class="reference"><a href="#cite_note-press_release-move-3">&#91;3&#93;</a></sup> stating that OpenSocial would no longer exist as a separate entity and encouraged the OpenSocial community to continue development work through the <a rel="nofollow" class="external text" href="http://www.w3.org/Social/">W3C Social Web Activity</a> in the <a rel="nofollow" class="external text" href="http://www.w3.org/Social/WG">Social Web Working Group</a> and <a rel="nofollow" class="external text" href="http://www.w3.org/Social/IG">Social Interest Group</a>. The OpenSocial Foundation stated that "the community will have a better chance of realizing an open social web through discussions at a single organization, and the OpenSocial Foundation board believes that working as an integrated part of W3C will help reach more communities that will benefit from open social standards."<sup id="cite_ref-newHomepage_4-0" class="reference"><a href="#cite_note-newHomepage-4">&#91;4&#93;</a></sup> On January 1, 2015, <a rel="nofollow" class="external text" href="http://www.opensocial.org">opensocial.org</a> began redirecting all page class="thumbinner" style="width:402px;"><a href="/wiki/File:Opensocial.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/8/84/Opensocial.jpg/400px-Opensocial.jpg" class="internal" title="Enlarge"></a></div>Structure of OpenSocial<sup id="cite_ref-5" class="reference"><a href="#cite_note-5">&#91;5&#93;</a></sup></div></div></div>\n<p>Based on <a href="/wiki/HTML" title="HTML">HTML</a> and <a href="/wiki/JavaScript" title="JavaScript">JavaScript</a>, as well as the <a href="/wiki/Google_Gadgets" title="Google Gadgets">Google Gadgets</a> framework, OpenSocial includes multiple APIs for <a href="/wiki/Social_software" title="Social software">social software</a> applications to access data and core functions on participating <a href="/wiki/Social_network" title="Social network">social networks</a>.<sup id="cite_ref-6" class="reference"><a href="#cite_note-6">&#91;6&#93;</a></sup> Each API addresses a different aspect.<sup id="cite_ref-7" class="reference"><a href="#cite_note-7">&#91;7&#93;</a></sup> It also includes APIs for contacting arbitrary third party services on the web using a <a href="/wiki/Proxy_server" title="Proxy server">proxy</a> system and OAuth for security.\n</p><p>In version 0.9 OpenSocial added support for a <a href="/wiki/Markup_language" title="Markup language">tag-based language</a>.<sup id="cite_ref-:0_8-0" class="reference"><a href="#cite_note-:0-8">&#91;8&#93;</a></sup>  This language is referred to as OSML and allows tag-based access to data from the OpenSocial APIs that previously required an asynchronous client-side request. It also defined a rich tag template system and adopted an expression language loosely based on the <a href="/wiki/Java_(programming_language)" title="Java (programming language)">Java</a> <a href="/wiki/Unified_Expression_Language" class="mw-redirect" title="Unified Expression Language">Expression Language</a>.\n</p><p>Starting in version 2.0, OpenSocial adopted support for <a href="/wiki/Activity_Streams_(format)" title="Activity Streams (format)">Activity Streams</a> format.<sup id="cite_ref-:0_8-1" class="reference"><a href="#cite_note-:0-8">&#91;8&#93;</a></sup>\n</p>\n<h2><span class="mw-headline" id="History">History</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a Background">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>OpenSocial is commonly described as a more open <a href="/wiki/Cross-platform" class="mw-redirect" title="Cross-platform">cross-platform</a> alternative to the <a href="/wiki/Facebook_Platform" title="Facebook Platform">Facebook Platform</a>, a proprietary service of the popular <a href="/wiki/Social_network_service" class="mw-redirect" title="Social network service">social network service</a> <a href="/wiki/Facebook" title="Facebook">Facebook</a>.<sup id="cite_ref-NY_Times_9-0" class="reference"><a href="#cite_note-NY_Times-9">&#91;9&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Development">Development</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=4" title="Edit section: Development">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>OpenSocial was rumored to be part of a larger social networking initiative by Google code-named "Maka-Maka",<sup id="cite_ref-Makamaka_10-0" class="reference"><a href="#cite_note-Makamaka-10">&#91;10&#93;</a></sup> which is defined as meaning "intimate friend with whom one is on terms of receiving and giving freely" in <a href="/wiki/Hawaiian_language" title="Hawaiian language">Hawaiian</a>.<sup id="cite_ref-makamaka_meaning_11-0" class="reference"><a href="#cite_note-makamaka_meaning-11">&#91;11&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Implementation">Implementation</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=5" title="Edit section: Implementation">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>An <a href="/wiki/Open-source_software" title="Open-source software">open-source</a> project, <a href="/wiki/Shindig_(software)" class="mw-redirect" title="Shindig (software)">Shindig</a>, was launched in December 2007, to provide a <a href="/wiki/Reference_implementation" title="Reference implementation">reference implementation</a> of the OpenSocial standards. It has the support of Google, Ning, and other companies developing OpenSocial-related software. It will also provide strong context-aware personalization, collaboration and content integration capabilities and a high quality out-of-the-box installation as well as be easy to integrate in other platforms and solutions.<sup id="cite_ref-12" class="reference"><a href="#cite_note-12">&#91;12&#93;</a></sup>\n</p><p>Both Shindig and Apache Rave are no longer in development and have been retired by the Apache foundation.\n</p>\n<h2><span class="mw-headline" id="Usage">Usage</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=6" title="Edit section: Usage">edit</a><span class="mw-editsection-bracket">]</span></span></h2>\n<p>Enterprise websites, such as <a href="/wiki/Friendster" title="Friendster">Friendster</a>, <a href="/wiki/Hi5" title="Hi5">hi5</a>, <a href="/wiki/LinkedIn" title="LinkedIn">LinkedIn</a>, <a href="/wiki/Myspace" title="Myspace">MySpace</a>, <a href="/wiki/Orkut" title="Orkut">Orkut</a>, and <a href="/wiki/Salesforce.com" class="mw-redirect" title="Salesforce.com">Salesforce.com</a> are major users of OpenSocial.<sup id="cite_ref-13" class="reference"><a href="#cite_note-13">&#91;13&#93;</a></sup>\n</p>\n<h3><span class="mw-headline" id="Friendster">Friendster</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=7" title="Edit section: Friendster">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p><a href="/wiki/Friendster" title="Friendster">Friendster</a> has deployed APIs from version 0.7 of the OpenSocial specification, making it easy for existing OpenSocial applications using version 0.7 to be launched on Friendster and reach Friendster\'s over 75 million users. The total time to crack the OpenSocial-based iLike on Ning was just 20 minutes, with the attacker being able to add and remove songs on a user\'s playlist and access the user\'s friend information.<sup id="cite_ref-18" class="reference"><a href="#cite_note-18">&#91;18&#93;</a></sup>\n</p><p>Hasel and Iacono showed that \xe2\x80\x9cOpenSocial specification were far from being comprehensive in respect to security\xe2\x80\x9d.<sup id="cite_ref-19" class="reference"><a href="#cite_note-19">&#91;19&#93;</a></sup> They discussed different security implications in the context of OpenSocial. JSON had emerged as the preferred data format and support for <a href="/wiki/ATOM" class="mw-redirect" title="ATOM">ATOM</a> was deprecated. The Gadget format was simplified to give the ability to define a template library within a Gadget specification.<sup id="cite_ref-OS200_24-1" class="reference"><a href="#cite_note-OS200-24">&#91;24&#93;</a></sup> While not finalized, the groundwork for OAuth 2.0 support was put in place.\n</p>\n<h3><span class="mw-headline" id="Version_1.1.0">Version 1.1.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=17" title="Edit section: Version 1.1.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>In response to enterprise environment needs, OpenSocial added support for advanced mashup scenarios. It enabled gadgets to "securely message each other in a loosely coupled manner."<sup id="cite_ref-OS110_25-1" class="reference"><a href="#cite_note-OS110-25">&#91;25&#93;</a></sup> This new feature was called Inter-Gadget Communication.\n</p>\n<h3><span class="mw-headline" id="Version_1.0.0">Version 1.0.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=18" title="Edit section: Version 1.0.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>OpenSocial acknowledged that the "one-size-fits-all" approach it was taking was not going to work for the diverse type of websites that had adopted the platform. To address this issue, OpenSocial modularized into four compliance modules: Core API Server, Core Gadget Server, Social API Server, and Social Gadget Server.<sup id="cite_ref-OS100_2-2" class="reference"><a href="#cite_note-OS100-2">&#91;2&#93;</a></sup> This allowed a developer to pick and choose the modules they wanted to use while using other services that aren\'t part of OpenSocial. Extensions were introduced to allow developers to extend OpenSocial containers.\n</p>\n<h3><span class="mw-headline" id="Version_0.9.0">Version 0.9.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=19" title="Edit section: Version 0.9.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>In response to feedback and observation of how developers were using the API, this version focused on making "application development, testing, and deployment easier and faster, while reducing the learning curve for new app developers."<sup id="cite_ref-OS090_26-1" class="reference"><a href="#cite_note-OS090-26">&#91;26&#93;</a></sup> The OpenSocial Javascript API was streamlined to make it lightweight while retaining the power of the old Javascript API. Proxied content was introduced to eliminate the need for developers to work around previous AJAX limitations. Proxied content allows a content to be fetched from a URL and displayed in a &lt;Content&gt; tag. In response to a common use of sending data to a remote server immediately after a request, OpenSocial 0.9.0 introduced data pipelining. Data pipelining allows the developer to specify the social data the application will need and make the data immediately available. OpenSocial Templates were introduced to create data-driven UI with a separation of markup and programmatic logic. OpenSocial Markup Language (OSML Markup) is a new set of standardized tags to accomplish common tasks or safely perform normally unsafe operations within templates. OSML is extensible. Developers can create a library of their own custom tags.\n</p>\n<h3><span class="mw-headline" id="Version_0.8.1">Version 0.8.1</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=20" title="Edit section: Version 0.8.1">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>This minor release placed a major focus on server-to-server protocols as "the Person schema has been aligned with the Portable Contacts effort, and an optional RPC proposal has been added."<sup id="cite_ref-OS081_27-1" class="reference"><a href="#cite_note-OS081-27">&#91;27&#93;</a></sup> <a href="/wiki/JSON-RPC" title="JSON-RPC">JSON-RPC</a> protocol was added to increase server to server functionality. The RESTful protocol that was introduced in v0.8.0 underwent a large revision with several fields being added, modified, and deleted.\n</p>\n<h3><span class="mw-headline" id="Version_0.8.0">Version 0.8.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=21" title="Edit section: Version 0.8.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>OpenSocial changed specifications for containers to implement a <a href="/wiki/RESTful_API" class="mw-redirect" title="RESTful API">RESTful API</a>. Many of the OpenSocial Javascript API changes made this version incompatible with previous versions. Existing gadgets continued to use v0.7.0. After updating the gadget, it would use v0.8.0. Security improved with the introduction of OAuth authorization and HTML sanitation, and container lifecycle events.<sup id="cite_ref-OS080_28-1" class="reference"><a href="#cite_note-OS080-28">&#91;28&#93;</a></sup> Persistence data was stored in <a href="/wiki/JSON" title="JSON">JSON</a>.\n</p>\n<h3><span class="mw-headline" id="Version_0.7.0">Version 0.7.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=22" title="Edit section: Version 0.7.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Released as the "first iteration that can fully support rich, social applications."<sup id="cite_ref-OS070_29-1" class="reference"><a href="#cite_note-OS070-29">&#91;29&#93;</a></sup> It added several standard fields for profile information, the ability to send a message to install an application, an Activity template to control activity notifications about what users have been doing, and a simplified persistence API to use feeds instead of global and instance-scoped application data. Another major announcement came from <a href="/wiki/Apache_Shindig" title="Apache Shindig">Apache Shindig</a>. Apache Shindig made gadgets open sourced. In coordination with this announcement, OpenSocial 0.7.0 introduced Gadget Specifications for developers to be able to define their gadgets using the Gadget API.\n</p>\n<h3><span class="mw-headline" id="Version_0.6.0">Version 0.6.0</span><span class="mw-editsection"><span class="mw-editsection-bracket">[</span><a href="/w/index.php?title=OpenSocial&amp;action=edit&amp;section=23" title="Edit section: Version 0.6.0">edit</a><span class="mw-editsection-bracket">]</span></span></h3>\n<p>Security was a large focus in version 0.6.0. Permission controls were tightened to prevent a gadget from returning information if it is not authorized to do so. New classes were added, such as the Environment class to allow a gadget to respond differently according to its environment and the Surface class to support navigation from one surface to another.
